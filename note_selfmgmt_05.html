自己分析とは何か

定義 ／ 注意点 ／ ステップ ／ 効果 ／ ジョハリの窓 ／ 成長のスパイラル

定義と注意点 ← S03:p.5–7
【定義】自己分析

自分を客観的に評価するために行うアプローチのこと。
キーワード：「客観的に」 ── 自分の主観・思い込みを排除して評価することが本質。

🎙 音声補足

自己分析において答えるべき3つの問い：← S05

  • 自分の長所・短所は何か
  • 今までどのようなことをしてきたか
  • 人格形成に影響を与えた出来事は何か ── 過去の試練・挑戦を通じて自分自身が成長していく過程。その瞬間々に多くの学びがある。
自己分析の注意点 ← S03:p.7
  • 過去の経験はことの大小で判断しない（大切なのは中身そのもの）
  • なりたい人物像をイメージして分析しない（本来の自分と異なる分析になる）
  • 「気が小さい」「失敗しがち」等の思い込みを捨てる（自己分析の邪魔になる）
  • 自分の短所・課題・劣っているかもしれない点も排除しない
自己分析のステップ ← S03:p.8
  1. 体験したことを思い出す
    例：レストランで2年間アルバイトをした
  2. 苦労したことに特に注目する
    例：煩雑な作業、ランチ・夕食時間帯のピーク時の多忙
  3. 体験から得たこと・学んだことを振り返る
    例：顧客視点に立った接客の重要性、仲間との協力・工夫
  4. 現在の考え方・感じ方を言語化する
    例：付加価値の重要性、信頼・感謝が人間関係の鍵
🎙 音声補足

苦労した時間帯で「何を工夫したか」「仲間とどう協力したか」「信頼関係・感謝の気持ちが芽生えたか」という観点から振り返ることが重要。苦労した瞬間こそ最も成長している可能性がある。← S05

自己分析の効果 ← S03:p.9
① 自分に自信を持てる

自己評価が低い人でも、分析を進めるうちに自分の強み・誇れる特徴に具体的に気づける。

② 自己肯定感を持てる

「やっていることは間違っていない」「存在に意味がある」という感覚 → モチベーション向上につながる。

③ 自分の中にある解決策に気づく

自己分析で把握した特徴を思い出すことで、他者に頼らず問題解決できる自信がつく。

④ 自分に合わないやり方に追随しない

強みを正確に把握していれば自分らしいアプローチで目標に取り組める。他者の方法をそのまま真似する必要がなくなる。

ジョハリの窓（Johari window） ← S03:p.10–14
【定義】ジョハリの窓

1955年、サンフランシスコ州立大学の心理学者ジョセフ・ルフト（Joseph Luft）とハリントン・インガム（Harrington Ingham）が「グループ成長のためのラボラトリートレーニング」席上で発表した「対人関係における気づきのグラフモデル」。後に二人のファーストネームから「Johari（ジョハリ）」と命名された。
人が円滑なコミュニケーションを進めるために考案されたコミュニケーション分析のモデル。 外部補足・確認済: Wikipedia・カオナビ

コミュニケーション改善の手順 ← S03:p.12–14
  1. 自己開示（Self-disclosure）：「開放」の窓を拡大する
    自分が知っていることをどんどん他人に見せる → 秘密が減り、相手の理解が深まる
  2. フィードバック（Feedback）を受ける：「盲点」を縮小する
    他人から自分についてのフィードバックを得る → 自分が気づいていない特徴がわかる
  3. 自己開示 ＋ フィードバックを繰り返す：「気づき」を得る
    「開放」の窓がどんどん大きくなり、今まで気づいていなかったことに気づく（＝気づき）
成長のスパイラル ← S03:p.15
🎙 音声補足

「気づき」を得ると「これはやるべきだ」というやる気（意欲）が湧く → 具体的な行動に移す → 結果が出る → 自信がつく → さらに意欲が高まる。この好循環が成長のスパイラル。自己開示とフィードバックの繰り返しがこの循環の起点となる。← S05

第1章まとめ

  • 自己分析とは、自分を客観的に評価するアプローチのこと
  • ジョハリの窓はコミュニケーション分析のモデル（1955年、ルフト＋インガム）
  • 自己開示とフィードバックの繰り返しで気づきが増える
  • 気づきが増えると意欲が高まり、成長のスパイラルを築ける

自己分析の活かし方

就職活動への応用 ／ 自己分析が活きる5つの場面

🎙 音声補足

第2章の焦点：第1章で学んだ自己分析の手法を、就職活動（初就職・転職）という実社会の場面でどのように使うか。就活は成長のスパイラルの中に位置づけられる。← S05

就活に自己分析を活かすステップ ← S03:p.22–23
  1. 過去・現在の学習や経験をリストアップ
    学んだこと・経験したことを箇条書きで書き出す
  2. 将来取り組んでみたいことをリストアップ
    ①のリストの横に並べて書く（対比することで気づきが生まれる）
  3. 人間的成長をしたエピソードを詳しく書き留める
    他者に説明するため、わかりやすく・長くなく・複雑すぎない事例。必ず紙に書き留める。
  4. 具体的な出来事から得たもの・学んだことを言語化
    自分の人生観・価値観への影響を具体的な言葉にする
自己分析の注意点（就活特有）← S03:p.23
  • 就職する目的をワンセンテンス・ワンメッセージで箇条書き
  • 目的ごとに「なぜそう思ったか」を3〜4行で補足する
  • 自分の性格・長所・短所を書いたうえで身近な人にフィードバックをもらい、ギャップ・一致点を確認
  • 10年後・20年後の自分の姿を具体的に予測する（どんな人間になりたいか・何を得たいか）
自己分析が就活に活きる5場面 ← S03:p.24–29
場面活かし方のポイント
① 応募業界・企業を絞るとき
  • やりたい仕事がその業界・会社に存在するか
  • やりたい仕事ができなくてもその業界・会社で働き続けられるか
  • 業界・企業文化が自分に合っているか
② 職務経歴書を作成するとき
  • やりたい仕事の適性と自分の強みがマッチしているか
  • 自分の強みを具体的なエピソードで裏付けているか（説得力が増す）
  • 採用側が求める人物像に自分を当てはめられているか
③ 面接質問を受けたとき
  • 自分オリジナルの言葉で答えられているか
  • 1問につき2〜3つの具体的エピソードを準備し、流れで使い分けるか
  • 自分の性格・特徴を前向きな表現で話せているか
④ 企業から内定が出るとき
  • 「その会社で働きたい」という意思表明を明確に伝えられるか
  • 面接を通じて何を学んだかを説明できるか
  • 入社後の展望を自分の言葉で語れるか
⑤ 入社判断をするとき
  • 就活全体を総括し、決断の理由を他者に説明できるか
  • 業界・会社・仕事への理解を具体的に説明できるか
  • 自分の成長意欲・今後の課題を具体的に説明できるか

第2章まとめ

  • 自己分析は、自分の特徴と志望動機に一貫性があると説得力が増す
  • 具体的な自己分析ができていると、就活あらゆる場面で正確な判断ができる
  • 自己分析は新しいことへの挑戦・未来への取り組みで特に有効

KAIZEN活動とは何か

定義 ／ 4ステップ ／ アプローチ ／ 見える化 ／ 3つの眼 ／ 本質・5S ／ 導入注意点

【定義】KAIZEN活動

製造業の世界で、工場の作業者が中心になって行う、作業効率の向上や作業安全性の確保などの生産に関するボトムアップ活動。← S03:p.34
特にトヨタ自動車の生産方式（TPS）の強みとして有名。現在は製造業以外のあらゆる業種にも応用されている。

🌐 外部補足・確認済

「KAIZEN」は英語圏でも日本語のままの世界共通語。漢字「改善」が「悪い点を直す」のに対し、カタカナ「カイゼン」は「現状に満足せず今よりもっと良くする」という前向きな継続改善の意味。大野耐一著『トヨタ生産方式』（1978年）→ 1988年英語版刊行で世界に普及。海外では「リーン生産方式（Lean Manufacturing）」として再整理・普及。 ← GAZOO.com・Skillnote

KAIZEN活動の4ステップ ← S03:p.35

重要ポイント：KAIZENは一度やれば終わりではなく、日常的に繰り返すこと。継続的な正しい意思決定の積み重ねで質を高める。

KAIZEN活動のアプローチ ← S03:p.36
  • その仕事の目的を問う：どうしても必要な仕事か？
  • 全部でなくとも一部を捨てられるところはないか
  • 今のやり方より安く・早くできる方法はないか（コスト・時間の無駄に気づく機会）
  • 部分的に変えるか、プロセス全体が不要なら一気に取り替えるか
KAIZEN活動に必要な3つの「見える化」 ← S03:p.37
① 組織共通認識のための見える化

組織全員が共通認識を持つため。見えていないと、存在することに気づけない。

② 担当者が自律行動するための見える化

担当者自身が正しい判断と行動を取れるようにする。

③ 次のKAIZENにつなげるための見える化

行動した結果を数値等で可視化し次の改善へ活かす。例：3か月の売上目標を数字で見える化 → 次の計画に反映。

→ 「見える化」によって迅速に正しい意思決定を繰り返し行えるようになる。 ← S03:p.37

KAIZEN活動の3つの眼 ← S03:p.38

目的によって見る眼を変える。

🦅

鳥の眼

高い空から全体を見る
→ 全体をつかむ

例：会社の売上・利益・キャッシュフロー全体を把握
🔍

虫の眼

小さく細かく個別を見る
→ 詳細を見る

例：特定のプロジェクト・得意先別・製品別情報の詳細
🐟

魚の眼

海の中を3次元に泳ぐ
→ 流れをつかむ

例：注文→配送→返品→クレーム→要望という一連の流れを把握

🌐 外部補足・確認済

5SはローマジのSeiri / Seiton / Seisou / Seiketsu / Shitsuke の頭文字から命名。英語圏では Sort / Set in Order / Shine / Standardize / Sustain に対応し "Five S" として通用する。日本発の概念だが世界標準となっている。 ← Wikipedia（5S）・ASQ（米国品質協会）

→ 5Sを徹底することで、仕事の中からムリ・ムダ・ムラをなくす。

  • 頭で考えるより、まず動く：実行してみることで見えてくるものがある。「自分には向いていない・できない」という考えはKAIZENの概念にない。
  • 人間の知恵が加わった「動き」が「働き」になる：知恵を入れて改善し、より良い働き方を見つけていく。
  • 問題解決とは根本原因を解決すること：表面をごまかす・部分的解決ではなく。
  • 仕事の状況が一瞬でわかるようにする：見える化の本質。
  • 非常識な目標を立てることで→斬新なアイデアが生まれ→一気に物事が解決することがある。
KAIZEN活動の特徴（補足）

KAIZENは仕事が増えるのではなく、従来からの仕事の習慣を変えること。期限のあるプロジェクトではなく、継続して積み上げていくことで質を高めるもの。また、従業員の考え方・目標の一本化に役立つ。← S03:p.39

KAIZEN導入時の注意点 ← S03:p.42
  • 手順が分からないために必要のない手順を増やしてしまう（手順増加 → 仕事の硬直化・柔軟性の低下）
  • 机上で定めた手順（現場で決めていない手順）は実行できないことが多い → 現場で決めていく
  • 決めた通りにならないことがある → イレギュラーな対応・作業の応用も考えておく
  • 手順書は明文化し、全員で共有する

第3章まとめ

  • KAIZENは従来の仕事の習慣を変えることで継続的に作業効率と品質を向上させる
  • 「仕事の見える化」によって迅速に正しい意思決定ができる
  • KAIZEN活動がきっかけで斬新なアイデアが生まれ、イノベーションを実現できる

KAIZEN活動の活かし方

ツール① 日報 ／ ツール② アンケート ／ ツール③ 業務マニュアル

1
日報 ← S03:p.47–50
【定義】日報

「その日の報告」のこと。ホウ・レン・ソウ（報告・連絡・相談）の具体的な実践ツール。
重要ポイント：報告内容を仕事にどう活かすかが本質。単に書くだけでは意味がない。

日報の目的
  • 仕事の進捗を記録に残す
  • 上司が現場の進捗状況を確認する
  • 異常の兆候を早期発見する
  • 情報を共有する
  • 職場の気づきは日報から始まる
日報の効果
  • 当たり前の日常に改善ヒントが隠れている
  • 過去の仕事を振り返る機会が生まれる
  • 物事を数字でとらえる習慣ができる（変化を追える）
  • どこにムダ・ムラ・ムリがあるかに気づく
  • 上司からのアドバイスを受けるツールになる
🎙 音声補足

日報 → 気づき → 成長のスパイラルへとつながる。「3日前に何をやったか覚えていますか？」→ 日報がなければ覚えていない。毎日の記録が成長の起点となる。← S05

2
アンケート ← S03:p.51–54
【定義】アンケート

フランス語で「調査」を意味する言葉。特定の問題を解決するために、質問項目が記載された調査票・質問用紙を用い、複数の人に同じ質問を行ってデータを収集する活動。コミュニケーションの手段でもある。

質問の作り方
  • 最初は簡単な質問から始める
  • できるだけ数値化する
  • 1問に2つ以上の内容を聞かない
  • 自由回答・複数回答も適度に混ぜる
  • 答えにくい・曖昧な質問は避ける
配信の注意点
  • 目的に合った対象を絞る
  • 回答者の属性を記録する（性別・世代など、後の分析で重要）
  • 有効回答数を設定する
  • 回答期限を明記する
アンケートの効果
  • 相手の潜在的なニーズをつかめる
  • 顧客の満足度を高める
  • 顧客名簿を作り継続的にアプローチできる（2回目・3回目と展開可）
  • 社員のモチベーションアップにつながる
  • アンケートの結果を最重要視する──声が大きい人や偏った意見に振り回されない
3
業務マニュアル ← S03:p.55–58
【定義】業務マニュアル

一定品質の業務を達成するために、処理手順・判断基準・ルールなどを解説するもの。初心者・未経験者への業務説明文書としても使用。個人参照・指導・教育用に活用する。

作成の目的
  • 業務の効率化 → 生産性・顧客満足度の向上
  • コンプライアンス遵守・モラル低下への対処
  • ノウハウを集約させ断絶を避ける
  • グローバル基準への対応
ない場合の問題点
  • 手順が不安定 → 無駄な時間が発生
  • 過去のノウハウが活かされない
  • 仕事の進め方に個人差が生まれる
  • 組織の方針が現場に反映されない
業務マニュアルを使用する4つの効果 ← S03:p.58
🌐 外部補足・確認済（SECIモデルとの接続）

「暗黙知→形式知」の変換は野中郁次郎（一橋大学名誉教授）のSECIモデルの「表出化（Externalization）」に相当する。業務マニュアルの作成はまさにこのプロセス。SECIモデル（1990年、後に竹内弘高との共著『知識創造企業』1995年で世界普及、10か国語以上に翻訳）はナレッジマネジメントの基礎理論。トヨタのKAIZEN活動はSECIモデルの実践例として頻繁に引用される。 ← Wikipedia・グロービス経営大学院

第4章・第5回 まとめ

  • 職場の気づきは日報から生まれ、目的意識の高い仕事が質を高める
  • アンケートはコミュニケーション手段として有効 → 結果を最重要視する
  • 良い業務マニュアルが作成できれば眠っていた知の資産を活かせる
  • 自己分析の効果は気づきを深めることで意欲を高めることにある
  • 自己分析は新しいことに挑戦する時に特に効果を発揮する
  • 継続的なKAIZEN活動によって効率と質が向上し、イノベーションも実現できる

ジョハリの窓：考案の経緯と正確な表記 外部補足・確認済
考案年・場所：1955年夏、アメリカで開催された「グループ成長のためのラボラトリートレーニング」席上で発表。
考案者の正式名：ジョセフ・ルフト（Joseph Luft、サンフランシスコ州立大学）とハリントン・インガム（Harrington Ingham）。フルネームは Harrington Ingham だが通称 Harry Ingham。
名称の由来：二人のファーストネーム "Jo（seph）" + "Hari（ngton）" を組み合わせて "Johari（ジョハリ）"。ジョハリという人物は実在しない。
英語4領域の正式名：開放＝open self、盲点＝blind self、秘密＝hidden self、未知＝unknown self。
参照: Wikipedia / カオナビ / 日本の人事部
トヨタ生産方式（TPS）とKAIZENの世界普及 外部補足・確認済
体系化者：大野耐一（おおの たいいち）。著書『トヨタ生産方式—脱規模の経営をめざして—』（1978年）は生産管理者のバイブル。1988年英語版刊行で世界に普及。
「カイゼン」と「改善」の違い：漢字「改善」は「悪い部分を直す」こと。カタカナ「KAIZEN」は「現状に満足せずさらに良くする」という前向きな継続改善の意味を含む。英語でも "Kaizen" として世界共通語になっている。
海外での再整理：TPSの研究から欧米で「リーン生産方式（Lean Manufacturing）」として体系化。製造業だけでなく病院・行政・IT（アジャイル開発）などに応用されている。
参照: Wikipedia: トヨタ生産方式 / GAZOO.com: 大野耐一
5Sの英語表記と各Sの意味 外部補足・確認済
5Sはローマ字読みの頭文字（Seiri / Seiton / Seisou / Seiketsu / Shitsuke）から命名。英語圏では下記の対応で "Five S" として通用する。

暗黙知・形式知とSECIモデル（野中郁次郎） 外部補足・確認済
提唱者：野中郁次郎（一橋大学名誉教授）。竹内弘高との共著『知識創造企業（The Knowledge-Creating Company）』（1995年、オックスフォード大学出版）で世界的に知られ、10か国語以上に翻訳。
暗黙知（Tacit Knowledge）：言語化・数値化が困難な個人の経験・感覚・ノウハウ・勘。
形式知（Explicit Knowledge）：言語や図表で明示できる知識（マニュアル・報告書・手順書など）。
SECIモデル：共同化（Socialization）→表出化（Externalization）→連結化（Combination）→内面化（Internalization）の4プロセスを螺旋状に循環して新たな知識が創造されるモデル。業務マニュアルの作成は「表出化」（暗黙知→形式知への変換）に相当。
参照: Wikipedia: SECIモデル / グロービス経営大学院

自己肯定感と自己効力感（Self-efficacy） 外部補足・確認済
授業スライド（S03:p.9）に「自己肯定感」という語が登場する。これは心理学の自己効力感（Self-efficacy）と密接に関連する概念。
提唱者：カナダ人心理学者アルバート・バンデューラ（Albert Bandura）。1977年の論文 "Self-efficacy: Toward a unifying theory of behavioral change"（Psychological Review, 84(2), 191-215）で概念化。
定義：「自分が特定の行動を遂行し、目標を達成できるという信念」（Bandura, 1977）。ある状況において必要な行動をうまく遂行できると自分の可能性を認知していること。
授業との接続：自己分析によって自分の強みに気づくことが自己効力感を高める。高い自己効力感は意欲・行動・成長のスパイラルの起点となる。授業でいう「自己肯定感」はこれに相当する（「自分のやっていることは間違っていない」「存在することに意味がある」という感覚）。
自己効力感 vs 自己肯定感：自己効力感＝「自分にはそれだけの能力がある」という認知（行動遂行に特化）。自己肯定感＝「ありのままの自分を肯定する感覚」（より広い自己受容）。
参照: Wikipedia: 自己効力感 / グロービス: 自己効力感
PDCAサイクルとKAIZENの比較 外部補足
KAIZENの4ステップはPDCAサイクル（Plan-Do-Check-Act）と構造が重なるが違いがある。
比較軸KAIZENPDCA
起点現場発・ボトムアップ計画・指示から始まるトップダウンも多い
規模感小さな改善の積み重ねプロジェクト単位での計画実行
継続性終わりのない継続的活動サイクルを繰り返す設計だが区切りがある
知識前提現場の暗黙知を活用形式知（計画書）が前提になりやすい
ジョハリの窓の現代的応用 外部補足・確認済
1955年の提唱後、企業のコミュニケーション促進・能力開発・キャリアコンサルティングで広く活用されている。就職活動での自己分析ツールとしても有名（授業の第2章と直結）。現在はリーダーシップ開発・チームビルディング・コーチングの場でも使われている。注意点：あくまで一理論であり過信は禁物。
