Chapter 1

可視化の目的

可視化とは？

定義
可視化（Visualization）とは、グラフなどにより、データの全体像を視覚的に表現すること。
S03:p.8
🎙 音声補足 ← S05-A
グラフだけでなく、地図上にプロットしたり色でデータを表現したりすることも可視化の一つ。可視化の手法はグラフに限らないことを念頭に置いてほしいと強調された。
種類例
1種類のデータの視覚化ヒストグラム（第3回）
複数のデータの視覚化散布図（第4回）

可視化の3つのメリット

#メリット今回の扱い
①全体像を把握する第5回
②発見する第5回
③データを伝える第6回
S03:p.9〜10

メリット① 全体像を把握する

平均・分散・相関係数が全く同じデータであっても、可視化した場合は常に同じグラフになるとは限らない。S03:p.11

アンスコムのカルテット（Anscombe's Quartet）

定義
アンスコムのカルテットとは、平均・分散・相関係数が全て同じであるにもかかわらず、散布図として描いたときに全く異なる形を示す4つのデータセット。
x の平均y の平均x の分散y の分散相関係数
9.07.510.03.750.816

各データセットは11組のデータを含む。「Quartet（カルテット）」は音楽用語で「4重奏」の意味。

S03:p.12〜13
図：アンスコムのカルテット — 統計量は同一でも散布図は全く異なる形を示す S03:p.12

→ 統計量だけではデータの全体像は把握できない。可視化が必要。

メリット② 発見する

発見できるものは2種類ある。S03:p.14

(1) 問題のあるデータを発見する

生データから発見しにくい外れ値・異常値が、可視化することで発見しやすくなる。

(2) データから新たな仮説を発見する

データを可視化することで整理され、新たな仮説の発見や今まで気づかなかったことへの気づきが生まれる。探索的データ解析（EDA）の有力な方法の一つ。S03:p.14

歴史的事例：スノウのコレラ地図

Context
コレラ：1884年に病原菌が特定されるまで「瘴気説（しょうきせつ）」（空気感染）が有力とされていた感染症。実際は経口感染が主な感染経路。S03:p.16

ジョン・スノウ（John Snow）：イギリス人医師。1854年（病原菌特定前）のロンドンでのコレラ大流行時に患者の数を地図上に可視化した。

🎙 音声補足 ← S05-A
コレラは1818年頃から世界的な大流行を7回も繰り返している、今でも存在する病気。地図の黒い点は横線1個で1人を表し、18個積み重なっている箇所では18人が亡くなったことを示している。
観察事実導かれた仮説
死者が地図の中心部（ブロード・ストリート付近）に集中空気感染（瘴気説）ではない
ブロード・ストリートのポンプ周辺に被害者が集中ポンプ（経口感染）が原因の可能性
醸造所での死者はゼロ敷地内に独自の井戸があり感染源のポンプを使っていなかった
S03:p.17〜19

→ 地図という可視化手法から新仮説を提案 → 地道な調査で確証を得る。探索的データ解析の典型例。

歴史的事例：ナイチンゲールのローズダイアグラム（鶏頭図）

クリミア戦争（1853〜56年）従軍後、病院での衛生管理の重要性を訴えるために報告書を作成。ローズダイアグラム（極座標図の一種）を用いた。S03:p.21〜22

数値（重要）
衛生管理の徹底による死亡率の改善：1855年2月 約8.2% → 1856年3月 約0.11%

同じデータを棒グラフにすると「冬に死亡率が上昇したのでは？」という誤解を招きやすい。どのグラフを選ぶかによって伝える情報が変わる。S03:p.23〜24

可視化の限界

Key Message
「1人あたりのチーズの消費量」と「ベッドシーツに絡まって死亡した人数」が同じような増加傾向を示す → これは疑似相関であり、全く関係のないデータを可視化しただけ。S03:p.25〜26

→ 可視化だけでは十分な結論は得られない。

第1章 まとめ

データを可視化する3つの目的：S03:p.27

  1. データの全体像を把握する
  2. 「新たな仮説」「外れ値・異常値」を発見する
  3. データの知見や、自己の主張を伝える
Chapter 2

1次元データの可視化

グラフの分類（1次元データ）

定義
データにおける1次元とは、データの種類が1つだけである場合のこと（空間的な次元とは別の概念）。S03:p.31
目的グラフの種類
ばらつきを見るヒストグラム、ストリップチャート、箱ひげ図、バイオリン図
値の比較をする棒グラフ
割合を見るパイチャート（円グラフ）、積み上げ棒グラフ
時間変化を見る折れ線グラフ、コレログラム

ヒストグラム（ばらつき）

  • 縦軸：度数または相対度数（割合）
  • 横軸：階級幅
  • 個々の棒をビン（bin）と呼ぶ
  • 重要：階級幅は一定でない場合も表現可能 → ビンの面積に意味がある
S03:p.32, p.45

ストリップチャート（ばらつき）

定義
ストリップチャート（別名：1次元散布図）：直線上にデータを1個1個そのままプロットしたグラフ。

「ストリップ」は布や板などの細長い切れのこと。S05-A

  • データのばらつきをそのまま表現
  • 表示がコンパクト
  • データの重なりが多いと解釈が困難
S03:p.33

パーセンタイルと四分位数

定義：パーセンタイル
昇順に並び替えたデータに対し、先頭から数えて全体の a% 目に位置するデータを a パーセンタイル と呼ぶ。
⚠ 重要ポイント
「先頭から数えて」とは、値ではなくデータの個数に着目すること。
定義：四分位数（しぶんいすう）
25%ごとのパーセンタイルに付けられる特別な名前。四分位範囲（IQR）= 第三四分位数 − 第一四分位数
名称パーセンタイル例（n=9: 1,2,4,7,8,9,10,12,13）
第一四分位数（Q1）25%4（3番目: (9+3)/4=3）
第二四分位数（Q2）= 中央値50%8
第三四分位数（Q3）75%10（7番目: (3×9+1)/4=7）
四分位範囲（IQR）Q3 − Q16（10−4）
S03:p.34〜38
🎙 音声補足 ← S05-A
高校数学で習った四分位数の定義と異なる場合がある。今回の定義は Excel / Google スプレッドシートの QUARTILE関数 で採用されている定義。高校で教えられる方法は「品位（品位点）」と呼ばれる別の定義によるもの。
図：四分位数と四分位範囲（IQR） S03:p.35

箱ひげ図（ばらつき）

定義
箱ひげ図：四分位数を用いた可視化手法。5数要約（最大値・Q3・中央値・Q1・最小値）でばらつきを表現。
図：箱ひげ図の各要素 S03:p.40

外れ値がある場合の定義

要素定義
最大値（ひげ上端）Q3 + IQR × 1.5 より小さい値の中での最大値
最小値（ひげ下端）Q1 − IQR × 1.5 より大きい値の中での最小値
外れ値IQR の 1.5倍を超えた値（ひげ外に点で表示）
S03:p.42
⚠ 読み方のポイント
箱ひげ図の各区間（最小値〜Q1、Q1〜Q2、Q2〜Q3、Q3〜最大値）にはほぼ同じ個数のデータが含まれる。ただし総データ数が奇数・外れ値がある場合は常に同じ個数にはならない。S03:p.43

棒グラフ（値の比較）

  • 量的データの比較・質的データのカウント結果の比較など幅広く使用
  • 差が明確に可視化できる
  • 原点（0）を必ず表示する必要がある
S03:p.44

ヒストグラム vs 棒グラフ

項目ヒストグラム棒グラフ
棒の幅変更可能一定
意味のある要素面積高さ
用途ばらつきの表現値の比較
⚠ 混同注意
見た目が似ているが、全く異なるグラフ。
S03:p.45

パイチャート・積み上げ棒グラフ（割合）

グラフ特徴向いている場面
積み上げ棒グラフコンパクトに表示報告書で1枚に収めたい場合
パイチャート面積を多く使う1/2や1/4の場合に明解
S03:p.46〜48
🎙 音声補足 ← S05-A
表示する面積がどれくらい余裕があるか、そしてどんなことを主張したいかを考えながらグラフを選択するとよい。

折れ線グラフ（時間変化）

  • データの時系列変化の表現に利用
  • 折れ線の傾きでデータの増減を視覚的に表現できる
  • データが重なると判別が困難
S03:p.49

第2章 まとめ

  • 目的：ばらつき・値の比較・割合・時間変化
  • 選択基準：表示エリアのサイズ、簡潔な表現、データの重なりの有無
  • 箱ひげ図：パーセンタイル・四分位数が基礎（第4章でも再登場）
S03:p.50
Chapter 3

多次元データの可視化

グラフの分類（多次元データ）

目的グラフ・可視化方法
値の関係を見る散布図、散布図行列、バブルチャート
値の比較をする複合グラフ、平行座標プロット（パラレルコーディネート）
質的データを含む棒グラフ、格子状バブルチャート
その他の可視化ヒートマップ、ネットワーク、メッシュ表現、レーダーチャート
S03:p.54

散布図行列（値の関係）

定義
複数の量的データの全組み合わせの散布図を行列状に並べたグラフ。3次元以上のデータを2次元平面上に表示できる。
  • 多次元データの全体像を把握しやすい
  • 量的データのみが対象（質的データは表示不可）
  • 右上と左下に同じデータの組み合わせが現れる（重複）

応用：重複する図を削除し、対角線にヒストグラム、右上三角に相関係数を表示することで表示エリアを有効活用できる。S03:p.60

S03:p.56〜59

バブルチャート（値の関係）

定義
散布図の点を面積情報（丸の大きさ）を持つ円にすることで、3次元データを表示できるグラフ。
グラフ表示できる次元
散布図2次元（x軸・y軸）
バブルチャート3次元（x軸・y軸 + 面積）
S03:p.61〜62

複合グラフ（値の比較）

定義
棒グラフと折れ線グラフなど異なる種類のグラフを1つに組み合わせたもの。軸が2本ある場合を特に2軸グラフと呼ぶ。
  • 1つのグラフを見るだけで傾向の比較が容易
  • 軸が2本に分かれるため、読み手への工夫が必要
S03:p.63〜66

平行座標プロット（値の比較）

定義
複数の平行な縦軸（平行座標）に各次元のデータをプロットし、同じ対象のデータを線でつないだグラフ。5次元・6次元など高次元データを一括比較できる。
比較項目折れ線グラフ平行座標プロット
横軸の並び順時間の順序（意味あり）意味なし（並び替えても影響なし）
傾きの意味増加・減少意味なし（同じ対象のデータを結んでいるだけ）
単位統一が必要異なっていても使える
⚠ 混同注意
見た目が折れ線グラフに非常に似ているため、傾きで増減を読んでしまわないよう注意。
S03:p.67〜71

ヒートマップ（その他）

定義
データの密度・量を色で表現する可視化方法。データ数が多い場合に特に効果的。色情報を用いることで3次元データを2次元空間上に表現可能。

応用例：ウェブサイト閲覧時の視線の位置（x・y軸）と停留時間（色の濃さ）を同時に可視化。S03:p.75

S03:p.74

ネットワーク（その他）

定義
接点（ノード）と辺（エッジ）で構成された図形。データ間の関係性（つながりの有無・前後関係など）を表現する。
種類特徴
有向グラフ矢印付きエッジ → "流れ"を表現
無向グラフ矢印なし（今回の例）
S03:p.76
🎙 音声補足 ← S05-A
ニューラルネットワークなどの機械学習の図でよく見る表現。今後も出てくる用語として覚えておくとよい。

  • QUARTILE関数の定義：今回学んだ四分位数の定義は、Excel/Google スプレッドシートのQUARTILE関数で採用されている定義。高校数学の「品位点」の定義とは異なる場合がある。S05-A
  • VAR.P関数：分散を計算する際に使うExcel/スプレッドシートの関数。アンスコムのカルテットのスライドに注記あり。S03:p.13
  • コレラの歴史：コレラは1818年頃から世界的な大流行を7回以上繰り返している。1884年にコッホによって病原菌が特定された。S05-A
  • クリミア戦争：1853〜56年に起きた戦争。ナイチンゲールが看護師として従軍したことで知られる。S03:p.21
🎯 補足C：試験・レポートで重要な観点
  1. アンスコムのカルテット：統計量が同じでも可視化すると異なる → 「なぜ可視化が必要か」の説明として最頻出
  2. 四分位数の計算：Q1 = (n+3)/4番目、Q3 = (3n+1)/4番目の公式（QUARTILE関数の定義）は出題可能性が高い
  3. 箱ひげ図の各要素と外れ値の定義（IQR×1.5）：図の読み方を含めて問われる可能性が高い
  4. 可視化の3つの目的：全体像把握・発見・伝達の3つを具体例とともに説明できるようにする
  5. グラフ選択の判断基準：データの種類（量的・質的）・次元数・目的の3つの観点から選択できるように
  6. 平行座標プロットの特徴：折れ線グラフとの違い（横軸の並び・傾きに意味なし・単位が違っても使える）は頻出
  7. ヒストグラムと棒グラフの違い：面積に意味あり vs 高さに意味あり
