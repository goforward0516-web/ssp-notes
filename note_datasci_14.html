🔄 前回の復習（第13回：画像データと行列計算）

Q. 画像データをデジタルデータ化した場合の特徴について説明してください。

【解答】
  • 画像は画素（ピクセル）の集合として表現される。画像を格子状に区切った1マスを「画素」と呼び、各画素に色の値が入ることで画像が構成される。← S03:p.3
  • カラー画像の場合、各画素にRGB（赤・緑・青）の3値を持つため、ベクトルとして表現すると 「画素数 × 3」の巨大なベクトル になる。← S03:p.3
  • そのため画像データの分析は非常に困難であり、さまざまな工夫が必要となる。
🔊 音声補足前回の内容に自信がない場合は第13回のノートで復習すること。この「巨大なベクトル問題」が、後で学ぶ次元圧縮や深層学習の必要性につながる重要な伏線。← S05-A

🔍 機械学習とは Ch.1

【定義】（統計的）機械学習

データから判断の根拠となる「ルール」を統計的な方法（＝数式）で作る過程。
ルールは多数のパラメータで構成される。機械学習は人工知能研究の一分野である。

🔊 音声補足「統計的」＝「数式で表す」という意味。正確には「統計的機械学習」と呼ぶ。← S05-A, S05-B
🌐 外部補足「人工知能（AI）」は定義が定まっていない言葉だが、基本的には「人間が行っているようなことを機械にやらせようとする研究分野」を指す。将棋AIはチェス・将棋に特化しており、ケーキを識別する用途には答えられない。適切なデータで訓練して初めてタスクをこなせるようになる。← S03:p.8, S05-A

⚙️ パラメータと学習 Ch.1

【定義】パラメータ

ルールを構成する個々の変数。例：チョコレートケーキを判断するルールには「形状パラメータ（三角柱・四角・円…）」「色パラメータ（白・黒・赤・茶…）」などが含まれる。

【定義】学習

各パラメータの値を、データを基に変化させていく過程。人間の「学習」とは意味が異なる点に注意。

🔊 音声補足チョコレートケーキの例：形状パラメータに「三角柱」、色パラメータに「茶色」が正解として与えられ、それをデータから順に教え込んでいく過程が「学習」。← S05-A, S03:p.11-13

📐 具体例：単回帰分析で機械学習を理解する Ch.1

🔊 音声補足第9回で学んだ単回帰分析も機械学習の一種として捉えることができる。← S05-A, S05-B
機械学習の概念単回帰分析での対応物
判断の根拠となるルール一次関数（直線）
ルールに含まれるパラメータ傾き a、y切片 b

来客数 = 広告額 × 0.5 + 45

← 傾き 0.5、y切片 45 が最小二乗法で求まった例　← S03:p.14-15

🔗 データサイエンスと機械学習の関係 Ch.1

【定義】データサイエンスにおける機械学習の位置づけ

機械学習はデータ解析をする際の選択肢の一つ。分析の目的・データ数（標本数）等に応じて、統計学の方法と機械学習の方法を使い分ける。

統計学の方法 vs 機械学習の方法

比較項目統計学の方法機械学習の方法
必要なデータ数 比較的少なくてよい
例：t検定では自由度 30 程度（データ数約30個）で正規分布に近似できる ← S05-A
S05-Bの「重度」→「自由度」に修正		 多くのデータが必要
最低でも 500件程度、場合によっては 1万〜2万件 以上← S05-A, S05-B
結果の解釈性
（説明可能性）		 比較的高い
なぜその結果になったか理解しやすい		 低い傾向（特にニューラルネットワーク使用時に顕著）
解析結果の理由が不明確になりやすい← S03:p.18
🔊 音声補足どちらが優れているということではなく、目的やデータ数によって使い分けることがデータサイエンスの立場。解釈性が重要な場面では統計学、精度・性能が優先される場面では機械学習、という使い分けが基本。← S05-A, S05-B
🌐 外部補足「解釈性（説明可能性）」は英語で Interpretability / Explainability と表記される。AI倫理の観点から重要性が増しており、XAI（Explainable AI：説明可能なAI）という研究分野が生まれている。

🗂️ 機械学習の3分類 Ch.1

🔊 音声補足教師あり学習と教師なし学習の違いは「入力データにラベル（正解データ）が含まれているかどうか」。データサイエンスでは主に教師あり学習と教師なし学習を使う。強化学習はゲームAIや自動運転で使われるが、データサイエンスの文脈では比較的使用頻度が低い。← S05-A, S05-B, S03:p.23
🔊 音声補足（強化学習の具体例）自動運転の例：「信号が青 → +10点、歩行者がいる → −100点」のようにスコアを設定し、シミュレーションを繰り返しながら最適な行動を学習する。← S05-A, S05-B

📗 第2章　教師あり学習・教師なし学習 Ch.2

第2章の学習目標

教師あり学習・教師なし学習の枠組みと活用例について説明ができる。← S03:p.28

🎯 教師あり学習 Ch.2

【定義】教師あり学習

各入力データに対応する出力データ（＝ラベル・正解データ）を与えて学習する方法。入力データと出力データの関係性を関数として学習する。

🔊 音声補足チョコレートケーキの識別例：入力データ＝ケーキの画像、出力データ＝「チョコケーキ」というラベル。この関係性を数式（関数）で表現するのが教師あり学習の目的。← S05-A, S05-B, S03:p.29

🔓 教師なし学習 Ch.2

【定義】教師なし学習

入力データのみを与えて、「意味のありそうな」データを出力する方法。出力データ（正解ラベル）は与えない。

🔊 音声補足チョコケーキとチーズケーキの例：「チョコ」「チーズ」というラベルを付けずに画像だけを与えると、機械が似た者同士をグループ化してくれる。どのグループが何を意味するかは分析者が解釈しなければならない。← S05-A, S05-B, S03:p.30

📊 4つの枠組み：連続値 vs 離散値 Ch.2

【定義】連続値

連続した値。例：売上1,000円・1,200円・1,300円…、テストの点数0〜100点

【定義】離散値

とびとびの値。例：合格(1)／不合格(0)、野菜(1)／果物(2)／加工食品(3)

⚠️ 注意：誤変換についてS05-A では「遺産値・理産値」、S05-B では「利算値・地産値・理産値」と誤変換されているが、すべて「離散値」が正しい。スライド S03:p.31 で確認済み。誤変換修正済み
⚠️ 補足ロジスティック回帰・プロビット回帰など、「回帰」と名が付いても離散値を出力する特別な回帰手法もある。「回帰＝連続値のみ」ではないことに注意。← S03:p.32, S05-A

📈 回帰 Ch.2

【定義】回帰

複数の説明変数間の関係性を学習する方法（教師あり学習・連続値出力）。

手法数式特徴
単回帰分析（第9回）y = ax + b説明変数1個
重回帰分析y = a₁x₁ + a₂x₂ + … + aₙxₙ + b説明変数n個。人間が理解できなくても予測性能が良い場合がある
🔊 音声補足重回帰分析では、売上予測の例なら「その日の気温・天気・来客数・曜日」などを複数の説明変数として同時に扱える。説明変数が2〜3個なら人間も理解できるが、1万個では理由の説明は困難。それでも予測性能が良い場合があるのが機械学習の特徴。← S05-A, S05-B
⚠️ 誤変換注意S05-A では「中回帰分析・十回帰分析・3回帰分析」と誤変換されているが、すべて「重回帰分析」が正しい。S03:p.32 で確認済み。誤変換修正済み

回帰分析の活用例 ← S03:p.33

  • 小売業：商品の需要予測、広告効果の検証
  • 金融業：顧客の信用情報の数値化
  • 医療：健康診断の各項目から健康度を数値化し、健康度が低い人に保健指導を実施
🔊 音声補足回帰分析は方法がシンプルなため幅広い分野で活用可能で、かつ比較的判断理由を理解しやすい（解釈性が高い）という特徴がある。← S05-A

🏷️ クラス分類 Ch.2

【定義】クラス分類

出力データとして与えられたクラスに対して、説明変数間の関係を学習する方法（教師あり学習・離散値出力）。Yes/No、犬/猫/トラ など複数クラスも可。

⚠️ 誤変換注意S05-A では「プラス分類・クラスボルン」と誤変換されているが、すべて「クラス分類」が正しい。S03:p.34 で確認済み。誤変換修正済み
🔊 音声補足第10回で学んだクラス分類がこれにあたる。決定木は解釈性が高いクラス分類の手法の一つ。← S05-A, S05-B

活用例 ①：非構造化データ ← S03:p.35

  • 画像のパターン認識：顔認識・物体認識・行動認識・医療画像診断
  • 音声認識・文字認識・文章のトピック認識
  • 例：「迷惑メール」報告からパターンを学習し自動分類

活用例 ②：構造化データ ← S03:p.36

  • 購買予測：顧客の属性情報から購入可否を予測
  • 解約予測：顧客の属性からサービス解約の可能性を予測
  • 新商品の分類：過去商品の分類データの特徴を基に新製品のカテゴリを決定

🗜️ 次元圧縮（次元削減） Ch.2

【定義】次元圧縮（次元削減）

高次元の入力データを、本質的に同値の低次元のデータに変換する機械学習の方法（教師なし学習・連続値出力）。

⚠️ 誤変換注意S05-A では「次元波縮・次元発色」と誤変換されているが、「次元圧縮」が正しい。S03:p.37 で確認済み。誤変換修正済み

主な分析手法 ← S03:p.37

  • 主成分分析（PCA：Principal Component Analysis）・独立成分分析：線形データに対して有効
  • t-SNE（t-distributed Stochastic Neighbor Embedding）：非線形データに対して有効

PCAの例：ケーキデータ

🔊 音声補足ケーキの「金額・売上・糖質・タンパク質…」という多数の特徴量を PCA にかけると、「第1主成分」「第2主成分」という2つの数値にまとめられる。ただしなぜそのようにまとめられたかは機械が自動で決めており、「第1主成分はケーキの人気度を表している」「第2主成分は甘さを表している」といった解釈は分析者自身が判断しなければならない。← S05-A, S05-B, S03:p.38
🌐 外部補足PCA は元の多次元データを「分散が最大になる方向」に射影することで次元を削減する手法。情報の損失を最小化しながらデータを圧縮でき、可視化や前処理として広く使われる。

🔵 クラスタリング Ch.2

【定義】クラスタリング

入力データ間の類似度から、データをグループ化（グルーピング）する方法（教師なし学習・離散値出力）。類似度の指標としてデータ間の距離（ユークリッド距離など）を用いる。

🔊 音声補足第10回で学んだ。AグループとBグループに分けることはできるが、「AとBがそれぞれ何を意味するか」は分析者が解釈しなければならない。次元圧縮と同じ問題。← S05-A, S05-B

クラスタリングの応用：異常検知（異常検出）

【定義】異常検知（異常検出）

過去のデータから形成されたクラスターの範囲から外れているデータを「異常」として検出する方法。クラスターに名前が付いていなくても「この範囲から離れている」という情報だけで異常を検知できる点が強み。← S03:p.39

🔊 音声補足活用例：自動車の異常な動きから飲酒運転を発見。各クラスターに名前が不要なのが異常検知の強み。← S05-A, S05-B

第2章まとめ ← S03:p.40

手法分類活用の方向性
回帰教師あり・連続値過去データを用いた予測。解釈性が高く活用範囲が広い

🧠 ニューロンとニューラルネットワーク Ch.3

【定義】ニューロン（神経細胞）

脳の最小単位。人間の脳には約1,000億個のニューロンがあり、これらがネットワークを構成して情報処理を行っている。← S03:p.44

【定義】ニューラルネットワーク（Neural Network）

脳内の神経細胞ネットワークから着想を得て、これをコンピュータ上で人工的に再現したもの。細胞から細胞に情報が次々に伝わることを数学（数式）で表現した計算モデル。← S03:p.45

⚠️ 誤変換注意S05-A/B では「ユーラル・ムーラル・デューラル・ニブルアリメント」など多数の誤変換がある。すべて「ニューラル」が正しい。誤変換修正済み
🔊 音声補足（画像認識のイメージ）NNに「白色細胞・茶色細胞・チョコ細胞・果物細胞」などが存在するとする。ショートケーキの画像を入力すると「白色細胞」と「果物細胞」が発火 → 「ショートケーキ細胞」が発火 → ショートケーキと判断。チョコケーキなら「茶色細胞」と「チョコ細胞」が発火 → 「チョコケーキ細胞」が発火。このように細胞の発火パターンで認識を行う。← S05-A, S05-B, S03:p.49-51

ニューラルネットワークの学習イメージ ← S03:p.52

① 全ての細胞間が接続された状態（全結合）からスタート
② 出力結果と正解との誤差を基に、不適切な結合を弱める（誤差逆伝播法 / バックプロパゲーション）
③ これを繰り返すことで認識に必要な結合だけが残り完成

🌐 外部補足誤差逆伝播法（Back Propagation）：出力層の誤差を出力層→入力層方向に逆伝播させ、各重みパラメータの更新量を計算するアルゴリズム。1986年の多層パーセプトロンとともに広く普及した。

⚡ 深層学習の特徴 Ch.3

【定義】深層学習（ディープラーニング）

多層のニューラルネットワーク構造を持つ機械学習手法の総称（一般に4層以上）。ResNet・VGGNet など様々なアルゴリズムが存在する。← S03:p.54

従来の機械学習の問題点

🔊 音声補足従来の機械学習では「どのパラメータ（特徴）を使えば問題を解決できるか」を人間が指定しなければならなかった。例：ショートケーキの認識なら「三角形の形状・白い色・イチゴが乗っている」などを人間が一個一個列挙する必要があった。しかし「ホールケーキはショートケーキじゃないの?」「他の果物が乗っていたら?」など、全ての特徴を人間が網羅的に指定するのは困難かつ非効率。← S05-A, S05-B, S03:p.53

深層学習のメリット（2つの特徴）

特徴内容
① 表現学習 どのパラメータ（特徴）を使用すべきかを自分で学習する。人間が特徴を一個一個指定しなくてよい。← S03:p.54
② エンドツーエンド学習（end-to-end） 「特徴抽出 → ベクトル化 → データ解析」という複数ステップを一つの学習プロセスで同時に行う。従来はステップごとに個別処理が必要だった。← S03:p.54

深層学習のデメリット（3つの課題）

課題内容
① 大量のデータが必要 十分な性能を発揮するために大量のデータが不可欠。1種類の画像認識でも数十枚〜数百万枚の正解画像が必要なことも。← S03:p.55
② 大量の計算資源が必要 高性能な GPU（Graphics Processing Unit） と膨大な電力が必要。← S03:p.55
③ 解釈性の問題 どの特徴を使って判断したかが分からない（ブラックボックス）。誤認識の原因（学習データの偏り？学習不足？アルゴリズム自体の問題？）も特定しにくい。出力結果を判断に利用する際に倫理的問題が生じる可能性あり（詳細は第15回）。← S03:p.56
🔊 音声補足1,000種類の画像を分類したいのに500枚しかデータがなければ、全種類のデータが揃っていないため学習は成功しない。「大量データが必要」という意味の体感例。← S05-A

📚 補足B：知っておくべき背景知識
