CH.1データの関係性

二変量データとは

二変量データ（2変量データ）
1つの観測対象（個人・店舗など）に対して 2種類のデータが存在する形式。
表記：n 組の (x₁, y₁), (x₂, y₂) … (xₙ, yₙ)
← S03:p.10
🎙 AUDIO SUPPLEMENT
例：Aさんの（体重, 身長）、Bさんの（体重, 身長）のように、一人に対して2種類の数値がある場合が二変量データ。3種類あれば三変量、4種類あれば四変量データとなる。3変量以上をまとめて多変量データと呼ぶ。← S05

データ間の3種の関係性

n組の二変量データ (x, y) の間には、以下の3つの関係が特に有名。← S03:p.11

相関関係 vs 回帰関係（混同に注意！）

相関関係回帰関係
変量の役割2変量とも目的変数説明変数 x → 目的変数 y
影響の方向双方向（相互に影響し合う）一方向（x が y に影響）
使いたい場面2変量の関係を表現したいx から y を予測したい
例身長と体重（どちらも観測値）気温（x）→ 売上（y）
学習回第4回（相関係数）第9回（回帰分析）
← S03:p.14
🎙 AUDIO SUPPLEMENT
相関の例：「身長が伸びれば体重が増えるし、逆に体重が増えれば身長が伸びる」という双方向の想定をしているのが相関。回帰の例：「売上から気温を予測することは回帰関係ではできない」——方向性が一方通行である点が決定的な違い。← S05

データの関係性の発見

データの関係性の発見
変量間の相関・連関を見つけること。← S03:p.15
🎙 AUDIO SUPPLEMENT — 相関と連関の使い分け
統計の世界では、量的データ（身長・体重など）の相関関係を求めることを相関、質的データ（血液型・職業など）の相関関係のことを連関と呼んで区別する。参考書によっては両方まとめて「相関」と書いたり、連関を「関連」と表記することもあるが、この授業では相関（量的データ）・連関（質的データ）で統一する。← S05

関係性の発見が必要な場面

  • アンケート調査：年齢・性別・職業・商品の好み等、多数の調査項目間の関係を分析したい時
  • 小売店：取り扱い数千種類の商品の間で「この商品が売れたらあの商品も売れる」を把握したい時
  • 医療：薬品間の関係を分析したい時
← S03:p.15

関係性発見のための2つの方法

方法対象特徴
相関分析（相関行列） 2変量間の関係 多変量データでも2変量ずつの組み合わせで相関係数を計算できる（相関行列）。ただし2変量間の分析に限られる。
アソシエーション分析
（バスケット分析）		 多変量間の関係 組み合わせ爆発を回避し、計算機で効率的に多変量間の関係を分析できる。
← S03:p.16
🎙 AUDIO SUPPLEMENT
相関行列の例：数学・国語・英語・化学の4科目（4変量）について、任意の2科目の組み合わせ（数学×国語、数学×英語 …）ごとに相関係数を表形式にまとめたもの。ただし「おにぎり＋お茶＋お手拭き」のような3つ以上の関係を調べたい場合には使えない。そこでアソシエーション分析が必要になる。← S05
⚠️ 組み合わせ爆発（combinatorial explosion）
取り扱い商品数が増えると、全組み合わせ数が爆発的に増加し人手での計算が不可能になること。例：コンビニの約3,000種類の商品から3個の組み合わせを列挙すると約45億通りになる。アソシエーション分析はこの問題を計算機で効率的に解決する。← S05

因果関係と疑似相関（重要な留意点）

因果関係
何らかの操作が原因となって、結果に影響が出ること。
「コップを落とした（原因）→ コップが割れた（結果）」← これは因果関係あり
「犬が吠えた（行動）→ 試験に合格した（結果）」← これは因果関係なし
← S03:p.17
⚠️ 最重要注意点
相関関係・回帰関係から、因果関係を導くことはできない。
データ分析の究極の目標は因果関係の存在をデータから示すことだが、これは非常に難しい（統計的因果推論という専門分野がある）。← S03:p.18
🎙 AUDIO SUPPLEMENT — 疑似相関の例
「全国の市区町村の病院の数と死亡率の間に正の相関があった。→ 病院の数を減らせば死亡率が下がる」という主張は誤り。
実際は人口減少という共通の原因が存在しており、「人口が減った → 経営が成り立たなくなり病院が減った」かつ「人口が減った → 病院が少なくなり他の市区町村で亡くなる人が増え、この地域の死亡率が下がった」という2つの結果の間に疑似相関が生じている。← S05
また相関は双方向の関係なので、「病院の数 → 死亡率」の方向だけを主張するのも不適切（逆方向「死亡率 → 病院の数」も説明しなければならない）。
🌐 EXTERNAL SUPPLEMENT
疑似相関（spurious correlation）とは、2変量間に直接の因果関係がないにも関わらず、第3の変数（交絡因子）の影響で相関があるように見える現象。有名な例として「アイスクリームの売上と溺死事故件数は正の相関がある」（実際の原因は気温の高さ）がある。
第1章 まとめ
  • データ間の関係性：相関関係（双方向）、回帰関係（一方向）、因果関係（原因→結果）
  • 2変量間の関係を表現したい → 相関　／　予測したい → 回帰
  • 相関・回帰から因果関係は導けない（最重要）
  • 関係性の発見手法：相関分析（2変量）、アソシエーション分析（多変量）
← S03:p.19

CH.2関係性の発見

POSデータ

POS（Point of Sales）システム
小売業における販売実績データの蓄積システム。コンビニエンスストアやスーパーマーケットで「いつ・どんなものが・誰に販売されたか」を記録・蓄積する仕組み。← S03:p.23
種類内容含まれるデータ
POSデータ 販売実績データ（レシート相当） 購入店舗・購入日時・購入商品名・金額・数量
ID付きPOSデータ POSデータ + 顧客データ 上記 + ポイントカード・電子マネーに紐づく個人情報（年代・性別等）
← S03:p.23
🎙 AUDIO SUPPLEMENT — ID付きPOSデータの活用例
コンビニでポイントカードや電子マネーで支払うと、その決済データに紐づく個人情報（年代・性別等）と購入商品の記録が結びつく。これがID付きPOSデータ。たとえばお店側は「この時間帯は30代ぐらいの人が来てお弁当を買ってくれる」という傾向を把握できるようになる。通常のPOSデータでは「何が売れたか」しかわからないが、ID付きにすることで「誰が」「どんな組み合わせで」買ったかも分かる点が大きな違い。← S05
🎙 AUDIO SUPPLEMENT
POSデータを活用することで「商品Aと商品Bはいつも一緒に買われるから近くに置こう」という商品配置や新商品開発に生かせる。コンビニは約3,000種類以上の商品を扱うため、商品の組み合わせ（3個の組み合わせで約45億通り）を人間の頭で考えるのは不可能 → アソシエーション分析が必要。← S05

アソシエーションルール

アソシエーションルール（相関ルール）
「商品Aを買っている人は、商品Bも同時に買っている」という組み合わせを表すルール。
表記：{条件部} ⇒ {結論部}
例：{商品A} ⇒ {商品B}　／　{商品A, 商品B} ⇒ {商品C}
← S03:p.25
🎙 AUDIO SUPPLEMENT — 方向性に注意
アソシエーションルールは一方向の関係（← 相関関係とは異なる！）。{商品A} ⇒ {商品B} と {商品B} ⇒ {商品A} は別のルールとして扱う。漫画の例：「1巻を買った人が2巻を買う」は意味があるが、「2巻を買った人が1巻を買う」はあまり意味がない。← S05
※「相関ルール」という別名があるが、これは前述の「相関関係」ではなく、単に複数商品の間に「関係がある」ことを指す用語。本授業ではアソシエーションルールで統一する。
🎙 AUDIO SUPPLEMENT — 取り扱い商品が3つの場合の組み合わせ例
商品A・B・Cの3種類でも、考えられるアソシエーションルールは12通りある（2項目ルール6通り + 多対1・1対多のルール6通り）。実際のコンビニでは数千種類あるため組み合わせ爆発が起きる。← S05

① 支持度（support）

支持度（support）
「頻繁に出現する商品の組み合わせ」の指標。
客全体に対し、X と Y を両方とも購入した人の割合。
計算式
支持度({A}⇒{B}) = (A と B を両方購入した客数) ÷ (総客数)
= 2 ÷ 5 = 0.4　（客1・客3がA・B両方購入）
🎙 AUDIO SUPPLEMENT
支持度が大きい → よくある組み合わせ。支持度が小さい → そもそも不人気商品なので検討対象外。← S05
← S03:p.28

② 信頼度（confidence）

信頼度（confidence / 確信度）
「ルールの強さ」の指標。
X を購入した人に対し、X と Y を両方とも購入した人の割合。
計算式
信頼度({A}⇒{B}) = (A と B を両方購入した客数) ÷ (A を購入した客数)
= 2 ÷ 3 = 2/3 ≒ 0.67　（Aを買った3人のうち2人がBも購入）
🎙 AUDIO SUPPLEMENT — 支持度と信頼度の比較例
{A}⇒{E}：支持度 = 3/5、信頼度 = 1（Aを買った人全員がEも購入）→ 共に高い → 検討対象
{C}⇒{A}：支持度 = 1/5、信頼度 = 1 → 信頼度は高いがそもそも商品Cが不人気なので検討対象外← S05
← S03:p.29,30,31

③ リフト値（lift）

リフト値（lift）
「売れ筋商品かどうか」の指標。
X との同時購入で、Y の購入者が何倍になったか？
計算式
リフト値({A}⇒{B}) = 信頼度({A}⇒{B}) ÷ (B の購入者割合)
= (2/3) ÷ (3/5) = 10/9 ≒ 1.11
リフト値の値解釈例
1 より大きい商品B は 商品A との組み合わせで売れる（正の相乗効果あり）{A}⇒{B} = 10/9 ≈ 1.11
= 1XとYは独立（一緒に買われることがランダムな場合と変わらない）—
1 より小さい商品E は 商品D に関係なく売れる（組み合わせ効果なし）{D}⇒{E} = 5/8 = 0.625
🎙 AUDIO SUPPLEMENT
リフト値1未満の商品はビニール袋のようなもの——どの商品を買っても必ず一緒に買う商品。このような商品まで分析対象にすると「どの商品とどの商品の関係が重要か」が分からなくなってしまう。だからリフト値でこれを除外する。← S05
← S03:p.32,33,34,35

🎙 AUDIO SUPPLEMENT — 結果の解釈例
緑ハイライトのルールを見ると「商品A・Bのいずれかまたは両方を買うと商品E（焼き菓子）も買う」というパターンが浮かび上がる。分析者はここから「商品A・Bはお母さん同士の集まりへの手土産用で、商品Eは自宅用のついで買いではないか」という仮説を立て、「焼き菓子をケーキの近くに移動しよう」という施策に活かす。← S05
⚠️ アソシエーション分析の性質
第8〜10回で学んだ分析手法は「分析結果がそのまま結論」だったが、アソシエーション分析は仮説を作るための探索的な分析。結果から関係性を自分で考え → 仮説を立て → 検証するという流れが必要。← S05

しきい値（検討対象の基準）は自分で決定する

「支持度が高い」「信頼度が高い」の基準（しきい値）はデータや目的に応じて分析者自身が設定する。← S03:p.39

CH.3関係性の活用

レコメンド

レコメンド（recommendation / 推薦）
ある商品を購入（選択）した顧客に対し、別の商品を提案すること。
← S03:p.45
🎙 AUDIO SUPPLEMENT
例：ネットショッピングの「よく一緒に購入されている商品」、動画配信サービスの「次にこの動画はいかがですか？」。動画配信では実際に検索して動画を見る人は全体の約2割で、残り8割は提案された動画を見ているとされる。そのためインターネットサービスではレコメンドが非常に重要な技術。← S05

協調フィルタリング（Collaborative Filtering）

協調フィルタリング
過去の購入履歴・評価履歴などを使用し、類似の購入行動が再び起こると仮定してレコメンドを作成する手法。
← S03:p.46

ユーザベース協調フィルタリング

「対象顧客と似た購入行動をとる他のユーザ（類似ユーザ）」の購入商品をレコメンドする。← S03:p.47,48

手順（アソシエーション分析を使った方法）

  1. 類似ユーザの抽出：対象顧客の購入済み商品を条件部として、他の各顧客に対する信頼度を計算。信頼度が最も高い顧客 = 類似ユーザ
  2. レコメンドの作成：類似ユーザが購入している商品（対象顧客が未購入）をレコメンド候補とする
🎙 AUDIO SUPPLEMENT — 具体例
客1（商品A・B購入済み）へのレコメンド：
{客2}⇒{客1} の信頼度 = 0（客2はA・B両方買っていない）
{客3}⇒{客1} の信頼度 = 1（客3はA・B両方買っている）→ 客3 = 類似ユーザ
客3は商品Eを買っているので → 客1に商品Eをレコメンド← S05

※使うデータは購入履歴だけでなく、「いいね」やブックマークなどの嗜好データも利用可能。

アイテムベース協調フィルタリング

「対象顧客が購入済みの商品と似た購入傾向を持つ商品（類似商品）」をレコメンドする。「似た商品 = 似た購入行動をされている商品」と定義（商品の属性情報は使わない）。← S03:p.53,54

手順（アソシエーション分析を使った方法）

  1. 類似商品のスコア算出：対象顧客（客1）を除いた残り4人（客2〜5）のデータを使用し、客1の「購入済み商品 ⇒ 未購入商品」の各アソシエーションルールの信頼度を計算し、未購入商品ごとに信頼度の合計（スコア）を算出 ← S03:p.55
  2. レコメンドの作成：最もスコアが高い未購入商品をレコメンド
🎙 AUDIO SUPPLEMENT — 具体例（客2〜5の4人ベースで計算）
客1（商品A・B購入済み、商品C・D・E未購入）へのレコメンド：
客1を除く客2〜5の4人のデータを使い、「購入済み ⇒ 未購入」の信頼度を計算する。

商品E のスコア：
{A}⇒{E} の信頼度 = 1.0（Aを買った客2〜5のうち全員がEも購入）
{B}⇒{E} の信頼度 = 1.0
{A,B}⇒{E} の信頼度 = 1.0
→ スコア合計 = 3.0

商品C のスコア：{A}⇒{C}、{B}⇒{C}、{A,B}⇒{C} の信頼度合計 = 0.5
商品D のスコア：同様に信頼度合計 = 0.5

考察：リフト値上位のルール分析

🎙 AUDIO SUPPLEMENT — ロード・オブ・ザ・リング
リフト値上位5件の条件部・結論部はすべて Movie ID 7153・5952・4993 の3本 → どれも「ロード・オブ・ザ・リング」シリーズ。結論：「シリーズ全作の評価が高い映画は1本視聴させれば続編視聴の確率が高くなる → 売上アップ効果が期待できる」という仮説が立てられる。← S05
🎙 AUDIO SUPPLEMENT — アカデミー賞受賞作ルール
Movie ID 318（ショーシャンクの空に・支持度が最も高い）・296（パルプ・フィクション）・50（ユージュアル・サスペクツ）が同一ルールに含まれる。3本はジャンルが異なるが全てアカデミー賞で複数部門ノミネートされた作品。→「ジャンルは違っても話題作を中心にしたルール」という解釈。→「アカデミー賞複数部門ノミネート作品特集ページを作れば連動視聴が見込める」という仮説が立てられる。← S05
⚠️ ドメイン知識の重要性
アソシエーション分析はデータ分析の知識だけでなく、ドメイン知識（今回は映画に関する知識）と組み合わせることで初めて意味のある仮説が立てられる。← S05

📚 補足B: 知っておくべき背景知識

  • 統計的因果推論：相関関係からは因果関係を導けないことは広く知られており、因果関係の証明を統計的に行う「統計的因果推論」という専門分野がある（本授業の範囲外）。
  • コサイン類似度・Jaccard係数：協調フィルタリングで「類似度」を計算する方法として、アソシエーション分析の信頼度以外にも有名な指標がある。
  • 横持ちデータ vs 縦持ちデータ：横持ち = 行にユーザ・列に商品（アソシエーション分析に適した形式）、縦持ち = 一取引一行の形式（通常のPOSデータの保存形式）。

🎯 補足C: 試験・レポートで重要な観点

  • 「相関関係から因果関係は導けない」は頻出。疑似相関の例（病院数と死亡率）を説明できるようにする。
  • 支持度・信頼度・リフト値の計算方法と解釈（どんな時に検討対象か、リフト値1未満の意味）を覚える。
  • ユーザベース vs アイテムベース協調フィルタリングの違いと、どちらも「信頼度」を指標に使う点。
  • 協調フィルタリングの利点と問題点（新商品対応不可、大規模データが必要）。
  • 相関 vs 回帰の違い（双方向 vs 一方向）。

🔗 補足D: 関連概念・次回への接続
