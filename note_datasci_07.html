データを扱う心得

① データ倫理に留意してデータを扱うことができる
② 基本的なセキュリティ・プライバシー対策ができる

前回の復習

Q. 右のような可視化方法（3D円グラフ）について問題点を説明してください。

Answer
3Dグラフにおけるチャートジャンクの例。死角の発生によりデータの比較が困難になる。C社は3Dでは12%・2位のように見えるが、通常の2D円グラフにすると実際には4位である。S03:p.2-3
3Dグラフにすることで見える範囲が異なったり、視覚の発生によってデータの比較が困難になり、判断が間違ってしまうことがある。 ← S05
CHAPTER 01

データ倫理

ELSI（エルシー）

定義
ELSI：Ethical, Legal and Social Issues（倫理的問題・法的問題・社会的問題）の頭文字。技術の進歩により発生する問題の総称。 S03:p.8
  • 当初は生命科学分野で注目
  • 近年はデータサイエンス・AI分野でも特に注目されている

データ倫理

定義
データ倫理：明確な定義はないが、一般的に「データの収集・共有および利活用によって、人びとおよび社会に対して負の影響を与える可能性のあるデータ実務を評価するもの」とされる。 S03:p.9
データ倫理が守られない場合、どれほど素晴らしい分析をしてもデータ分析が全て無駄になる可能性がある。
例えば、使ってはいけないデータを使っていた場合、データ分析の全フェーズが無駄になる可能性がある。この授業では15回のうち1回分をまるごとデータ倫理の説明に割いている。 ← S05

データ収集時の留意点①：バイアス

定義
バイアス：偶然ではないデータの誤差や偏り S03:p.12

① 選択バイアス（計画段階で発生）

例
  • 固定電話による世論調査 → 固定電話を持つ世代の意見しか反映されない
  • 高齢者の意識調査をインターネット調査のみで実施 → 高齢者が対象から漏れる
男女両方いる集団にも関わらず、データの収集方法が適切でなく女性にしかデータが取れなかった場合、そのデータを分析すると女性の意見しか反映できないものになってしまう。 ← S05

② 情報バイアス（実施段階で発生）

例①：体重の対面調査

「体重を教えてください」を対面調査で実施 → 答えにくい質問のため、実際より軽い値を答えてしまい、データ全体が実際より軽くなってしまう

例②：AIタグ付けへの先入観

「今回は猫の画像の方が多い」と事前情報を与えると、判断が難しい画像を猫と分類しやすくなり、実際の比率と異なるデータが生成される

③ 測定バイアス（情報バイアスの一種）

計測機器や担当者の癖によって一定方向に偏る誤差。ランダムに入り込む誤差はデータ数が十分あれば問題ないが、測定バイアスは問題になる。 S03:p.15

温度計を上から見るか目線の位置で見るかによって読み取り値が変わる。同じ計測機器を使っても担当者の癖や機器自体の癖によってデータにバイアスが出る可能性がある。 ← S05

データ収集時の留意点②：オプトイン・オプトアウト

オプトイン（Opt-in）オプトアウト（Opt-out）
意味 自ら参加を選択してデータ収集に同意 掲示等での同意後、後から拒否できる
方法 文書・口頭で事前説明し同意を得てから収集 掲示等により同意取得。本人がデータ使用を後から拒否できる
使用例 心理学の実験・薬の試験など 防犯カメラの映像データを研究に使用する場合など
S03:p.16
データを集める場合はオプトインまたはオプトアウトどちらかの方法を必ず取り、後から拒否できる方法で必ずログを取る必要がある。手続きを取っていないと、分析が最後まで完了したのに「このデータを使ってほしくなかった」と言われて分析結果が全て無駄になってしまう。 ← S05

データ分析時の留意点：研究不正

研究不正に該当する主な行為は3つ。 S03:p.17-19

① 捏造（ねつぞう）

定義
存在しないデータを存在するかのように意図的に作り出す行為
例

アンケートを100人に取りたかったが99件しか集まらなかったため、1件分を自分で適当に埋めた

② 改ざん

定義
データや分析過程等の意図的な改変
例

予測精度が15%しかなかったが、締め切りが近いからと50%と報告した

悪いものを良く見せかけたいという心理は誰にでもあるが、悪かったと報告するのも1つの研究成果といえる。 ← S05

③ 盗用（とうよう）

定義
他人の作成したデータや分析結果を自分の成果として報告する行為
例

他の人が作成した分析報告書を自分の名前で発表した / 他社が収集したデータを自社で集めたかのように分析・報告した

💡 オープンデータについて：利用規約に従って適切に利用し、自分で分析した結果を発表することは盗用にあたらない。 S05
研究不正というと「自分は研究者じゃないから関係ない」と思うかもしれないが、大学でレポートを出すときや会社で分析プロジェクトを行うときも常に関わってくる問題。研究不正に当たる行為をすると、大学では単位取り消し、会社では解雇になる可能性もある。 ← S05
CHAPTER 02

セキュリティ

情報資産

定義
情報資産：データおよびそれを管理・活用するための機器・システム・ノウハウ全般 S03:p.24
  • 各種データ（個人データを含む）
  • データを管理するためのハード・ソフト・ネットワーク
  • データを収集・分析するためのシステム・ノウハウ
データは「21世紀の石油」といわれ、企業にとっては資産。誤った扱いをすると資産価値が下がったり、なくなってしまう可能性がある。情報資産を正しく保護しないと企業価値の低下や社会的・法的責任が発生してしまう。 ← S05

情報資産保護への対策

対策内容
セキュリティ対策データの漏洩・破損防止、システムが使用できない状態にしないための対策
プライバシー対策変化するプライバシー意識への対応、個人情報関連法律への対応
対策が不足すると…社会的・法的責任の発生 → 企業・サービスへの信頼の喪失 → 売上・営業成績の低下

情報セキュリティのCIA

定義
ISO/IEC 27001：ISO（国際標準化機構）とIEC（国際電気標準会議）が共同で制定した情報セキュリティマネジメントシステムの国際規格。情報の機密性・完全性・可用性の維持を定める。 S03:p.26
ISOはクレジットカードのサイズなど製品の基準を決めることで、世界中での取引を円滑にする国際標準化機構。IECは国際電気標準会議。
C
Confidentiality
機密性
アクセス許可のある人・システムだけがアクセスできる状態
I
Integrity
完全性
情報が意図通りの状態で維持されている状態
A
Availability
可用性
いつでも利用でき、問題が出てもすぐに復旧できる状態

機密性（Confidentiality）

機密性の確保が不足すると…不正アクセス（データの改ざん・改編）の発生 / データの内部・外部流出

機密性を確保するための対策

対策内容
閲覧権限の設定スプレッドシート等でアクセスできるユーザーと権限を設定
端末・ネットワークの制限特定のパソコンや部屋からしかアクセスできない設定
情報の暗号化暗号化により、万一データが盗まれても内容を判読不可にする
S03:p.28-30

完全性（Integrity）

完全性の確保が不足すると…情報の改ざん・破壊によるサービスの停止 / データの質低下による不適切な分析結果 / 誤入力による作業の手戻り

完全性を確保するための対策

対策内容
編集ログの保存いつ誰がデータを変更したかを記録に残す
規定外入力のシステム制限郵便番号欄に数字以外は入力できないようにする等
デジタル署名によるデータ保証送信元がデジタル署名付きメールを作成し、受信先が改変がないことを確認
S03:p.32-34
紙のアンケートをパソコンに入力する際に誤入力があると、分析時に発覚して入力作業からやり直す「作業の手戻り」が発生し、通常の分析プロセスにさらに時間がかかる。 ← S05

可用性（Availability）

機密性・完全性の維持が前提。その状態でいつでもデータが利用でき、問題が出てもすぐに復旧できる状態。 S03:p.35

可用性の確保が不足すると…必要なタイミングでデータが利用できず営業機会を損失 / システム故障による欠損データ発生・前処理手順の増加

可用性を確保するための対策

  • システムやデータのバックアップ：定期的に取る・分散させる
  • 身近な例：ファイルの定期的な保存・データバックアップ・落雷対策機器
S03:p.36
会社のデータを一箇所のデータセンターに集めていた場合、地震などでそのデータセンターが被害を受けると全く営業できなくなる可能性がある。そのためデータセンターを複数箇所・日本以外の地域にも設置するという対策もある。情報セキュリティのCIAは企業だけでなく個人レベルでも関係する話。 ← S05
CHAPTER 03

プライバシー（1）

プライバシーと個人情報の違い

定義
プライバシー：「私人の秘密、他人の干渉を許さない各個人の私生活上の自由のこと」。近年は「自分の情報をコントロールする権利」も含まれるとされる。 S03:p.41
プライバシー個人情報
定義明確な定義はない個人情報保護法により明確に定義
変化時代・社会とともに変化する社会通念法律改正に伴い変化
対応常に知識の更新が必要法律の改正に合わせた対応が必要
S03:p.42

プライバシー侵害の事例

インターネット時代の典型的なプライバシー侵害の例。 S03:p.43-45

① 個人データを無許可で公表

例

SNSに無許可で写真を投稿 → 写真をきっかけに友人の住所がバレたり、付きまとわれる可能性

② デマ・誤情報等によるプライバシーの侵害

例

全く関係のない事件の関係者としてネット上でデマが拡散 → 被害者への大きな被害、拡散した側も損害賠償を求められる可能性

③ プライバシー情報の分析による不利益の発生

例

学生の内定辞退率を本人の同意なく分析し外部に販売 → 他社がその情報を見て内定を出さないなど就職活動での不利益

③の事例はデータ収集時の目的とデータ使用時の目的が不一致であることから起こるプライバシー侵害の典型例。

OECD8原則

定義
OECD8原則：1980年にOECD（経済協力開発機構）がまとめたプライバシーガイドライン。2013年に改定。各国の個人情報保護関連法に大きな影響を与えている。 S03:p.46

1980年→2013年改定の背景：インターネットの普及（1992年個人向けサービス開始）・遺伝子研究の進歩による集団プライバシー問題の発生

① 目的明確化の原則
収集前に収集目的を明確にする。データの利用は目的と一致する場合のみ
② 利用制限の原則
収集目的以外の用途にデータを使用しない（本人同意済み・法令に基づく場合を除く）
③ 収集制限の原則
適法かつ公正な手段で収集する。本人に通知または同意を得て収集する
④ データ内容の原則
利用目的達成に必要な範囲内で、正確・安全・最新の内容に維持する
⑤ 安全保護の原則
合理的な安全保護措置により、滅失・改ざん・漏洩等から保護する
⑥ 公開の原則
データの取り扱い方針等を公開し、利用目的・データ管理者等を明記する
⑦ 個人参加の原則
自己のデータの有無と内容を確認する権利、開示拒否時に異議申し立てができる権利
⑧ 責任の原則
データ管理者は①〜⑦の原則を実施するための措置を遵守する責任がある
S03:p.47-55

他の章との対応：③収集制限 ＝ オプトイン・オプトアウト / ④データ内容 ＝ 可用性と関連 / ⑤安全保護 ＝ 完全性と関連

③収集制限の原則の実施例：Cookieの同意取得

用語
Cookie（クッキー）：ウェブサイトのアクセス情報（前回訪問時の情報）を一時的にテキストデータとして保存し、2回目以降のアクセス時に読み取ることでIDの自動ログインや適切な情報提案に活用されるもの。ID・パスワード・閲覧履歴等の個人データが含まれるため、近年はウェブサイトで明示的な同意取得が一般的になっている。 S03:p.49
CHAPTER 04

プライバシー（2）

日本における個人情報保護法

日本の個人情報は「個人情報の保護に関する法律（個人情報保護法）」で守られている。 S03:p.60-61

年出来事
2003年個人情報保護法 公布・一部施行
2015年改正（2017年施行）3年毎の見直し規定 / 事業者定義変更（5000件以上保有条件が撤廃）/ 要配慮個人情報・匿名加工情報の規定新設 / 個人情報保護委員会の新設
2020年改正（2022年施行）仮名加工情報の新設 / 漏えい時の報告・本人通知の義務化 / 不適正利用の禁止の明確化
個人情報の世界は変化が非常に速いため、個人情報保護委員会（https://www.ppc.go.jp/）のウェブサイト等で常に最新情報にアップデートすることが最も重要。

個人情報とは

定義
個人情報：生存する個人に関する情報で、特定の個人を識別できる情報、または個人識別符号が含まれる情報 S03:p.63

個人情報の例：氏名・住所・顔写真・生年月日・電話番号 / 指紋認証データ・静脈認証データ / マイナンバー・パスポート番号・保険証番号・運転免許証番号（＝個人識別符号）

💡 個人データ：個人情報データベースを構成する個人情報（住所・生年月日・性別など）。単独では個人を識別できない情報でも、データベースとして組み合わせることで個人情報となりうる。 S03:p.66

要配慮個人情報

定義
要配慮個人情報：個人情報の中でも特に取り扱いに注意が必要な個人情報。その情報に紐づくことで差別や偏見が生じる可能性のある個人情報。 S03:p.67

例：人種・信条・病歴・ゲノム情報・犯罪歴・犯罪の被害にあった事実 など

ルール内容
取得時必ず本人の同意が必要
第三者提供オプトアウト方式は禁止（オプトイン方式のみ可）

匿名加工情報と仮名加工情報

匿名加工情報仮名加工情報
加工の程度個人識別情報が復元できない程度に加工他の情報と照合しない限り識別できない程度に加工（照合すれば復元可能）
第三者提供本人の同意なしで提供可原則として禁止
利用目的収集時の目的に縛られる利用目的変更の制限等から除外（目的外の分析が可能）
S03:p.68
仮名加工情報を使うメリットは目的外の分析ができるようになること。個人情報保護法はデータに対する規制だけでなく、加工を通じて新しいサービスやイノベーションを生み出すためのサポートもしている。 ← S05

越境データとGDPR

定義
越境データ：国境を越えてやり取りされるデータ（通信・取引・映像・企業間データなど） S03:p.72

越境データ量の推移

年データ量
2005年4.7 Tbps
2015年154 Tbps（約33倍）
2020年718 Tbps（2005年比約153倍）

GDPR（一般データ保護規則）

定義
GDPR（General Data Protection Regulation）：2016年4月制定・2018年5月施行。EU域内の個人データの保護が目的。 S03:p.74
  • EU域外でも、EU域内の個人に対してサービスを提供する場合は対象（例：EU向けに輸出した車の走行データが日本に送られる場合も対象）
  • 越境データ（EU域外へのデータ持ち出し）には十分性認定が必要
  • 日本はEUの十分性認定を受けている
定義
十分性認定：EUと同等レベルの個人情報保護ルールが整備されているとEUが認定すること
GDPRが改正されたら日本の制度も変えないといけない形になるため、データというのは一国の問題ではなく世界的なトレンドや世界的な個人情報の扱いを常にチェックしていくことが重要。 ← S05

補足C：試験・レポートで重要な観点

  1. バイアスの種類と発生タイミング：選択バイアス（計画段階）・情報バイアス（実施段階）・測定バイアス（情報バイアスの一種）の違いを説明できること
  2. オプトインとオプトアウトの違い：具体例を含めて説明できること
  3. 研究不正の3類型：捏造・改ざん・盗用の定義と具体例を説明できること
  4. 情報セキュリティのCIA：3つを定義と対策とともに説明できること
    機密性→権限設定・端末制限・暗号化 / 完全性→ログ・入力制限・デジタル署名 / 可用性→バックアップ・分散
  5. OECD8原則の名称と内容：8つを説明できること（特に①目的明確化・②利用制限・③収集制限は最重要）
  6. 匿名加工情報と仮名加工情報の違い：復元可能性・第三者提供の可否・利用目的の制限の3点で比較できること
  7. 要配慮個人情報：定義・例・オプトアウト禁止の理由を説明できること
