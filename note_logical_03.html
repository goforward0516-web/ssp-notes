CHAPTER 01

状況を分析するとはどういうことか？

📌
第1章 学習目標

「状況を分析する」とはどのようなことか、また、それを的確に実施するには具体的にどのような方法や手順を取ればよいかについて説明できる ← S03:p.5

1-1. 状況把握が問題解決の鍵
状況が的確に分かれば、抱えている問題や課題の半分以上は解決したことと同じ ← S03:p.6
🎙 音声補足

問題に悩んだり解決の糸口が見つからないときの多くは、状況を適切に理解していないことによる。入り口が違えば答えは全く異なってくる。 ← S05

具体例：福島第一原発事故の捉え方

同じ事故であっても、捉え方によってまったく異なる問題定義になる。どれが正しいというものでもない。 ← S03:p.6

捉え方の視点問題の定義
防災・設備原発の津波被害への対策の問題
応急対応二次災害を食い止める対処方法の問題
避難・生活避難誘導および仮設住宅での生活確保の問題
技術・リスク原発に対する技術的な信頼性やリスクに関する問題
情報・社会放射線被ばくに関する政府の発表や風評被害の問題
← S03:p.6, S05
🎙 音声補足

問題を解決するにあたっては、その問題をどのように捉えているのか、どういう捉え方をしているのかということを常に意識し明確化することが重要。 ← S05

1-2. 視座によって状況の見え方が変わる
「状況」を分析するためには、必ず特定の視座が存在する。「自分は今、どこに視座を置いて考えているのか」を常に意識することが大切 ← S03:p.7
🎙 円錐の比喩

ある状況を紫色の円錐で象徴する。真横から見た人は「三角形」、真下から見上げた人は「正円」と言う。でも実態は円錐という立体。一つの状況も、捉え方・姿勢・見方によってそのあり様は変わる。どの角度から捉えるかによって解決の方向性は全く変わる。 ← S03:p.7, S05

1-3. 状況を認知するとは？（1）フッサールの現象学

エトムント・フッサール（Edmund Husserl） — オーストリアの哲学者。現象学の創始者。 ← S03:p.8

「主観と客観は一致するか？」
⇒ 『ここにあるコップは幻想か、現実か？』 ← S03:p.8
🎙 音声補足

コップという外部世界の事物がそこに本当に存在するかどうかをどうやって確かめたらよいか。目の前に見えて触ることもできるから存在するに決まっている…と言わないでほしい。なぜなら、今あなたは夢の中にいるかもしれないから。夢の中にいるのではないことを自身が証明することは、原理的に不可能。 ← S05

「客観的世界が実在している」という思い込みを一時的に保留（エポケー）した上で、この確信がなぜ成り立っているのかを問い直す。意識の外部世界を否定するのではなく、「外部世界がある」という確信が成立する条件を問う。（現象学） ← S03:p.8
🔗 外部補足

映画「マトリックス」の例：仮想世界に生きる人たちはそこが現実だと信じており、それが偽りの世界であることを確かめる術を持たない。フッサールの問いと同様のジレンマを描いている。 ← S05

🔗 外部補足（現象学の主要概念）

フッサールは意識の働き（思考する作用）をノエシス、意識に現れる対象（コップ・机など）をノエマと呼んだ。エポケー後に残るのはこのノエシス＝ノエマの相関関係であり、そこから意識の構造を純粋に記述しようとするのが現象学的還元。ハイデガー・サルトル・メルロ＝ポンティが後継者として現象学を発展させた。

1-4. 状況を認知するとは？（2）ソシュールの言語論

フェルディナン・ド・ソシュール（Ferdinand de Saussure） — スイスの言語学者。 ← S03:p.9

「言葉によって世界は分節され（分けられ）認知される。」 ← S03:p.9
虹の色数の例
文化圏虹の色数色名
日本7色紫・藍・青・緑・黄・橙・赤
アメリカ6色赤・橙・黄・緑・青・紫
マヤ族（メキシコ原住民）5色黒・白・赤・黄・青
← S03:p.9, S05
🎙 音声補足

虹の本質は赤から紫までの色のスペクトルが並んだ光の帯であり、色自体は連続的に並んでいる。隣り合った色と色の間に線引きはない。どこからどこまでを一つの色とするかはその国の文化が規定するものであって、「何色が正しいか」という問い自体が不毛。 ← S03:p.9, S05

「いかなる言葉も実体的な何かを名状しているのではなく、他の言葉との差異・関係を通して初めて意味を与えられる」
「命名（言葉）を通して、初めて事物の存在を認めることができる」 ← S03:p.10
🎙 音声補足

これは「事物が命名に先立って存在し、命名は後から行われる」という従来の考え方を逆転した斬新な発想。世界という連続体を言葉によって分割・区別することで「コップはコップ、スプーンはスプーン、土は土、川は川」と認識している。 ← S03:p.10, S05

🔗 外部補足

「自由」や「夢」といった目に見えない概念も、言葉を通して初めて人は理解できる。言葉が存在しなければ、それらの概念の存在自体も認識できない。 ← S05

🔗 外部補足（ソシュールの言語記号論）

ソシュールは言語記号をシニフィアン（記号表現：音・文字のイメージ）とシニフィエ（記号内容：概念・意味）の二面からなるとした。たとえば「イヌ」という音がシニフィアン、「犬という動物の概念」がシニフィエ。この結びつきは恣意的（文化・歴史で決まる必然性のない関係）であり、「犬」を「Dog」「Chien」「狗」と呼んでも意味は同じ。この発想が「言語は差異の体系である」という構造主義の出発点となった。

1-5. 状況を認知するとは？（3）差異・関係による定義
A という事物（状況）は「A と似ているもの」との違いや共通点（関係性）をもって記述できる ← S03:p.11
「曇り」の認識方法
形式表現評価
A = A「曇りとは曇りである」同語反復（トートロジー）：誤りではないが何も説明していない
A = B「曇りとは雨のことである」論理矛盾：説明になっていない
A ≠ B「曇りとは、晴れでも雨でも雪でもないもの」これでのみ事物を論理的に認識できる
← S03:p.11, S05
🎙 音声補足

A という事物を認識し説明するためには、それ以外のものとの違い、あるいはそこの共通性・関係性をもってしか説明や定義ができない。 ← S05

1-6. 状況分析図の作成手順（復習）
STEP 01
「似たもの」を探す
STEP 02
「軸」を探す
STEP 03
図にして分析する
← S03:p.12
ステップ1：似たものを探す ← S03:p.13
  • 状況を分析したい対象（A）を把握するため、Aの比較要素を取り出す
  • 比較要素には、Aと似ているものを探す
  • 「似ている」とは、何らかのカテゴリーで括ることができるということ
ステップ2：軸（関係性）を探す ← S03:p.14
  • 対象（A）と比較要素（B, C, D…）を比べて、共通する事項（＝軸）を抽出する
  • 例：「曇り」と「雨」→ どちらも「空を雲が覆っている状態」（共通項）
ステップ3：図にして分析する ← S03:p.15

「空を雲が覆っているか」×「空から何かが降ってくるか」の2軸でマトリクス図を描くと：

空を雲が覆っている（YES） 空を雲が覆っていない（NO）
何かが降ってくる（YES） 雨・雪 —
何も降ってこない（NO） 曇り ★ 晴れ
← S03:p.15
「曇り」とは、「空を雲が覆っているが、何も降っていない天気」のことであると分析できる。
▎第1章 まとめ ← S03:p.16
  • 状況を的確に把握できれば、「問題」の半分以上は解決したことと同じ
  • 状況を分析するためには、必ず特定の視座が存在する
  • 知りたいことを「深く知る」には、知りたいことと似ている比較要素を出し、それらとの関係の分析を行うのが良い
CHAPTER 02

状況分析図を描いてみよう（マトリクス図を描く）

📌
第2章 学習目標

具体的な事例を元にマトリクス図を描くことを通して、状況分析の方法を説明できる ← S03:p.19

CASE：マトリクス図を使って工場の事故を分類する
🏭 事例の状況 ← S03:p.20
  • 最近、工場で床に流れた油が原因で転倒事故が起きた
  • 部下の作業員が考え事をしながら作業をしていたところ、油が床に飛び散ったことに気付かず、足を滑らして転倒した
  • 工場長のBさんはこの事故の状況について分析を行いたい
（1）「似たもの」を探す ← S03:p.21

目的：今回生じた転倒事故の特徴を明らかにしたい → 似たもの＝他の転倒事故

🎙 音声補足

他の転倒事故の中で今回の転倒事故を位置づけることで、今回の事故の特徴を浮き彫りにできる。重要なのは「どのような目的で図を描きたいか」という目的に照らして、何を似たものにするかを決めること。 ← S03:p.21, S05

発散思考で似たものを列挙 ← S03:p.22
  1. 昨年、中堅社員が急ぎの電気計器部品を調達するため倉庫内を小走りに手押し台車で移動中、側溝に乗り上げて脱輪して転倒した
  2. 新人作業員が運搬作業中に、凍結していた路面に足を取られて転倒した
  3. 先月、フリーローラーの上に置いた板に乗り、ボルトを締めつける作業に慣れない作業員が頭から転落した
発散思考とブレインストーミング（復習）

ブレインストーミングとはアレックス・F・オズボーン（Alex F. Osborn）が考案した集団思考法。各自が他を批判することなく、自由に多くの意見を出し合って独創的なアイデアを引き出す方法。 ← S03:p.23

🎙 音声補足

ブレインストーミングは複数の人間が行う集団思考法だが、図解思考はそれを一人の頭の中で行う。自分の頭の中に何人もの自分をイメージして、複数の自分の間でブレストを行う方法。 ← S03:p.23, S05

発散思考のポイント ← S03:p.23
  1. 少しでも正しそうであれば思いつきでもよい
  2. 関連するものでもよい
  3. 書き出す量は多いほど良い（質より量）
  4. 書き出した内容を見て新たに思いつくものも書き出す（連想）
🎙 音声補足

適切な内容を初めから考えようとすると、一見重要そうでない実は重要な似たものを見落とす危険がある。メンタルも大切で、不安や悩みを抱えているときは発散思考が機能しにくい。気持ちにゆとりがある時、ポジティブな感情で臨むこと。 ← S05

共通項を出していくポイント ← S03:p.25-27
方法①：特徴を考える ← S03:p.25

それぞれの比較要素を見比べて、共通する特徴を探し出す。

例：犬・猫・うさぎの共通特徴
  • 哺乳類（生物学的分類）
  • ペットになることがある
  • ひげが生えている

抽象概念（「哺乳類」）だけでなく、具体的な言葉で共通項を繰り出すことが状況分析の解像度を高める。

方法②：効果や影響を考える ← S03:p.26

各要素が周囲のステークホルダーにどのような効果・影響を与えているかを考え、共通する効果や影響をイメージする。

例：犬・猫・うさぎを飼うことで…
  • メリット：寂しくない・癒しになる
  • デメリット：餌が必要、世話の手間
方法③：背景や理由を考える ← S03:p.27

あらゆる存在が社会の中で周囲と関係を持ちながら何らかの位置づけを持っていることを踏まえ、その存在の背景や理由を考える。

例：なぜ人間はこれらの動物をペットとして飼うか
  • 一人暮らしが増えた
  • 経済的に豊かになった
  • ストレスの多い社会になった
🎙 音声補足

発散思考というと適当にイメージする態度と思われるかもしれないが、発散思考はロジカルな側面を持っている。発想を広げる手段として論理力を活用するのが発散思考。 ← S05

（2）「軸」を探す ← S03:p.24

4つの転倒事故を比較して共通項・相違点を探す。

軸発見事例
第I軸（作業員要因） 注意力不足 vs 経験・スキル不足 床油転倒・手押し台車転倒 → 注意力不足
凍結路面転倒・フリーローラー転落 → スキル要求
第II軸（環境要因） 特殊事情 vs 常日頃の作業環境 床油転倒・凍結路面転倒 → 特殊事情
手押し台車・フリーローラー → 常日頃の環境
← S03:p.24, S05
図タイプの選択フローチャート
似たものは因果関係または順序関係にある？
YES フロー図
NO ↓
抽出した「軸」が重なり合っている？
YES ベン図
NO ↓
抽出した「軸」が3項目以上になる？
YES ツリー図
NO ↓
「軸」は定量化できる？
YES 点グラフ
NO マトリクス図
← S03:p.28
📋 今回の判定：因果関係なし / 重なりなし / 2軸のみ / 定量化できない → マトリクス図が適切
（3）図にして分析する ← S03:p.29
マトリクス図：工場の転倒事故分類
作業環境：常日頃の作業環境に原因 作業環境：特殊事情が原因
作業員：経験・スキルの不足 フリーローラーでの作業で頭から転落 新人作業員が凍結路面に足を取られて転倒
作業員：注意力不足 手押し台車で移動中、側溝に乗り上げて脱輪して転倒 床の油で転倒
★今回の事故
← S03:p.29
今回の事故は、「普段発生しない特殊な状況の下で、作業員の注意力不足が重なり生じた事故」と位置づけることができる。 ← S03:p.30
収束思考のポイント（図を描く際に実施）← S03:p.31
  1. 適切な表現に変える
  2. 重要でない事項は削除する
  3. 冗長な表現を省きコンパクトにする
  4. 似た内容の事項はまとめるか、一つに絞る
▎第2章 まとめ ← S03:p.32
  • 今回生じた転倒事故の特徴を明らかにしたい場合は、「他の転倒事故」を似たものに位置づけるのが良い
  • 「因果関係になく」「重なり合わず」「3項目以上でなく」「定量化できない」ので、マトリクス図が適切
CHAPTER 03

状況分析図を描いてみよう（ツリー図を描く）

📌
第3章 学習目標

具体的な事例を元にツリー図を描くことを通して、状況分析の方法を説明できる ← S03:p.35

CASE：ツリー図を使って「新型コロナ」を捉える
🎙 音声補足

5つの図の中でおそらく最も利用する局面の多いツリー図を使ってみる。 ← S05

お題：新型コロナとはどういった感染症なのか ← S03:p.36

（1）「似たもの」を探す：スコープ設定が重要 ← S03:p.37
似たもの候補評価
「新型コロナと似た言葉」❌ 言葉の特徴を比べたいわけではない
「流行のファッション・新商品」❌ 明らかに違うカテゴリー
感染症（結核・コレラ・O157・ペスト・ノロウイルス・インフルエンザ etc.）✅ 「新型コロナはどのような感染症か」を明らかにしたいなら感染症カテゴリーが適切
← S03:p.37, S05
似たもの探しのポイント：目的依存のスコープ設定 ← S03:p.38
似たものの範囲の設定の仕方は目的に依存する。まず初めに、状況を分析しなければならない背景・目的・理由・動機を明確にする。
DXを例にした目的別の似たもの設定
🔗 外部補足

DX（Digital Transformation）とは：デジタルテクノロジーを使用してビジネスプロセス、文化、顧客体験を新たに創造し、変わり続けるビジネスや市場の要求を満たすプロセスのこと。 ← S03:p.38

背景目的似たものの範囲似たものの例
DXに関する新規事業を生み出したい DXがもたらす価値を知る 顧客にDXと似たような価値を提供する手法 IT化・ブロックチェーン・情報銀行
社内業務プロセスのDXを推進中 業務プロセス改革におけるDXの特徴を明らかに 業務プロセスを変革する手段 AI活用・IoT導入・RPA導入
ビジネスモデル変革にDXを活用したい BM変革手法としてDXを位置づけた特徴を明らかに ビジネスモデル変革の手法 オープンイノベーション・デザイン思考・アジャイル開発・サステナビリティ戦略
← S03:p.38
🔗 外部補足

RPA（Robotic Process Automation）：PC上で行っている事務作業を自動化できるソフトウェアロボット技術。仮想知的労働者とも呼ばれる。 ← S03:p.38

（2）「軸」を探す ← S03:p.39
比較共通点・相違点導き出す軸
新型コロナ vs ノロウイルス病原体タイプ：同じウイルス性病原タイプ
新型コロナ vs O157病原体タイプ：O157は細菌性（異なる）
新型コロナ vs インフルエンザ感染経路：同じ飛沫感染・接触感染感染経路
新型コロナ vs ノロウイルス感染経路：ノロウイルスは経口感染（異なる）
ペスト・天然痘 vs 現代の感染症流行時期：近代以前に収束 vs 現代流行流行時期
← S03:p.39
🌳 今回の判定：因果関係なし / 重なりなし / 軸3項目以上 → ツリー図が適切
📝 注：2項目でもツリー図は描ける。点グラフとマトリクス図は「3項目以上」では描けないため、便宜的にツリー図をこの位置に置いている。 ← S03:p.40
（3）図にして分析する：感染症のツリー図 ← S03:p.41
⚠️ 感染経路は複数の感染経路を持つ感染症もある（重なり合う）ため、ここでは単純化している。本来はツリー図とベン図を組み合わせる発想も必要な場合がある。 ← S05
新型コロナとはウイルスを病原体とし、飛沫/接触感染を感染経路とし、現代に流行しているインフルエンザと同じ感染症である。 ← S03:p.42
▎第3章 まとめ ← S03:p.43
  • 似たものを出す際は発散思考を行う
  • 「新型コロナはどのような感染症か」を明らかにしたい場合は、「感染症」を似たものにすればよい
  • 似たものと比較し、共通している事項（軸）を見つける
  • 図を描く際は収束思考を行う

（1）「似たもの」を探す ← S03:p.48

（2）「軸」を探す ← S03:p.49
共通点（軸の候補）発見の根拠

🎙 音声補足

似たもの探しは重要だが、もっと重要なのはここから軸を探すこと。似たものの洗い出しは、適切な軸を探すためのヒントに過ぎない。 ← S05

点グラフのための軸の増やし方 ← S05
  1. 変形：思いついた軸の内容を変形させる（例：利益率 → 社員一人当たり利益率・利益額）
  2. 分割：既存の軸を分割する（例：従業員規模 → 正社員規模 + 非正規社員規模）
  3. 逆を探す：思いついた軸の逆（例：従業員数 → 役員の人数）
軸の絞り込み方 ← S05
  1. まとめる：複数の軸を大括りにする（例：正社員規模＋非正規規模 → 従業員規模）
  2. 捨てる：目的に最も重要な2つを残して残りを落とす
⚠️ 重要：2軸は全く異なるものにすること ← S03:p.51

例えば「昨年の売上高」と「3年前の売上高」の2軸は非常に似た傾向になり（右肩上がりの直線上に並ぶ）、解釈がほとんど意味をなさない。全く異なる視点の軸を選ぶことが必要。

📚 補足B：知っておくべき背景知識
  • エトムント・フッサール（1859-1938）と現象学：オーストリアのプロニッツ（現チェコ）生まれ。もとは数学研究者だったが哲学に転向し、現象学を創始。主著は『論理学研究』（1900）・『イデーン』（1913）・『ヨーロッパ諸学の危機と超越論的現象学』など。意識の働きを「志向性（ノエシス）」と呼び、意識に現れる対象を「ノエマ」と名付けた。弟子にハイデガー・サルトル・メルロ＝ポンティを輩出し、現象学は20世紀哲学の主要潮流となった。（[外部補足]）
  • フェルディナン・ド・ソシュール（1857-1913）と言語記号論：スイスのジュネーブ生まれ。近代言語学の父。主著『一般言語学講義』はソシュールの死後、弟子たちが1906-1911年の講義ノートをまとめ1916年に出版。言語記号は シニフィアン（記号表現：音や文字のイメージ）と シニフィエ（記号内容：概念・意味）の二面からなるとした。この結びつきは恣意的（必然性がなく歴史・文化で決まる）であり、例えば「犬」という概念は日本語では「イヌ」、英語では「Dog」と呼ばれても構わない。言語はモノに後から名前を付けるのではなく、言語（差異の体系）が世界を切り分け認識を成立させる——これが構造主義思想の源流となった。（[外部補足]）
  • オズボーンのブレインストーミング4原則（1953年）：Alex F. Osborn（1888-1966）が著書『Applied Imagination』で体系化。授業で紹介された4ポイントは、オズボーンが定めた4原則に対応している。① 批判厳禁（他者のアイデアを否定しない）→ 思いつきでよい② 自由奔放（どんな突飛なアイデアも歓迎）→ 関連するものでよい③ 質より量（できるだけ多く出す）④ 結合改善（他者のアイデアを組み合わせ発展させる）→ 書き出した内容から連想する。現在も広告・製品開発・教育現場で広く活用される。（[外部補足]）
  • DXの定義（経済産業省）：日本の経済産業省は「企業がビジネス環境の激しい変化に対応し、データとデジタル技術を活用して、顧客や社会のニーズを基に、製品やサービス、ビジネスモデルを変革するとともに、業務そのものや、組織、プロセス、企業文化・風土を変革し、競争上の優位性を確立すること」と定義している（2018年DXレポート）。（[外部補足]）
