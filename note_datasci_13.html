第1章

画像をデータにする

第1章 学習目標

画像データをデジタルデータとして表現する方法を説明できる。← S03:p.7

🔊 音声補足
コンピュータは数字（0と1）しか扱えない。画像・ことば・音楽・音声などの非構造化データを処理できるようにするためには、これらのデータを数字で表現する工夫が必要。← S03:p.8

画像データとは何か

📌 定義
画像データ：写真（静止画像）・映像（動画像）など、目で見て捉えられる情報全般。コンピュータで扱うためにはデジタルデータに変換する必要がある。← S03:p.9, S05

画像データの最大の特徴：データサイズが巨大になること。

100×100ピクセルのカラー画像 → 3万次元のベクトル
100 × 100 × 3（RGB） = 30,000次元　← S03:p.9
🔊 音声補足
100×100ピクセルはかなり小さい画像。実際には何千×何千のデータが毎日インターネット上に大量に投稿されている。こういった巨大なベクトルを持つ画像データをどう扱うのか、この先で詳しく説明する。← S05

画素（ピクセル）とは何か

📌 定義
画素（ピクセル, pixel）：画像を構成する最小単位の正方形。画像は無数の画素が格子状に並んで構成されている。← S03:p.12
🔊 音声補足
チューリップの画像をパソコンのプロパティで確認すると「2517 × 2517 ピクセル」と表示されている。これは、横に2517個・縦に2517個の画素が並んでいることを意味する。← S03:p.10, S05
図：画像のデジタル化プロセス（約5000倍拡大で画素が見える）　← S03:p.12
🔊 音声補足
手書き文字「13」を格子状の枠に当てはめて塗りつぶし、塗りつぶしたマスを「0」、塗られていないマスを「1」で表す。電気がついていない（黒）→ 0、電気がついている（明るい）→ 1、と覚えると分かりやすい。こうして画像を数字の集合で表現できる。← S03:p.13-14, S05

グレースケール画像の表現

📌 定義
グレースケール画像：色の濃さを数字で表現することで白黒の階調を持つ画像。← S03:p.15
段階数表現ビット数
2段階0（黒）/ 1（白）1bit
4段階0, 1, 2, 32bit
256段階0〜255（256階調）8bit
🔊 音声補足
256段階（256階調）にすると、左側の図でグラデーションのように細かく滑らかに表示できるようになる。256階調のグレースケール画像は画素数分の数字の集合で表現できる。例：11行×12列 = 132画素 → 132個の数字の集合。← S03:p.15-16, S05

カラー画像の表現：RGBモデル

📌 定義
RGBモデル：赤（Red）・緑（Green）・青（Blue）の3原色それぞれを0〜255（256階調）の数値で表し、その組み合わせで色を表現するモデル。← S03:p.17
図：RGBモデルによる色の表現例　← S03:p.17
🔊 音声補足
カラー画像は、RGBそれぞれの数値の集合（行列）が3枚重なることで表現される。右側の3枚の画像はチューリップをR・G・Bに分解したもの。同じ位置にある数字を1個ずつ取り出すことで1つの色を表現している。← S03:p.18-20, S05

カラー画像のベクトル表現と次元数

各ピクセルにRGB（3つの数値）が含まれるため、カラー画像をベクトルで表現すると：

次元数 = 画素数 × 3
← S03:p.21-22
⚠️ 重要な計算例
2517 × 2517ピクセルのカラー画像の場合：
2517 × 2517 × 3 = 19,005,867 ≈ 約1900万次元← S03:p.22
🔊 音声補足
カラー画像をベクトル表現するには、左から右・上から下にRGBの値を順番に並べていく。1つ1つのピクセルに数字が3つずつ入っているので「画素数×3次元」となる。1900万次元という数字が、画像データがいかに巨大かを示している。← S05

画像データの種類

種類表現方法特徴
ラスター画像
／ビットマップ画像		 画素（ピクセル）の集合 写真・複雑な画像の表現に向く。拡大すると粗くなる
ベクター画像 点とベクトルで表現 拡大に強い。写真などの複雑な画像は苦手
🔊 音声補足
ラスター画像とビットマップ画像は厳密には違う意味だが、どちらも「画素が集まって画像を表現している」という点では同じ。ベクター画像の「ベクター」はベクトルのことで、画像を点とベクトルで表現している。まずはラスター画像・ビットマップ画像の考え方をしっかり押さえておこう。← S03:p.23, S05

第1章 まとめ

  • ラスター画像／ビットマップ画像では画素の集合として画像を表現する
  • カラー画像では各画素にRGB（赤・緑・青）の値を持つ
  • カラー画像をベクトルで表現すると「画素数 × 3」次元の巨大なベクトルになる
← S03:p.24

第2章 学習目標

← S03:p.27

スカラー・ベクトル・行列の関係

図：スカラー → ベクトル → 行列の関係　← S03:p.28
🔊 音声補足
数字1つ1つがスカラー。前回学習した数字を1列に並べたものがベクトル。このベクトルを横にずらっと並べたもの（複数のベクトルを結合したもの）が行列と考えられる。ベクトルや行列には特別な性質があり、これを使うことで計算が楽になったり、人工知能の応用的な機能を作ったりすることができる。← S03:p.28, S05

行列の定義とサイズ表現

📌 定義
行列（Matrix）：数字を格子状に並べたもの。
  • 行（row）：横向きに並んだ数字の並び
  • 列（column）：縦向きに並んだ数字の並び
  • 成分（要素）：行列内の個々の数字。a_{ij} で「i行j列目の要素」を表す
← S03:p.29
📌 サイズ表現
「m行n列行列」のように表す（行→列の順）。
例：2行2列、3行3列、3行2列← S03:p.30
🔊 音声補足
行列という名前は「行」と「列」という言葉の組み合わせ。行は横方向、列は縦方向。サイズを表すときも行・列の順番で表す。添字 a_{11} は「1行目1列目の要素」を意味する。どこにある数字かを特定するために添字を使う。← S05

行列の演算①：足し算と定数倍

足し算：同じサイズの行列同士で行う。同じ位置の要素を足し合わせる。

📖 外部補足
サイズが異なる行列（例：3行3列と2行2列）の足し算はできない。必ず同じサイズ同士でなければならない。

定数倍：行列の全ての要素にスカラー（定数）αを掛ける。

🔊 音声補足 — 計算例
足し算の例： 
[1 3]   [2 1]   [1+2  3+1]   [3 4]
[2 2] + [5 2] = [2+5  2+2] = [7 4]
[3 4]   [6 4]   [3+6  4+4]   [9 8]
定数倍の例： 
    [1 3]   [3×1  3×3]   [3  9]
3 × [2 4] = [3×2  3×4] = [6 12]
← S03:p.33-34, S05

行列の演算②：掛け算

⚠️ 掛け算のルール（重要）
左行列の行と右行列の列を対応させて内積を計算する。

2行2列の行列A × 2行2列の行列B の場合：

[a11 a12] [b11 b12] [a11·b11 + a12·b21 , a11·b12 + a12·b22]
[a21 a22] × [b21 b22] = [a21·b11 + a22·b21 , a21·b12 + a22·b22]
🔊 音声補足 — 計算手順
  1. 左行列は行（横）に注目、右行列は列（縦）に注目する
  2. 左行列の1行目と右行列の1列目を取り出し、対応する要素を掛けて足し合わせる → 結果の行列の(1,1)成分
  3. 左行列の1行目と右行列の2列目 → 結果の(1,2)成分
  4. 左行列の2行目と右行列の1列目 → 結果の(2,1)成分 …というように順番に計算
← S03:p.35-48, S05

計算例：

[1 2] [1 3] [1×1+2×2 1×3+2×4] [5 11]
[3 4] × [2 4] = [3×1+4×2 3×3+4×4] = [11 25]

掛け算の注意点 2つ

注意①：掛け算が可能な組み合わせを確認する

左行列の列数 ＝ 右行列の行数 でなければ計算できない。

📖 外部補足
(m×n行列) × (n×p行列) = (m×p行列) となる。n が一致しないと計算不能。

注意②：順序は基本的に交換できない

A×B ≠ B×A（一般的に）。掛ける順序を変えると結果が変わる。

🔊 音声補足
スカラー（普通の数値）の掛け算は順序を変えても結果が同じだが、行列では右から掛けるか左から掛けるかで結果が変わる。行列を扱う際は順序に必ず注意すること。← S03:p.49, S05

行列の掛け算の意味：ベクトルの移動

📌 重要な概念
ある「ベクトル」に左から「行列」を掛けることで、ベクトルの位置を移動できる。← S03:p.58
図：行列の掛け算によるベクトルの移動（平行移動・回転）　← S03:p.54, p.57
🔊 音声補足
例1（x軸方向に2倍）：行列 [[2,0],[0,1]] をベクトル [2,1] に左から掛けると [4,1] になる。ベクトルの位置がx軸方向に移動した。

例2（30°回転）：cos/sinで構成された回転行列をベクトル [√3,1] に左から掛けると [1,√3] になる。ベクトルが30°回転した。

これが「行列の掛け算の意味」。行列はベクトルを移動させるためのルール（変形ルール）。← S03:p.54-57, S05
第2章 まとめ← S03:p.59
  • 行列の演算：足し算（同じ位置の要素を足す）、定数倍（全要素に掛ける）、掛け算（行×列で積和）
  • 掛け算の注意：①可能な組み合わせの確認 ②順序交換不可（例外：単位行列）
  • ベクトルに左から行列を掛けることで、ベクトルを移動（平行移動・回転等）できる

画像の変形：アフィン変換

第2章で学んだ「行列の掛け算はベクトルを移動させる」を画像に応用する。

画像は画素の集合であり、各画素の位置はベクトルで表現できる。

行列 × 画素の位置ベクトル = 移動後のベクトル
全画素に適用することで画像全体を変形できる　← S03:p.70
📌 定義
アフィン変換（Affine Transformation）：特定の3×3行列を画素の位置ベクトルに掛けることで、画像の拡大・縮小・移動・回転などの変形を行う手法。← S03:p.64-69, S05
図：アフィン変換行列の読み方と計算例（(0,100) → (50,110) へ移動）　← S03:p.67
🔊 音声補足
行列は3×3なので、2次元の座標ベクトルに「1」を追加して3次元にしてから計算する。計算結果の3つ目の「1」は便宜上追加したものとして無視する。

変換の例：
・0.6倍縮小 + x,y方向に50移動 → 画像が小さくなって右下にずれる
・90°回転行列 × x軸方向250移動行列 → 画像が横向きに回転
・0.5倍 × 30°回転 × x,y方向に100移動 → 小さく斜めに回転
← S03:p.64-69, S05

行列の応用：ベクトル空間上のデータ操作

行列の掛け算は画像に限らず、ベクトル化できるあらゆるデータに応用できる。

図：行列の応用 — 検索キーワードから関連商品を計算　← S03:p.72-73
🔊 音声補足
ベクトルが「情報の集合」で、行列が「関連度のルール」だとすれば、行列をベクトルに左から掛けることで、そのベクトルに関連する別の情報（ベクトル）を計算できる。同じ行列をキーワードBに使えばケーキに移動できる。検索のアルゴリズムでもこのような行列×ベクトルの考え方が使われている（実際にはより複雑な計算をしている）。← S03:p.72-73, S05
📖 外部補足
この概念は自然言語処理における「Transformerアーキテクチャ」の注意機構（Attention Mechanism）に直結する。Query・Key・Value行列を用いて単語間の関連度を計算する仕組みも、本質的には「行列×ベクトル」の操作の積み重ね。

周辺画素の活用：欠損値予測と畳み込み処理

1つの画素単体では「色情報」しかわからない。しかし、周辺の画素も含めて処理することで様々な情報を取り出せる。

🔊 音声補足 — 欠損値の予測
画素1個1個は人間の目に見えないほど小さいため、隣接する画素と色があまり変わらない。この性質を利用して、欠損した画素の値を周辺画素から予測・補完できる。← S05
📌 定義
畳み込み処理（Convolution）：注目する画素の周辺（例：3×3の範囲）も含めて演算することで、以下のような情報を抽出する手法。別名：フィルタ処理・畳み込み演算。← S03:p.75, S05
処理内容説明
ノイズ除去周辺画素の平均をとることで細かいノイズを平滑化
特徴抽出（エッジ検出）明るさが急激に変化するエリアを検出→物体の輪郭を自動抽出
縦線・斜め線の検出特定方向の明暗パターンを検出
📖 外部補足
畳み込み処理のフィルタ（カーネル）は、小さな行列（例：3×3）として表現される。この行列を画像全体にスライドさせながら掛け算をすることで特徴マップを生成する。これはまさに「行列の掛け算をデータ処理に活用している」例。

第3章 まとめ

  • 各画素の位置ベクトルに行列を掛けることで画像の変形（アフィン変換）が可能
  • 行列の応用はベクトル化できるデータ全般に利用可能（検索など）
  • 周辺の画素を含めて扱うことで欠損値の予測・特徴抽出（畳み込み処理）ができる
  • 特徴抽出は現在の人工知能技術で非常に重要
← S03:p.76

第12回（ベクトル）・第13回（行列）を通じて学んだ線形代数は、現代のデータサイエンス・AIの基盤。

補足B：知っておくべき背景知識

📖 外部補足
線形代数（Linear Algebra）との関係
今回学んだ行列の演算は「線形代数」という数学の分野の基礎。機械学習・データサイエンス・コンピュータグラフィックスのほぼすべての処理に線形代数が使われている。
📖 外部補足
8bit = 256段階の理由
2の8乗 = 256。コンピュータは2進数を使うため、各ビットが0か1を表す。8つのビットで最大256種類の値（0〜255）を表現できる。
📖 外部補足
NumPyとは
Pythonの数値計算ライブラリ。行列・配列の演算を高速に行える。画像はNumPyの多次元配列として扱われる（shape: 高さ × 幅 × チャンネル数）。
