第1章仕事とは「問題解決のプロセス」

仕事と問題の不可分な関係

私たちの生活は毎日同じように見えても全く同じではなく、仕事も同様である。定型的な業務は人間からAIやロボットに代替されていく方向に向かっており、業務の全てが完全に代替されるというよりは一部のみ自動化されるケースが多い。一方で、非定型業務（やり方・進め方をいちいち考えなければ進まない仕事）における人間の役割が期待される。 ← S03:p.6

核心メッセージ

直面する問題にポジティブに向かっていき、それを解決することに仕事の価値がある。したがって、仕事とは「問題解決のプロセスそのもの」である。 ← S03:p.6

問題解決力とは何か

出典定義・主張
奈良井安『問題解決力を鍛えるトレーニングブック』
（かんき出版、2002年、34ページ） S03:p.7		 問題とは『求められている状態』と『現実の状態』との間の好ましくない「差」のこと

（ディスカヴァー・トゥエンティワン、2008年、7ページ） S03:p.7		 アインシュタインの言葉「いかなる問題も、それを作り出した同じ意識によって解決することはできない」を引用し論を展開。問題を同じ意識で把握しようとすると視野が狭まる
🎙️ 音声補足

問題解決とは、問題そのものを冷静かつ客観的に捉えるところから始まる。解決の前に大切なのは、対象とする問題を的確に定義し、高い視座と広い視野で認識すること。次に原因を分析し、最後に解決策を導き出す。これが問題解決の基本プロセス。 ← S05-A/B

定義

問題解決は、発想技術・原因分析・状況把握・意思決定の総動員が求められるとともに、その解決に当たっては、状況に応じてコーチングや交渉力も活用する、高度で複合的な思考活動 ← S03:p.8

フレームワーク例② 親和図法

親和図法（しんわずほう）は、真の問題（真因）を特定し、括りだして解決方策を検討しやすくする方法。共通の真因について施策を考えることで、複数の問題を一気に解消できるメリットがある。 ← S03:p.11

外部補足・確認済

親和図法は川喜田二郎（東京工業大学名誉教授・文化人類学者）が1960年代に考案したKJ法と同一の手法。KJ法は株式会社川喜田研究所の商標登録（商標登録第4867036号）であるため、QC（品質管理）の世界では著作権・商標の問題を回避するため「親和図法」という名称が使われる。「新QC7つ道具」の一つとしても採用されている。 [外部補足・確認済: Wikipedia「KJ法」/ Slideshare「親和図法・KJ法」]

第2章フロー図を使って問題解決を図る

どのような問題にも必ずそれを生み出している原因がある。真の原因をつきとめることができれば問題を正しく捉えられるとともに、解決の手立てははっきりと見えてくる。問題が起きたときすぐに事に当たろうとするのでなく、「この問題はなぜ起きているのか？」を考える習慣をつけることが重要である。 ← S03:p.24

図4: フロー図による根本原因特定の概念図 S03:p.25（再掲p.16）
フロー図による問題解決 基本3ステップ
Step 1問題を具体的に定義する（抽象的な課題設定では解決の糸口を探りにくい）
▼
Step 2因果の連鎖を図にする（直接原因から始め、原因の原因を次々に辿る）
▼
Step 3図を精査し解決の糸口を探る（これ以上さかのぼれない「根本原因」を特定）

特徴: 列挙した理由の連鎖を論理的に整理しつつ、これ以上さかのぼれない理由や原因に着目し、それを解決する。 ← S03:p.26

CASE 2: 期限内に調査を完了させるには？

【状況】 マーケティング会社J社の調査員。外食産業の企業動向調査で、郵送アンケートの不明箇所を電話確認しているが、報告期日まで3週間なのに集計作業に入れない。このペースだと1週間近く提出が遅れる見込み。聞き取り要員を3人→6人に増やせば期日に間に合うが、人件費が膨らみプロジェクトが赤字になる。 S03:p.31

図6: 調査期限遅延フロー図（3段階の深掘りプロセス） S03:p.32-37
🎙️ 音声補足（フロー図作成の注意点）

原因を深掘りしても問題が特定できない場合には、一旦深掘りをやめて、最初に定義した問題に戻り、他に見落としていた原因がないかを考え直すことが必要。原因は1つとは限らないため、考えうる限りの原因を洗い出す。レベル感を見たとき、細かすぎる原因は1つにまとめたり、逆に抽象的すぎる原因は複数に分解して具体的な表現に見直すなど、精査しながら描くことが重要。 ← S05-A

第2章まとめ S03:p.38
  • フロー図は根本問題を突き止めるのに効果的な図
  • 作成手順: ①問題を具体的に定義する → ②因果の連鎖を図にする → ③図を精査し解決の糸口を探る
  • 対処療法（調査員の増員）ではなく、根本原因（情報共有不足）を解消することで増員なしに解決できた
第3章ツリー図を使って問題解決を図る

多くの場合、問題は複数の細かい問題にばらすことが可能。「それは具体的にはどのような問題なのか」を自問することで問題は小分けにしやすくなる。我々の思考回路が問題を丸ごと一体的に捉えてしまう傾向があり、そういった認識の枠組みが問題解決を困難にしている場合がある。問題を分割することで解決の糸口を探るにはツリー図が最も適している。 ← S03:p.42

ツリー図による問題解決 基本3ステップ S03:p.44
Step 1問題を具体的に挙げる（一見些末と思われることも全て洗い出す）
▼
Step 2類型化する（グループに束ねる。迷ったら5W1H・PDCA・人・もの・お金等を参考に）
▼
Step 3個別に解決策を探る（抜け漏れ・ダブりをチェックしながら）
🎙️ 音声補足（類型化のメリット・デメリット）

一般的な分類方法（5W1H・PDCA等）を用いるメリット: 他の人にとっても理解されやすい分類にできる。デメリット: 誰も気づかなかった独自の解決策は導かれにくい。目的に照らして適切な類型方法を選ぶことが大切。 ← S05-A

第4章ベン図を使って問題解決を図る

仕事上の問題は、その人の立場や経験、知識の深さなどによって見え方が異なる。専門的知見を持つ人と一般常識レベルの人の間では捉え方が大きく異なり、立場・経験・専門性の深さ・価値観などの違いが意見の対立を生じさせる場合は厄介である。唯一の正解があるわけではないからこそ、関係者が合意できる答えを探すことが重要になる。 ← S03:p.61

📚 補足B: 知っておくべき背景知識

  • 横田尚哉（よこた・ひさや）について
    株式会社ファンクショナル・アプローチ研究所代表取締役社長。元パシフィックコンサルタンツ株式会社本社部長。GEの改善手法「バリュー・エンジニアリング（価値工学）」を取り入れ10年間で総額1兆円の公共事業改善に乗り出しコスト縮減総額2000億円を実現。著書に「ワンランク上の問題解決の技術」「問題解決のためのファンクショナル・アプローチ入門」等（いずれもディスカヴァー・トゥエンティワン刊）。TV番組「情熱大陸」出演歴あり。[外部補足・確認済: 講演依頼.com / Schoo講師プロフィール / ディスカヴァー公式サイト]
  • 親和図法 = KJ法 について
    親和図法は、文化人類学者・川喜田二郎（東京工業大学名誉教授）がフィールドワークで得た膨大なデータを整理するために1960年代に考案。1966年に著書「発想法」（中公新書）で発表。KJとは「Kawakita Jiro」の頭文字。KJ法は株式会社川喜田研究所の商標登録（商標登録第4867036号）のため、QC（品質管理）の「新QC7つ道具」では「親和図法（Affinity Diagram）」という名称が使われる。[外部補足・確認済: Wikipedia「KJ法」/ Slideshare「親和図法・KJ法」]
  • 欠点・希望列挙法の開発元GEホットポイント社について
    ホットポイント社（Hotpoint）は、GE（ゼネラル・エレクトリック社）の子会社として家電製品を製造・販売していた企業。GE（ゼネラル・エレクトリック社）はトーマス・エジソンが1892年に設立した世界最大級の電機・エネルギー複合企業（現在はGE Aerospace等に分社化）。[外部補足・確認済: 複数の発想法解説サイト]
  • アインシュタインの引用について
    「いかなる問題も、それを作り出した同じ意識によって解決することはできない」という言葉は、横田尚哉「ワンランク上の問題解決の技術」（ディスカヴァー、2008年）7ページに引用されているもの。問題を生み出した思考レベルのままでは解決できない = 視点の転換が必要、という意味合いで使われる。[外部補足・確認済: S03スライドに出典明記 / ディスカヴァー公式サイト書籍情報]

🎯 補足C: 試験・レポートで重要な観点

  • 3つの図の使い分けを問われたら: フロー図=根本原因の特定 / ツリー図=問題の分割 / ベン図=対立する視点の融合
  • 各図の3ステップはすべて必須（試験頻出）
  • 根本原因と対処療法の違いを具体例で説明できるようにしておく（CASEを活用）
  • ベン図では主張の円ではなく価値観の円の接点を探ることがポイント（ここが最頻出）
  • ツリー図の抜け漏れ・ダブりのチェック機能が有用である理由を説明できること
  • 「問題解決は発想技術・原因分析・状況把握・意思決定の総動員を要する高度で複合的な思考活動」という定義を押さえる
  • 対処療法と対症療法は混同しないよう注意（本講義では「対処療法」を使用）
  • フロー図では原因が1つとは限らないため、考えうる限りの原因を洗い出すことが重要

🔗 補足D: 関連概念・次回への接続

  • 弁証法（ヘーゲルの弁証法）との関係
    ドイツの哲学者G.W.F.ヘーゲル（1770-1831）が体系化した思考法。テーゼ（正命題）とアンチテーゼ（反命題）の対立を止揚（アウフヘーベン）してジンテーゼ（総合命題）を導く弁証法的思考。ベン図での問題解決（「立場を超えてより高い次元で合意できる答えを探す」）はこれと構造的に近い。[外部補足・確認済: 哲学的文脈は一般的知識]
  • Why-Whyツリー（なぜなぜ分析）との関係
    フロー図の「根本原因を特定する手法」は、製造業で広く使われる「なぜなぜ分析（Why-Why Analysis）」と概念的に近い。なぜなぜ分析はトヨタ生産方式でも活用されており、問題に対して「なぜ？」を5回繰り返すことで根本原因に到達する手法。[外部補足]
  • ロジカルシンキング「5つの図」の全体像（前回以前の内容）
    本講義では「5つの図」を軸とした図思考が展開されているが、本回（第6回）では問題解決に特に適したフロー図・ツリー図・ベン図の3つが扱われた。残り2つの図については前回以前の授業で学習済み（本ノートのソースからは内容を確認できない）。[外部補足: S03授業構成スライドより確認]
