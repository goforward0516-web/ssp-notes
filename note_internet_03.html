情報デザインの定義
🔑 核心定義
情報デザイン（Information Design）
あらゆる情報をわかりやすくまとめて、伝えること ← S03:p.9

より詳細な定義はスライドに記載されている。← S03:p.10

情報デザインとは、あらゆる情報をわかりやすく可視化してまとめ、構造化し、構成する要素の関係性をわかりやすい伝えることである。また、情報というものを人がどうすれば便利に効率的に使うことができるかを考え、その形に落とし込み準備するスキルと知識である。
🎙 音声補足
「情報デザイン」の語は「情報」と「デザイン」の組み合わせ。デザインとは客観的な表現が行えるものを指す。アート（主観的な表現）との違いを意識すること。例として「家のデザイン」は他の人が元のデザインを見て再現できる＝客観的。 ← S05-A
概念表現の種類具体例
アート主観的な表現絵画・彫刻など作家の内面から生まれる作品
デザイン客観的な表現建築・工業製品・路線図など第三者が再現可能なもの

← S03:p.8

情報デザインの簡単な事例

情報デザインは身近なところで広く使われている。以下の3つが代表例として挙げられた。← S03:p.15-17

① 路線図

JR東日本などの路線図は情報デザインの典型的な事例。膨大な駅・路線の情報を1枚の紙で表現している。← S03:p.15

🎙 音声補足
路線図の工夫ポイント：①山手線を緑の円で表現（形状で路線種別を表す）、②主要駅を黒い楕円＋白文字で表記、③各路線を専用カラーで色分け、④方角・位置も可能な限り再現。ICカード普及で路線図を見る機会は減ったが、電車誕生時から鉄道会社が考え抜いた情報デザインの結晶である。 ← S05-A
② ピクトグラム（絵文字）

言語・国籍・障害の有無に関わらず、視覚的なマークで情報を伝える手法。← S03:p.16

🎙 音声補足
海外でその国の言語がわからなくても、トイレのマークさえ見ればトイレの場所がわかる。言語的な障壁を取り除くことができる表現手法がピクトグラム。← S05-A
③ 数値の図式化（グラフ・チャート）

大量の数値データをグラフ・円グラフとして視覚化することで、直感的な理解を実現する。← S03:p.17

🎙 音声補足
例：総務省「通信利用動向調査」の棒グラフを見れば、スマートフォンの利用が増加し携帯電話・PHSが減少していることが色・濃淡だけで直感的にわかる。Excelのように数値が羅列された状態から、分析してグラフ化することで初めて「情報デザイン」として機能する。 ← S05-A

CH.2インタフェースの考え方
インタフェースとは
🔑 定義
インタフェース（Interface）
意味：接面または境界面。
ハードウェアやソフトウェア、人間（ユーザ）などが、お互いに情報をやりとりする際の接する部分を指す。 ← S03:p.26
🎙 音声補足
スマートフォンの画面を指でタップするとき、指と画面が「接する部分」がまさにインタフェース。本科目（インターネット入門）ではソフトウェアの範囲に絞って学習する。 ← S05-A
インタフェースデザイン（設計）とは

人間が情報機器などを扱う際に、必要な操作方法や表示方法をユーザ視点でデザイン（設計）すること。← S03:p.30

🎙 音声補足
天気アプリを開発するとき、「ただ天気を表示するだけでは不十分」。どうしたら直感的に天気の状況をユーザに認知させられるか？詳しい天気情報をどう簡単に操作させるか？これを開発前の設計段階で考えるのがインタフェースデザイン。 ← S05-A
ユーザインタフェース（UI）の種類

ユーザインタフェースとは、人間がコンピュータを扱う際に必要な操作方法や表示方法のこと。← S03:p.31

種類正式名称特徴例
CUI Character User Interface テキストによる命令をキーボード入力でコンピュータを操作する コマンドプロンプト、ターミナル
GUI Graphical User Interface アイコンなどの視覚的な要素をポインティングデバイスや指のタッチで操作する Windowsデスクトップ、スマホ操作

← S03:p.31

🎙 音声補足
GUIはユーザにとって最も身近なインタフェース。ユーザの操作性・視認性を高めることでシステムや製品の満足度が高まるため、非常に重要。例えばゲームアプリで操作方法が最初から直感的にわからなければ、ユーザはすぐにゲームをやめてしまう。チュートリアルの導入などもUI設計の一環。 ← S05-A
GUIにおける重要箇所

GUIを構成する主要な要素は以下の通り。← S03:p.33

要素名説明
ウィンドウ画面全体を表示する枠。アプリケーション起動時に現れる表示領域

ジェスチャーインタフェース：人間の体の動きを読み取って入力・操作を行うインタフェース。例: Nintendo Switch、XBOX Kinect など。← S03:p.42

VUI（Voice User Interface）：声を使ったインタフェース。例: iPhoneのSiriなど。← S03:p.42

🔑 UX（ユーザエクスペリエンス）
ユーザが商品やサービスを通して得られる体験。「楽しい」「嬉しい」「面白い」などをどう与えられるかが重要。
サービス提供後にユーザの意見をフィードバックして改良・向上させ続けることが継続利用につながる。 ← S03:p.44
🎙 音声補足
「インタフェースデザイン → UI → UX」は時間軸に沿った流れ。まず設計（インタフェースデザイン）をして、次に操作性を考え（UI）、提供後にユーザ体験をフィードバックしながら改良し続ける（UX）。 ← S05-A

CH.3情報システムにおけるユニバーサルデザイン
ユニバーサルデザイン（UD）とは
🎙 音声補足
導入として「もし手が使えなかったら？足が使えなかったら？目が見えなかったら？」という問いかけ。扉や階段は一般の人には問題ないが、身体的な障壁があるユーザには「使えないデザイン」になる。これを踏まえて自動扉・スロープ・点字ブロックなどが提示された。 ← S05-A
🔑 定義
ユニバーサルデザイン（Universal Design）
年齢や文化、国籍、障害の有無にかかわらず、すべての人が使いやすいように工夫された用具・建造物などのデザインを目指す概念。
→ 誰もが最初から使うことができるデザイン ← S03:p.51
UDとバリアフリーの違い

ユニバーサルデザイン

作る前段階から障壁が発生しないように設計する。「最初から誰でも使える」。

例：自動扉・スロープ・UD フォント

バリアフリー

既に存在する障壁を後から取り除く。「後付けで問題を解消する」。

例：既存ドアに手すりを追加

← S03:p.52

ユニバーサルデザインの7原則

アメリカのロナルド・L・メイス氏（Ronald Lawrence Mace, 1942–1998）によって提唱された。ノースカロライナ州立大学デザイン学科の建築家・教育者で、1989年に同大学にユニバーサルデザインセンターを設立した人物。← S03:p.53

#原則概要
1公平に使えることどんな能力の人でも公平に利用できる
2柔軟に使えることさまざまな好みや能力に対応できる
3単純で直感的に使えること経験や知識に関わらず理解しやすい
4情報が知覚できること必要な情報を効果的に伝えられる
5失敗に対して寛大であること誤った操作に対する危険や悪影響を最小化
6身体的な努力を少なくすること疲労なく効率よく快適に使える
7接近して使えるように大きさと空間の余裕を確保すること体格・姿勢・移動能力に関わらず利用できる
情報システムにおけるユニバーサルデザイン

WebサイトやアプリケーションにUDを適用する具体的な施策：← S03:p.56

施策カテゴリ具体例対象ユーザ
文字表現 ユニバーサルデザインフォント（UDフォント）：文字の各部を離して視認性を向上 視認性に問題がある人全般
視覚表現 ピクトグラム：言語が変わってもアイコン自体は変わらない（PowerPoint等） 外国語話者・識字困難者
情報可視化 インフォグラフィックス：情報やデータを視覚的に表現（地図・標識・技術文書） 広く一般
配色 色のユニバーサルデザイン：色覚多様性を考慮した配色（赤×緑→赤×青など） 色覚異常がある人（日本人男性の約5%）
音声 音声読み上げ機能：iPhoneの「コンテンツを読み上げ」機能など 視覚障害者
字幕 字幕機能：動画コンテンツに字幕を付与。多言語表示にも対応 聴覚障害者・外国語話者
入力 音声入力機能 手が不自由で入力が難しい人

← S03:p.57-63

🎙 音声補足
UDフォントの具体例：「ナ」という文字で比較すると、通常フォントは文字がつながって書かれるが、UDフォントは各部を分離することで視認性を高めている。Wordなどのテキスト系アプリでフォント選択をすると「UDフォント」という選択肢が表示される場合がある。 ← S05-A
🎙 音声補足
インフォグラフィックスの例として福岡のソウルフード（ラーメン・もつ鍋・餃子）の店舗数データを視覚化したもの、レストラン数を示したものが紹介された（出典：Fukuoka Facts）。 ← S05-A
アクセシビリティとユーザビリティ

アクセシビリティ

誰にとっても使いやすく、利用しやすいこと。幅広いユーザを対象とする。

← S03:p.64

ユーザビリティ

開発者が目的とする特定のユーザに対して使いやすくすること。

← S03:p.64

🎙 音声補足
アクセシビリティはUDの7原則の中でも特に「原則6：身体的な努力を少なくすること」に深く関係する。目が不自由なユーザには音声読み上げ、耳が不自由なユーザには字幕・代替テキスト、入力が難しいユーザには音声入力がアクセシビリティ機能の代表例。 ← S05-A
🎙 音声補足（実例）
毎年開催される格闘技ゲームの世界大会（EVO等）で、目が見えないプレイヤーがアクセシビリティ機能を使って健常者と対戦し勝利した事例が紹介された。多くのサービス・アプリに既にアクセシビリティ機能が実装されているが、使う必要がないため気づかないことも多い。 ← S05-A
💡 情報デザイン × ユニバーサルデザインの関係
「情報デザイン（あらゆる情報をわかりやすく伝える）」＋「ユニバーサルデザイン（誰もが使えるデザイン）」を組み合わせることで、すべての人に公平なシステム・アプリが構築できる。← S03:p.67-68

CH.4情報デザインの関連技術
情報デザインの関連技術

情報デザインの技術は以下のような身近なモノに集約されている。← S03:p.75-79

関連技術・媒体情報デザインとしての要素
新聞・図書館 どこに何があるかを提示するために情報デザインの技術が集約されている。新聞は写真＋見出し＋記事本文をブロックで区切り膨大な情報を整理。図書館はジャンル別・50音順の配架・検索機能で利用者に提示。
スマートフォン 説明書がない状態でユーザがすぐに使えるインタフェースになっている。iPhone購入時に分厚い説明書が不要なのがその証拠。
プログラミング 目的があり、仕様がプログラミング言語ごとに決まっており、ルールに則して行う。ソースコードは他の人も同じものを再現できる（客観的表現＝デザイン）。
Webデザイン わかりやすさ・視認性・使いやすさ・継続利用・ユニバーサルデザインなどを包括している。ネットショッピングで使いにくければ購入まで至らない＝UXまで含めた実践の場。
情報デザインにおける技術の定義

藤本 貴之氏（東洋大学総合情報学部）は以下のように述べている。← S03:p.80

  • 複雑な情報を整理し、わかりやすく編集して伝える技術
  • 情報や技術への煩雑な経路を、わかりやすく構造化して筋道を立てる技術
情報デザイン技術の活用方法
🎙 音声補足（具体例）
コンビニで新しいパンを売りたいとき：年齢層・購入時間・パンの種類・一人当たりの購入単価などのデータを収集し、マーケティング分析した上で資料化してプレゼンする。このとき、集めた情報を「わかりやすく表現できない」なら相手に伝わらず、情報が無意味になってしまう。 ← S05-A
担当教員の研究事例
🎙 音声補足（講師研究紹介）
藤田光治教員が大学院時代に行ったスマートフォン文字入力インタフェースの研究事例が紹介された。当時の3つの入力方式（フリック入力・QWERTY入力・トグル入力）の利便性・入力速度を被験者実験で比較（→情報の収集）。その結果から、両手で持ちやすく大きなボタンのレイアウトを採用した新しい入力インタフェースアプリを開発（→インタフェースデザイン→UI実装）。スワイプで特殊文字に切り替え可能にするなどスマートフォンの特性を活かした設計。これが情報デザインの過程の一例。 ← S05-A

📚 補足B: 知っておくべき背景知識
  • メイス氏（Ronald L. Mace）について[外部補足]： ロナルド・ローレンス・メイス（1942年8月3日–1998年6月29日）はアメリカの建築家・製品デザイナー・教育者・コンサルタント。ユニバーサルデザインという用語を初めて生み出した人物として知られる。ニュージャージー州ジャージーシティ生まれ。9歳のときにポリオに感染し、以後車椅子を使用した。ノースカロライナ州立大学デザイン学科を1966年に卒業。1973年には米国初のアクセシビリティ建築基準法（ノースカロライナ州法）の策定に深く関与した。1989年にノースカロライナ州立大学にアクセシブルハウジングセンター（後のユニバーサルデザインセンター）を設立した。UD7原則はメイス氏を中心とする複数の建築家・デザイナーらが協力して1997年4月に策定した（Version 2.0）。
  • 色覚多様性（色覚異常）の割合[外部補足]： 先天赤緑色覚異常は日本人男性の約5%（20人に1人）、日本人女性の約0.2%（500人に1人）に見られる。X染色体連鎖性遺伝のため男性に多い。日本全体では約320万人が該当するとされる。P型（赤の感受性が低い）・D型（緑の感受性が低い）が大多数を占め、赤と緑・赤と黒・緑と茶などが見分けにくい。「色覚異常」という名称は2005年に日本眼科学会が改訂した用語で、以前は「色盲」「色弱」と呼ばれていた。配色UD対応（例：赤×緑の組み合わせを避け赤×青にする）は、この割合を考慮した設計として特に重要。
  • 路線図のデザイン[外部補足]： 鉄道路線図の代表的なデザインとして、ロンドン地下鉄のハリー・ベック（Harry Beck）が1931年に考案した路線図が世界的に有名。実際の地理的位置より「使いやすさ・わかりやすさ」を優先した抽象化デザインで、情報デザインの歴史的名作とされる。
🎯 補足C: 試験・レポートで重要な観点
  • 情報デザインの定義を正確に言える（「あらゆる情報をわかりやすくまとめて伝えること」）
  • アートとデザインの違い：アートは主観的・デザインは客観的な表現
  • CUI / GUI の違いを説明できる（テキスト入力 vs 視覚的なアイコン操作）
  • インタフェースデザイン → UI → UX の時間軸を説明できる（設計→開発→提供後フィードバック）
  • ユニバーサルデザイン vs バリアフリーの違い：UDは「最初から」、バリアフリーは「後から障壁を除去」
  • UD7原則を列挙できる（特に「単純で直感的」「身体的努力を少なく」「情報が知覚できること」は重要）
  • アクセシビリティ vs ユーザビリティ：前者は幅広いユーザ対象、後者は特定ユーザ対象
  • 情報デザインの関連技術の例を複数挙げられる（Webデザイン・プログラミング・スマートフォン・新聞・図書館）
  • 藤本 貴之氏の定義（「複雑な情報を整理して伝える技術」「構造化して筋道を立てる技術」）を理解しておく
