第1章　「対策を検討する」とはどういうことか？

学習目標：「対策を検討する」とはどのようなことか、また具体的な方法・手順を取ればよいかについて説明できる

対策を検討する意義

← S03:p.6 スライド内容（原文）
  • めざすゴールは同じでも、行き方は「一つ」ではない
  • 仕事や研究のプロセスに無駄が多ければ、完了までの道のりは遠くなる
  • 対策検討の図解は、最良の解決策、最短の解決策を探し出すための武器になる
Audio Supplement — 富士山の例え

2013年6月22日にユネスコ世界文化遺産に登録された富士山には、吉田ルート・御殿場ルート・富士宮ルートなど複数の登り口がある。どこから入っても最終的に行き着くのは同じ頂上。仕事も目的・ゴールが同じであっても、進め方・方法は必ずしも一つではない。プロセスに無駄が多ければ完了まで遠くなるし、適切な経路を取れば早く終えられる。対策検討の図解はその「最適経路」を探すための武器になる。 ← S05

External — 富士山世界遺産登録の確認
富士山のユネスコ世界文化遺産登録は2013年6月22日。正式名称「富士山ー信仰の対象と芸術の源泉」。登録名はカンボジア・プノンペンで開催された第37回世界遺産委員会で決定。三保の松原も含む25件の構成資産が登録対象。スライドの「2013年6月」は正確。 [外部補足・確認済: Wikipedia / nippon.com / 富士山世界遺産センター]

対策検討の図解とは？

← S03:p.7 スライド内容（原文）
状況（あるいは事象、概念、課題など）をいくつかの「要素」に分けて、要素相互の関係を読み解きながら図化
Audio Supplement
「状況」は広義。DX（デジタルトランスフォーメーション）という概念、気候温暖化という事象、日本の人口減少という問題など、あらゆるものを含む。これらに対応・対策を考える際、いったんいくつかの要素に分割して、要素相互の関係を読み解きながら図化する——これが対策検討の図解である。 ← S05

問題や課題を「分割する」とは？

← S03:p.8 スライド内容（原文）
  • 「問題」はしばしば大きなまとまりで捉えているために、解決が困難になっている
  • 問題や課題を対処しやすい大きさに分割することが、「対策の検討」には重要

Audio Supplement — 日英の言語的差異

英語では "problem" を "problems"（複数形）で表現でき、複数の問題として認識できる。しかし日本語の「問題」は単数形のみで複数形の表現がない。日本人が問題を一つのまとまりとして捉えようとするのは、この日本語の特性が影響している可能性もある。

→ これを意識的に分割して小分けの問題にすることで、対策の難易度を下げられる。問題や課題を対処しやすい大きさに分割することが「対策の検討」には重要。 ← S05

「対策検討」と「状況分析」（第3回）との関係

Audio Supplement
状況分析では「対象（青い丸）に似たものを集め、その違い・共通項を抽出して理解を深める」のに対し、対策検討では「対象（青い丸）の内部を要素に分解し、要素間の関係を分析して具体的な対策を検討する」。左の青い丸を拡大して中を見ているのが右の対策検討の図。 ← S05

対策検討図の作成手順（3ステップ）

Step 1「要素を探す」詳細 — 例：納期遅れ問題

  • 問題などを扱いやすい大きさに分けて書き出す（分割されたものを「要素」と呼ぶ）
  • 問題・原因・影響などを具体的にするのが目的
  • 要素を列挙する際は発散思考を用いて、思いつく限りの要素を出す ← S03:p.11

Step 2「軸を探す」詳細

要素と要素を見比べて、共通項を探す。この共通項のことを「軸」と呼ぶ。軸を出すことで、対策を立てやすい「視点」が作り出される。 ← S03:p.12

以下の問いを順番に確認して適切な図タイプを選ぶ。 ← S03:p.22・34・46・59（統合）

Case Study ① ／ ツリー図

機械の購入先を選ぶ

電機メーカーに勤めるYさんは、組立加工の機械を新たに購入する予定。3社から売り込みがあるが、どの会社の製品にすればよいか悩んでいる。
課題：機械の購入先を適切に選ぶ方法を検討したい。 ← S03:p.19

① 要素を探す

「購入先を選ぶ視点」を要素とする。発散思考でできる限り多く出す。 ← S03:p.20

  • 故障したらすぐ来てくれる
  • 安定した会社
  • 機能が充実している
  • 機器の値段が安い
Audio Supplement
芋づる式に発想するのが一手。「故障したらすぐ来てくれる」→「廃棄する時に無料で引き取ってくれる」なども追加できる。また「安定した会社」を「資本金・年間売上が大きい会社」と「歴史の古い会社」の2つに分割する方法もある。要素はできる限り多く出す。 ← S05

Case Study ② ／ 点グラフ

新店舗の立地場所をどこにするか

ドラッグストアを経営する会社の事業戦略室勤務のKさんは、T県に新規チェーン店を出店するため、都市部の駅前商店街という条件の中で3候補地を検討中。いずれも敷地面積・土地価格はほぼ同じだが、最も売上の見込まれる場所に進出したい。 ← S03:p.29

① 要素を探す（売上に影響する立地要因）

ポイントは「売上に影響を与える要因は何か」を考えること。さらに言えば売上に影響すると思われる「立地要因＝立地場所の特徴」を列挙すればよい。 ← S03:p.30

④ 別軸パターン — 同じ要素でも軸次第で示唆が変わる

同じ6要素でも「原因のタイプ」×「重要性」という全く異なる軸で整理できる。 ← S03:p.51-54

新しい軸1：原因のタイプ

  • 評価基準の問題：各人のITスキルを測る基準が明確でない、上司の主観的判断で給与が変わる、自己開発能力を測る評価項目がない、自分で目標設定できない
  • 評価方法の問題：能力の高低が正当に評価されていない、減点主義である

新しい軸2：重要性

  • 重要性が高い：上司の主観で給与が大きく変わる、能力の高低が正当に評価されない → 社員の職業人生に甚大な影響
  • 重要性が低い（相対的に）：その他の4要素
ポイント：同じ要素でも軸の取り方次第で得られる示唆が変わる。納得の得られる示唆が得られない場合は何度でも軸を再設定してマトリクス図を描き直すことが重要。 ← S03:p.54

第4章　対策検討図を描いてみよう（フロー図）

Case Study ⑤ ／ フロー図

赤字の原因を探り、対策を検討する

イベント会社に勤める新人のNさん。新規顧客から受注したイベント開催の業務が赤字になってしまった。いけなかった点はいろいろと思い当たるが、真の原因がよくわからない。 ← S03:p.65

① 要素を探す（失敗したこと）

← S03:p.66-67 スライド内容（原文・6要素）
  • 施設使用料の見積もりの想定が甘く、実際には予定の1.5倍近くになった
  • プロジェクターやホワイトボードなどの備品は追加費用がかかった
  • 懇親会のケータリング費用を少なく見積もっていた
  • 急遽別の講演者に変更
  • 途中で顧客との認識のずれが発覚
  • 講演料は計画上の倍近くに膨れ上がった
→ 要素間の一部には因果関係がありそう　→ フロー図が適切
Audio Supplement
フロー図では要素を列挙する際、企画立案→実施→後処理までの時間の流れを時系列で想起することで、因果関係のある要素が出てきやすい。 ← S05

② 軸を探す（フロー図の場合）

フロー図では軸を設定しない。代わりに関係のある要素を線でつないでいく。 ← S03:p.68

③ 図にして対策を検討する（フロー図・完成版）

「なぜ？」を繰り返して根本原因（真の原因）まで遡る。 ← S03:p.69-71

結論：費用がかさんだ直接の原因は①講演料倍増、②施設・備品費超過、③ケータリング費過小見積もりの3点。しかしその元をたどると、事前の情報収集が不十分だったことに行き当たる。今後は事前の情報収集を綿密に行う必要がある。 ← S03:p.71
Audio Supplement — フロー図の特性
フロー図は「起きた事象から原因を遡っていき、これ以上遡れない原因（＝真の原因）は何かを発見する」のに適した手法。「なぜ？」を繰り返すことで根本原因まで辿り着ける。問題が噴出していて真の原因が分からない時に特に有効。 ← S05

📚 補足B：知っておくべき背景知識

MECE（ミーシー）と構造化思考

[外部補足] MECE（Mutually Exclusive, Collectively Exhaustive）は「漏れなく・ダブりなく」整理する考え方。マッキンゼー出身のバーバラ・ミント（「ピラミッド原則」著者）が体系化したとされるが、実際の命名・普及はマッキンゼー社内での実践知から。ツリー図はMECEを視覚的に確認できる代表的なツール。 [外部補足・確認済: Barbara Minto "The Pyramid Principle"]

なぜなぜ分析（5 Whys）との関係

[外部補足] フロー図で「なぜ？」を繰り返して真の原因を探る手法は、トヨタ生産方式の「なぜなぜ分析（5 Whys）」に通じる。大野耐一（元トヨタ副社長）が体系化。「問題が起きたとき、なぜを5回繰り返すと根本原因にたどり着ける」という手法。本授業のフロー図はこの思想と一致する。 [外部補足・確認済: 大野耐一『トヨタ生産方式』]

富士山の世界遺産登録（スライド内の例）

[外部補足・確認済: 文化庁・ユネスコ] 富士山のユネスコ世界文化遺産登録は2013年6月22日（第37回世界遺産委員会、カンボジア・プノンペン）。正式名称「富士山ー信仰の対象と芸術の源泉」。25件の構成資産が登録対象。スライドの「2013年6月」表記は正確。

人・もの・お金（3Mフレームワーク）

[外部補足] 経営資源を「ヒト・モノ・カネ（＋情報）」に分類するフレームワーク。ツリー図の軸設定でよく使われる。「3Mフレームワーク」とも呼ばれる。情報を加えて「ヒト・モノ・カネ・情報」の4軸で整理することもある。

🎯 補足C：試験・レポートで重要な観点

  • 図タイプ選択フローチャートを正確に再現できる：4問の判定（因果関係→重なり→3項目以上→定量化）の順序と対応する図を正確に書けること
  • フロー図の例外性を説明できる：「フロー図のみ軸を設定しない」理由＝要素が因果関係・順序関係にある場合、軸ではなく「なぜ」でつないでいく
  • 各ケーススタディで「何を要素にしたか」を答えられる：
    ケース何を要素にしたか使用図
    機械購入先購入先を選ぶ視点ツリー図
    新店舗立地売上に影響を与える立地要因点グラフ
    人事評価制度現行の人事制度の問題点マトリクス図
    DX推進策DXの打ち手（施策）そのものベン図
    赤字原因失敗したことフロー図
  • 「分かる」は「分ける」から始まるという第1章の核心を説明できる
  • 状況分析×対策検討 = 25通りの組み合わせがある意義を説明できる
  • ツリー図の「掛け算型」vs「足し算型」の違いを事例で説明できる
  • マトリクス図で軸を変えると示唆が変わる理由と実例（人事評価ケースの2パターン）を説明できる

🔗 補足D：関連概念・次回への接続

[外部補足] 以下は授業で直接触れていないが関連する知識。

ロジックツリー（Logic Tree）との関係

[外部補足] 本授業のツリー図は、コンサルティング業界でよく使われる「ロジックツリー」に相当する。① イシューツリー（問題を要素に分解）、② ハウツーツリー（解決策を体系化）、③ ホワイツリー（原因を追跡）の3種類があり、本授業の用途と対応する。

散布図（Scatter Plot）との関係

[外部補足] 本授業の「点グラフ（2軸で各候補地を点として描く）」はデータサイエンスの散布図に相当する。統計的には「2変数の関係を可視化」し、通常は相関関係の強弱を確認するために使う。本授業では定量化できる2軸を設定して比較・意思決定に活用する点で実践的な応用例となっている。

第5回以降との接続

[外部補足] 第3・4回が「基礎の図解手法（状況分析・対策検討）」であるのに対し、第5回以降は同じ5種の図を特定の目的（発想力・問題解決力・意思決定力・交渉力）に応用する高度な活用方法を学ぶ。

現象学・言語学の知見（状況分析の理論的背景）

[外部補足] S03:p.75では「いかなる言葉も実体的な何かを名状しているのではなく、他の言葉との差異・関係を通して初めて意味を与えられる」という記述がある。これはフェルディナン・ド・ソシュール（言語学者）の差異の概念に基づく。知りたいことを深く知るには、それと似たものとの関係・違いを分析することが有効だというロジカルシンキングの理論的根拠。
