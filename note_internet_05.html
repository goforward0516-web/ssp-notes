🎙 音声補足

インターネット上のWebサーバにアクセスするには、そのサーバのIPアドレスやURIを事前に知っている必要がある。しかし、すべてのURIを覚えておくことは現実的ではないため、検索エンジンが必要となる。「知らない情報を得るためにアクセスしようとしているのに、事前にURIを知っている必要があるのは本末転倒」というのが、検索エンジン誕生の動機である。

検索エンジンとは
📖 定義

検索エンジン（Search Engine）: インターネット上で、キーワード等から該当するWebサイトを検索するシステムやサービス。検索サービスとも呼ぶ。広義には、インターネットに限定されず情報を検索するシステム全般を指す。

← S03:p.6

検索エンジンには、ジャンル問わず様々なWebサイトを検索する汎用検索エンジン（Google、Bing、Yahoo!等）と、特定ジャンルに絞った専門検索エンジン（Google Scholar、CiNii Research等）がある。

WWW普及以前の検索システム
Archie（アーキー/アーチー）
📖 定義

Archie: 世界初の検索エンジンといわれる。インターネット上のFTPサーバにあるファイルを検索するサービス（1990年9月10日公開）。FTPサーバ上のファイルを巡回してファイルの索引を作成し、ファイル名を利用した検索のみが可能。

← S03:p.10
🌐 外部補足

Archieの開発者: Alan Emtage（カナダ・モントリオールのMcGill大学大学院生）、Bill Heelan、Peter Deutschの3名。名前は「archive（アーカイブ）」から「v」を抜いたもの。ピーク時（1993年頃）は1日約5万件のクエリを処理し、カナダへのインターネットトラフィックの半分を占めるほど普及した。

← Wikipedia / McGill University
🎙 音声補足

Archieは「どのサーバにどのファイルがあるか」を調べるサービスである。ファイル名での検索しかできなかった点が大きな制限。スライドでは開発者名・大学名の明示はないが、McGill大学で開発されたことは広く知られている。

← S05
Gopher（ゴーファー）
📖 定義

Gopher: 1991年にミネソタ大学で開発されたIPネットワーク（インターネット）上のテキストベース情報検索システム/プロトコル。インターネット上に形成されたGopherネットワークで分散されたドキュメントを検索可能。ファイルに索引をつけて階層化し、メニューに従って検索できた。

← S03:p.11
🎙 音声補足（Gopherの衰退）

GopherはWWWの対抗馬として位置づけられ、1990年代始めはWWWとGopherが両立していた。しかし衰退した理由は2つある: ①1993年にミネソタ大学がGopherサーバの実装に対してライセンス料を請求すると発表、②GopherクライアントにHuman実装されていた機能がWebブラウザ（Mosaic）にすぐに実装された。その結果、徐々にWWWの利用者が増え、Gopherは使われなくなった。

← S03:p.11, S05

Bingは2009年にリブランドされたMicrosoftのロボット型検索エンジン。名称変遷: MSN Search（1998）→ Windows Live Search（2006）→ Live Search（2007）→ Bing（2009）。2009年から米国Yahoo!の検索エンジンとしても採用。

← S03:p.16
⚠️ 注意点

Microsoft Copilot（旧Bing AIチャット）: 2023年に公開された自然言語処理技術を活用した検索型AIサービス。質問形式の文章で検索・要約・コード生成が可能。ただし誤った内容もあたかも正しいように尤もらしい回答がされる場合がある。回答を鵜呑みにせず、提示されたURIを参照して確認することが必要。

検索エンジンの基本的な構成
🎙 音声補足（動作フロー）

ユーザーが検索ボックスに検索クエリを入力して実行すると、そのクエリはクエリパーサによって解析される。それをもとにデータベースから該当するインデックスを検索し、その結果に対して順位付けを行い、ユーザーに見やすく提示する。クローラとインデクサは常時裏側で動作しデータベースを最新に保ち、クエリ解析以降は検索ごとに動作する。

← S05
Webクローラとインデクサ
📖 定義

Webクローラ（Web Crawler）: 世界中のWebページを自動的に巡回して内容を収集するプログラム。「クローラ」「スパイダー」「ボット」とも呼ばれる。

クローラの主な処理:

  1. 世界中のWebページを自動的に巡回して内容を収集
  2. 収集したWebページの情報をデータベースに保存（未収録・更新済みのものも保存）
  3. Webページ内のリンク（URI）から他のWebページをたどって繰り返す
← S03:p.23
📖 定義

インデクサ（Indexer）: クローラが収集したWebページを効率的に検索できるように分析・整理する処理（インデクシング）を行うプログラム。Webページを全文解析し、コンテンツ種別判定・キーワード抽出・タグ、ページ公開日・更新日等のメタ情報の取得などを行う。処理結果はデータベースに保存される。

← S03:p.23
PageRankアルゴリズム
📖 定義（核心原則）

PageRank: Webページの価値を定量化し、順位付けを行うアルゴリズム。基本的な考え方:「品質の高いWebページからリンクされているWebページは品質が高い」

← S03:p.24
🌐 外部補足（出典）

PageRankの原論文: Brin, S. & Page, L. 「The Anatomy of a Large-Scale Hypertextual Web Search Engine」(1998, Proceedings of WWW7)。名前は開発者の「Larry Page」とWebの「page（ページ）」の両方に由来する。特許はStanford大学に帰属し、GoogleはStanfordからライセンスを受けた（大学は交換として180万株を取得、2005年に売却し約3億3600万ドルを得た）。

← Wikipedia/PageRank

PageRankの考え方は3要素に分解できる:

要素内容考え方
① 被リンク数リンクされている数が多いほど価値が高い多くのWebページから認められているとみなす
② リンクの重要度品質の高いWebページからのリンクはより価値が高い誰からリンクされているかが重要
③ リンク元のリンク数リンクが厳選されたWebページからのリンクが価値高いリンク集のようなページからのリンクは価値低い
← S03:p.24
🎙 音声補足（計算式の解説）

PageRankの計算式（簡略版）: ページXの価値 = Σ（リンク元ページiの価値 ÷ リンク元ページiのリンク数）。式を覚える必要はない。式中の分子が要素②（リンクの重要度＝リンク元ページの価値）を、分母が要素③（リンク元のリンク数）を表し、これらの和が要素①（被リンク数）を表現している。

← S05, S03:p.25
🎙 音声補足（図の解説）

ページDはどこからもリンクされないため価値は低い。ページAはB・Dの2つからリンクされるため価値が高くなる。ページCは価値の高いページAからリンクされているため、この中で最も価値が高いと判断される。現在の検索エンジンではPageRankに加えて様々な評価指標が加味されて順位付けされる点に注意。

← S05, S03:p.26
SEOとブラックハットSEO
📖 定義

SEO（Search Engine Optimization）: Webサイトの運営者が、WebページをGoogleなどの検索結果の上位に表示されやすくするための工夫や対策。

← S03:p.27
SEOの王道 vs 邪道

王道: 検索エンジンの目的は「ユーザーの検索意図に合致した品質の高いWebページを提示すること（ユーザー体験の向上）」。よって、質の高いWebページを作成することが近道。

邪道（ブラックハットSEO）: 検索エンジンの仕組みを逆手にとった不適切なSEO。継続的に対策されている。

← S03:p.27
🎙 音声補足（なぜSEO対策が行われるか）

検索結果は上から順に見て気になったものをクリックするため、上位にあるものほど選択されやすい（アクセス数が多くなる）。この特性は古くから経験的に知られている。そのためWebサイト運営者が検索結果の上位に表示されるよう工夫（SEO）をするようになった。

← S05
REP（Robot Exclusion Protocol）
📖 定義

REP（Robot Exclusion Protocol）: 検索対象から一部のWebコンテンツを除くためのプロトコル。robots.txtというテキストファイルに決められた書式で記入することで設定できる。IETFによりインターネット標準として定められている（RFC9309）。

← S03:p.28

Webページを検索エンジンの検索対象外にしたい場合の例:

  • Webサイトの管理者用ページ
  • 検索エンジンから直接アクセスして欲しくないページ（別ページへの誘導を意図しているページ等）
  • 検索エンジンの評価に悪影響を与えそうなページ（SEO対策）
← S03:p.28
robots.txt の記述例
User-agent: *
Disallow: /

User-agent: Googlebot
Allow: /

意味: Googleのクローラのみ許可し、それ以外のクローラは不許可

← S03:p.28
⚠️ 注意点

robots.txtの指定に拘束力はなく、すべてのクローラが従うとは限らない。ただし、主要な検索エンジンのクローラはこれに従うと考えて問題ない。

← S03:p.28

AI検索エンジン
📖 定義

AI検索エンジン: 生成AIを活用した検索エンジン。AIがユーザーの検索意図のみならず、その意図に沿って複数のWebサイトをまとめて検索し、結果を整理・加工して提示する。これまで人が検索結果から複数のWebサイトを回って情報を集めていたことをAIが代行する。

← S03:p.34

⚠️ 注意点

📚 補足B: 知っておくべき背景知識

[外部補足] ← 授業で言及されたが詳細はソースにない / 理解を深めるための追加情報

  • Archieの開発者・名前の由来: Alan Emtage（McGill大学大学院生）、Bill Heelan、Peter Deutschが1990年に開発。名前は「archive（アーカイブ）」から「v」を抜いたもの（コミックの「Archie」とは無関係とEmtage本人が述べている）。ピーク時の1993年頃には1日約5万クエリを処理した。
  • FTPとは: File Transfer Protocol。インターネット上でファイルを転送するプロトコル。Archieの検索対象はこのFTPサーバ上のファイルだった。
  • Mosaicとは: 1993年にリリースされたWebブラウザ。画像とテキストを同時表示できた最初の広く普及したブラウザ（NCSA開発）。GopherクライアントがもっていたメニューNavigation機能がMosaicに実装されたことが、Gopherの衰退の一因となった。
  • BackRubとPageRankの経緯: Larry PageとSergey Brinが1996年にStanford大学のPhD研究として開発。「BackRub」という名称はバックリンク（被リンク）の分析に由来する。1998年に「The Anatomy of a Large-Scale Hypertextual Web Search Engine」論文として発表。PageRankの名前は開発者Larry Pageの名前とwebの「page」の両方に由来する（U.S. Patent 6,285,999, 2001年取得・2019年失効）。
  • BERT（Bidirectional Encoder Representations from Transformers）: Jacob Devlinら（Google）が2018年10月にarXivで発表（論文: Devlin et al., NAACL 2019）。双方向Transformerエンコーダを利用し、文の左右両方の文脈を同時に考慮することで高い言語理解精度を実現。11のNLPタスクで当時のSOTA（最高性能）を更新。2019年にGoogle検索に本格適用。
  • Creative Commonsライセンスの6種類: 4要素（BY=著作権者表示、SA=同一ライセンスで共有、NC=非営利のみ、ND=改変不可）の組み合わせで6種類が構成される。許容度の高い順: CC BY → CC BY-SA → CC BY-NC → CC BY-NC-SA → CC BY-ND → CC BY-NC-ND（最も制限的）。CC0はすべての権利を放棄するパブリックドメイン相当のツール（ライセンスではない）。
  • CiNii Researchとは: 国立情報学研究所（NII）が運営する日本の学術情報データベース（https://cir.nii.ac.jp/）。旧CiNii Articles・CiNii Books・CiNii Dissertationsを統合した次世代システム（2022年4月リニューアル）。日本語論文を中心に検索できる。
  • Internet Live Statsとは: リアルタイムのインターネット統計情報を提供するサービス（internetlivestats.com）。授業のWebサイト数グラフの出典。ホスト名ベースでのカウント（非アクティブサイトを除く）。最後のデータポイントは2018年頃（その後データ更新停止）。
🎯 補足C: 試験・レポートで重要な観点
  • ディレクトリ型 vs ロボット型の違いを「収集方法・利点・欠点」の観点で説明できるようにしておく（特に「なぜロボット型が主流になったか」）
  • PageRankの3要素（被リンク数・リンクの重要度・リンク元のリンク数）を説明できること。「品質の高いWebページからリンクされているページは品質が高い」という核心原則を覚える
  • SEO の王道と邪道（ブラックハットSEO）の違い。検索エンジンのアップデート（フロリダ・パンダ・ペンギン）の対象となった問題を理解しておく
  • robots.txt（REP）の役割と「拘束力がない」という重要な特徴
  • AI検索エンジンの注意点:「尤もらしい回答 ≠ 正しい情報」。根拠URLを確認することの重要性
  • 検索演算子の実践的な使い方（AND/OR/NOT/filetype:/site:）。具体的な検索クエリの例を書ける
  • Creative Commonsライセンスの概念と、画像利用時のライセンス確認の必要性
  • 歴史上の主要な年号: Archie(1990)、Gopher(1991)、Yahoo!(1994)、BackRub(1996)、Google創業(1998)、Bing(2009)、BERT導入(2019)
🔗 補足D: 関連概念・次回への接続

[外部補足] ← 授業では直接触れていない関連知識

  • PageRankの影響と現在: PageRankは学術論文の引用分析（Eugene Garfield, 1950s）に着想を得ている。PageRankは現在もGoogleのランキング信号の一つだが、2019年にすべての関連特許が失効しパブリックドメインに。現在のGoogleは200以上のランキング要因を使用しており、PageRankは数百ある要素の1つに過ぎない。
  • SEOとコンテンツマーケティング: SEOの王道「質の高いWebページを作る」という考え方は、現代のコンテンツマーケティングの基本と一致する。E-E-A-T（Experience, Expertise, Authoritativeness, Trustworthiness）はGoogleが品質評価に使う指標。
  • 検索エンジンとプライバシー: パーソナライズドサーチはフィルターバブル（自分に都合の良い情報だけが表示される状態）を生む可能性がある。プライバシー重視の検索エンジンとして DuckDuckGo がある。
  • 生成AIとハルシネーション: AI検索エンジンの「尤もらしい回答 ≠ 正しい情報」という問題は「ハルシネーション（幻覚）」と呼ばれる現象。LLMが学習データに基づいて確率的に最もらしいテキストを生成するが、事実確認を行わないために発生する。
