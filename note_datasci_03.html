第1章 度数分布表

CH1 度数分布表とは

事例: ゲームアプリの満足度調査（第2回の続き） ← S03:p.8

第2回で設計した調査では、層別抽出法で各年代の割合を母集団から算出し、合計50人を対象にアンケートを実施した。

S05 比例配分法: 特に前回は説明しなかったが、母集団の各層の比率に応じて調査対象数を決める方法を層別抽出法の中でも特に比例配分法と呼ぶ。← S05:L010
外部補足 層別抽出法とは、母集団をあらかじめいくつかの層（グループ）に分けてから各層から標本を抽出する方法。代表性の高いサンプルが得られる利点がある。

アンケート結果として50人分の年齢データが得られた。生データの状態（羅列）では概要が把握しにくく、これを要約して見やすくするのが度数分布表の役割。 ← S03:p.9

生データ（50人分） ← S03:p.9
18, 32, 24, 21, 19, 45, 22, 21, 20, 22,
33, 27, 35, 23, 51, 42, 20, 19, 24, 35,
27, 26, 18, 29, 28, 33, 18, 21, 23, 31,
16, 32, 46, 19, 31, 32, 50, 41, 30, 42,
21, 31, 50, 51, 35, 37, 15, 36, 33, 51
CH1 度数分布表の7要素

① 階級と度数

定義 階級: データが取りうる値に対し、一定の幅ごとに区切ったグループ ← S03:p.10
定義 度数: それぞれの階級に当てはまるデータの個数 ← S03:p.10
S05 「10歳以上20歳未満」という表記は「××以上、〇〇未満」という書き方が正式。表ではよく「10歳〜20歳」と略記される。← S05:L014

② 階級幅と階級値

定義 階級幅: 1つの階級に含まれるデータの範囲 ← S03:p.11
定義 階級値: 階級を代表する値。一般に階級の中央の値（上限値 + 下限値）÷ 2 ← S03:p.11
例 10歳〜20歳の場合: 階級幅 = 20 - 10 = 10歳、階級値 = (20 + 10) ÷ 2 = 15歳 ← S05:L018-L019

③ 累積度数

定義 累積度数: 度数を最小の階級から順に足し合わせていった値 ← S03:p.13
S05 「下の階級」とは表の下の行ではなく、値が最も小さい階級のこと。← S05:L021

④ 相対度数

定義 相対度数: 各階級に属する度数を、度数の合計（全データ数）で割った割合 ← S03:p.15
相対度数 = 度数 ÷ 全データ数　例：17 ÷ 50 = 0.34

⑤ 累積相対度数

定義 累積相対度数: 最小の階級から特定の階級までの相対度数の合計 ← S03:p.16

完成した度数分布表

度数分布表には「階級値」「度数」「累積度数」「相対度数」「累積相対度数」を記載するのが一般的 ← S03:p.18

重要 累積相対度数が0.80 → アンケート回答者の8割は40歳未満であることが一目でわかる。← S03:p.17
CH1 ヒストグラム
定義 ヒストグラム: 度数を棒の高さで表現することで、視覚的にデータを説明できる可視化方法の一つ ← S03:p.19
S05 ビン（bin）: ヒストグラムの各棒のこと。Excelなどでヒストグラムを作成するとビンの数を指定する表示が出てくる。ぜひ覚えておくこと。← S05:L034

縦軸を度数から相対度数に変えてもグラフの形は同じ。相対度数で表現する方が一瞬でわかるグラフになるのでおすすめ。 ← S03:p.20, S05:L036

ヒストグラムの特徴

← S03:p.21, S05:L038-L039

  • 階級幅が一定でない場合も表現できる（例: 100万円刻みと200万円刻みが混在する家計調査のヒストグラム）
  • 上限値が定まらない場合も表現できる（例: 「4000万円以上」という上限なしの階級）

ヒストグラムの注意点：離散値 vs 連続値

データの種類ビン間の空白例
離散値空白を入れる人数、5段階評価
連続値空白を入れないお金、速度、時間
S05 同じ「1, 2, 3, 4, 5」というデータでも、作成者が離散値として扱うか連続値として扱うかは判断による部分もある。自分でヒストグラムを作るときは、扱うデータの性質を考えてビン間の空白の有無を判断すること。← S05:L041

第2章 データの代表値

ヒストグラムは視覚的に優れているが、人の視覚能力に依存するため人によって捉え方が異なる可能性がある。そこでデータを数字で表現することで、さらに的確にデータの概要を伝えることができる。 ← S03:p.27, S05:L043

代表値別名特徴
算術平均相加平均重心の意味。外れ値に弱い
メジアン中位数・中央値外れ値の影響を受けにくい
モード最頻値度数が最多の階級値。外れ値を考慮しない
外部補足 このほかにも幾何平均（掛け算に基づく平均）、調和平均（逆数の平均の逆数）、加重平均（重みを付けた平均）などがある。本授業では算術平均のみ扱う。
CH2 算術平均（相加平均）
定義 算術平均（相加平均）: N個のデータの総和をNで割った値 ← S03:p.29
x̄ = (x₁ + x₂ + ⋯ + xₙ) ÷ N
S05 「相加（そうか）」の「加」は「加えていく」の意味。次々とデータを加えていくことで求める平均であることを表す。「相」は「相続」の「相」と同じく「次々と」の意味。← S05:L046

生データからの計算

x̄ = (18 + 32 + 24 + ⋯ + 33 + 51) ÷ 50 = 30.12（歳）

← S03:p.30

度数分布表からの概算

各階級の「階級値 × 度数」を全階級分合計し、データ数で割る。 ← S03:p.31

x̄ = (15×8 + 25×17 + 35×15 + 45×5 + 55×5) ÷ 50 = 31.4（歳）

→ 生データの平均30.12歳に近い値が得られる。度数分布表のみからでも平均の概算が可能。 ← S05:L050

算術平均の意味：重心

S05 5個のデータ（15, 20, 24, 51, 55）の平均 = 33。数直線上に重さ1の重りを各データの位置に載せると、33の位置で左右のバランスが釣り合う（重心）。算術平均は重心の意味を持つ。← S05:L051

平均の問題点

問題の種類説明
① 複数データを組み合わせた場合 年齢とプレイ時間など複数の変数を組み合わせると、全ての変数が同時に平均になる対象は存在しない場合がある（平均値の周辺にデータが存在しない）。← S03:p.34
② 外れ値が存在する場合 2021年所得金額の例: 視覚的には100〜400万円の人が多いが、平均所得金額は564.3万円（中央値440万円）。2000万円以上の外れ値が平均を押し上げている。平均は外れ値に弱い。← S03:p.35
CH2 メジアン（中位数・中央値）
定義 メジアン: データを昇順（降順）に並び替えたときに中央にくる値 ← S03:p.36
データ数計算方法
奇数個データの中央の値（例: 3個なら2番目）
偶数個データの中央の2値の平均値（例: 50個なら25番目と26番目の平均）

生データからの計算

データ数50個（偶数）→ 25番目（29）と26番目（30）の平均 ← S03:p.37, S05:L058

メジアン = (29 + 30) ÷ 2 = 29.5（歳）
重要 所得金額の例: 平均564.3万円 vs 中央値440万円。ボリュームゾーン（100〜400万円）に近い中央値の方が実態を反映している。メジアンは外れ値の影響を受けにくい。← S03:p.38, S05:L060

度数分布表からの概算

データ数が偶数の場合、25番目と26番目が属する階級の階級値の平均を求める。 ← S03:p.39-40

  • 25番目 → 累積度数25の「20歳〜30歳」に属する → 階級値 25
  • 26番目 → 「30歳〜40歳」に属する → 階級値 35
メジアン ≈ (25 + 35) ÷ 2 = 30（歳）　※生データのメジアン29.5歳に近い

度数分布表のみからでもメジアンの概算が可能。

外部補足 なぜ30歳（概算）と29.5歳（生データ）でズレるのか:
度数分布表からのメジアン算出では、データが属する階級値（その階級の代表値）を使う。実際には25番目のデータは29歳、26番目は30歳だが、度数分布表上では両者とも「20〜30歳」「30〜40歳」の階級値（25と35）で近似するため、結果として概算値になる。データを階級にまとめた時点で個々の値の情報が失われるため、この誤差は避けられない。生データがあれば生データから計算するのが正確。
CH2 モード（最頻値）
定義 モード: 度数が最も多い階級の階級値（最頻値） ← S03:p.41

外れ値を考慮しないため、外れ値の影響がない。

例 度数が最も多いのは「20歳〜30歳」（度数17） → 階級値 25 がモード。← S05:L065

モードが不適切な場合

状況問題
全階級の度数が均一モードが存在しない
ヒストグラムが多峰（山が複数）1つのモードでデータ全体を表現できない
CH2 算術平均・メジアン・モードの関係

3つの値の大小関係からヒストグラムの概形がわかる。 ← S03:p.43, S05:L067

外部補足 右歪・左歪の用語整理:
「右裾が長い（右に外れ値がある）」分布を右歪（正の歪み）分布という。山が左に偏り、右側に長い尾を持つ形。このとき モード < メジアン < 平均 の順になる（平均が外れ値に引っ張られる）。所得・資産分布などに多い。
逆に「左裾が長い」分布を左歪（負の歪み）分布という。山が右に偏り、このとき 平均 < メジアン < モード の順になる。

第3章 データのばらつき

例 AさんとBさんの5科目テスト（平均点は共に65点）: ← S03:p.48
科目1科目2科目3科目4科目5平均
Aさん100点90点25点10点100点65点
Bさん65点75点55点50点80点65点
重要 同じ平均でもばらつきを見ることで、AさんとBさんの勉強の戦略の違いが明らかになる。ばらつきは平均だけでは見えない情報を与えてくれる。← S05:L071
CH3 偏差と平均偏差
定義 偏差: 個々のデータから平均を引いた値（平均からの距離） ← S03:p.49
データ計算偏差
1515 - 33-18
2020 - 33-13
2424 - 33-9
5151 - 33+18
5555 - 33+22
合計0
重要 偏差を全て足すと必ず0になる → このままではばらつきの指標として使えない。← S03:p.50, S05:L074
定義 平均偏差: 偏差の絶対値の平均 ← S03:p.50-51
平均偏差 = (|-18| + |-13| + |-9| + |18| + |22|) ÷ 5 = 80 ÷ 5 = 16
S05 平均偏差の問題点: 絶対値の計算はプログラムで実装する場合に条件分岐（if文）が必要となり、処理が複雑になる。どんな値に対しても同じ処理で正の数にできる方法が必要。← S05:L076
CH3 分散（σ²）
定義 分散（σ²）: 偏差の2乗の平均 ← S03:p.52
S05 2乗すれば負の数も正になるため、絶対値と同様に条件分岐なしで処理できる。これが昔の偉い人が考えた解決方法。← S05:L077
σ² = [(-18)² + (-13)² + (-9)² + (18)² + (22)²] ÷ 5 = 1382 ÷ 5 = 276.4（歳²）
注意 2乗したため単位が「歳²」になり、平均（単位: 歳）と直接比較できない。→ 標準偏差で解決。← S05:L079
CH3 標準偏差（σ）
定義 標準偏差（σ）: 分散の平方根。平均と同じ単位でばらつきを表す ← S03:p.54
σ = √σ² = √276.4 ≈ 16.63（歳）
重要 「平均 ± 標準偏差」 でデータ全体像を表現: 33 ± 16.63（歳）
「このデータは33歳を中心に、±16.63歳の範囲にほとんどのデータが入っている」
※「平均 ± 分散」（33 ± 276.4）とは書かない（単位が異なるため）。← S03:p.54

計算例: Aさんと Bさん

σ_A = √[(100-65)² + (90-65)² + (25-65)² + (10-65)² + (100-65)²] ÷ 5 = 39.24（点）
σ_B = 11.40（点）

→ σ_A（39.24）＞ σ_B（11.40） → AさんのほうがBさんより点数のばらつきが大きい。 ← S03:p.55, S05:L083

度数分布表からの概算（平均 = 31.4）

σ² = [(15-31.4)²×8 + (25-31.4)²×17 + (35-31.4)²×15 + (45-31.4)²×5 + (55-31.4)²×5] ÷ 50 = 135.04（歳²）
σ = √135.04 ≈ 11.62（歳）

← S03:p.56-57

CH3 （補足）VAR.P と VAR.S の違い

← S03:p.58, S05:L087-L089

関数割り算使用場面式
VAR.PN で割る全数調査（悉皆調査）σ² = Σ(xᵢ - x̄)² ÷ N
VAR.SN-1 で割るサンプル調査σ² = Σ(xᵢ - x̄)² ÷ (N-1)
S05 なぜN-1を使うか: サンプル数が1のとき、データのばらつきは定義できない（分散は存在しない）。しかしVAR.P（Nで割る）で計算すると、存在しない分散の値が出てしまう。それを防ぐためにN-1で割る。
サンプル数が大きくなるほど、NとN-1の差は無視できるほど小さくなる（例: 100000 vs 99999）。← S05:L088-L089

📚 補足B: 知っておくべき背景知識

偏差の合計が0になる理由: 平均は「重心」であるため、平均より大きい値の偏差の合計と小さい値の偏差の合計が必ず釣り合い、合計すると0になる。これは算術平均の定義から数学的に必ず成立する性質。

度数分布表からの概算値にズレが生じる理由: 度数分布表を使った平均・メジアン・分散の計算では、各データを「階級値（階級の中央値）」で代表させる。実際の個々のデータの値は失われているため、どうしても近似誤差が生じる。データ数が多いほど誤差は小さくなる傾向がある。生データが手元にある場合は生データから直接計算するのが正確。

なぜN-1で割るか（ベッセルの補正）: サンプルから計算した分散は、真の母分散を過小評価する傾向がある。N-1で割ることでこの偏りを補正する（不偏分散）。サンプル数が増えるほどNとN-1の差は相対的に小さくなり、両者の結果は一致に近づく。

ヒストグラムと棒グラフの違い: 棒グラフはカテゴリ変数の比較に使い棒の間に空白がある。ヒストグラムは連続変数の分布を表し、連続値の場合は棒の間に空白がない。見た目が似ているが用途が異なる。

平均・メジアン・モードの使い分け指針: 外れ値がない対称な分布では3つの値はほぼ一致し、どれを使っても良い。外れ値がある場合や分布が歪んでいる場合はメジアンの方が実態を反映しやすい（所得・資産データなどに多い）。多峰分布ではモードは複数存在しうるため代表値として不適切。
