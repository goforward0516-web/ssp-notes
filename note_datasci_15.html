DXとは（定義）

【定義】DX（Digital Transformation）

企業がビジネス環境の激しい変化に対応し、データとデジタル技術を活用して、顧客や社会のニーズを基に、製品やサービス、ビジネスモデルを変革するとともに、業務そのものや、組織、プロセス、企業文化・風土を変革し、競争上の優位性を確立すること。

出典: 経済産業省「デジタルガバナンス・コード2.0」p.1脚注 ← S03:p.9脚注[1]

💡
DXの主眼は組織文化の変化（＝経営の問題）。データやデジタル技術はその実現手段として利活用されるものであり、手段そのものが目的ではない。 ← S03:p.9

DXという表記について: "Transformation"の "Trans" を "X" と表記し、頭文字をとって "DX"。 ← S05

DXに関係する取り組みの4フェーズ

フェーズ名称概要
①デジタイゼーション（Digitization）アナログ・物理データのデジタル化
例: 紙の書類を電子データ化する（領収書の電子化）
②デジタライゼーション（Digitalization）業務の効率化による生産性の向上
データやAIを活用して業務プロセスを改善
③DX（Digital Transformation）新規製品・サービスの創出
④顧客起点の価値創出によるビジネスモデルの根本的な変革
⚠ 注意
「デジタイゼーション」と「デジタライゼーション」は1文字違いだが意味が異なる。混同しないよう注意が必要。 ← S03:p.10, S05

出典[2]: 情報処理推進機構「DX白書2023」p.14 (2023) ← S03:p.10脚注[2]

【定義】RPA（Robotic Process Automation）
ロボティック・プロセス・オートメーション。コンピューター上で行われるルーティン業務（データ入力・転記・チェックなど）をソフトウェアロボットが自動的に実行する技術。デジタライゼーション（業務の効率化）の代表的な手段の一つ。部門単位でも導入可能。 ← S03:p.11, [外部補足]
音声補足（DX取組企業の増加）
日本のDX取り組み企業割合は急増。2021年度調査: 55.8% → 2022年: 69.3%（たった1年間で約14%増加）。

広義のDXと変革規模

音声補足
広義のDX = デジタイゼーション + デジタライゼーション + DX（狭義）を包含したもの。デジタイゼーション・デジタライゼーションは部門単位でも実行可能だが、DXは企業全体・関連企業・社会構造全体に影響を及ぼす。 ← S03:p.11, S05

DXの具体例（狭義のDX）

① フードデリバリーサービス

従来は店舗ごとに配達員を雇用しフードデリバリーを行っていた。フードデリバリーサービスの登場により、各店舗が個別に人を雇う負担と配達機材の確保が不要になり、専門業者に集約することでお客への利便性向上と店舗の運営効率が実現。社会構造自体が大きく変化した（＝DX）。 ← S03:p.12, S05

  • 注文管理: 全てオンライン上で処理

← S03:p.12, S05

② サブスク（サブスクリプション）サービス

従来: CDを買い切り（音楽1曲・1アルバムを購入）。現在: 月額定額を支払うだけで提供される音楽を何度でも聴き放題。買い切りモデルからサブスクモデルへのビジネスモデルの根本変革（＝DX）。 ← S03:p.12, S05

データ・AIの利活用内容
  • 商品（楽曲）の管理・在庫管理
  • レコメンド: ユーザーの好みに合った楽曲・商品を推薦
← S03:p.12, S05

③ 介護施設における見守りシステムの導入

従来: 介護スタッフが一人ひとりの入居者に目を配り、食事・体調などを手動で確認・記録する必要があった。センサーやカメラで入居者の状態を自動集約することで、スタッフ間の情報共有が円滑になり業務プロセスが変革（＝DX）。 ← S03:p.13, S05

  • 介護記録の電子化
空いた時間をスタッフ間・入居者とのコミュニケーションに充てることで、質の高い介護を提供できる。 ← S03:p.13, S05

2第2章 データ・AIと倫理

📌 学習目標: データ・AIの利活用時に倫理的判断が必要な理由を説明できる ← S03:p.19

データ・AIの利活用とトラブル例

倫理の話に入る前に、実際のトラブル例を通して「なぜ倫理が必要か」を理解する。← S05

例1: 精度が高すぎるレコメンドのリスク

シナリオ
過去の購入歴からレコメンドを作成し、特定商品を紹介するダイレクトメールを郵送。← S03:p.20
利点問題
顧客満足度向上、売上アップレコメンド内容・方法によっては、嗜好・病気等が公になる可能性（プライバシー侵害）
音声補足（具体的なトラブル事例）
あるベビー用品に関するダイレクトメールを未成年の女性に郵送 → 同居する父から「なぜそのようなDMを送るのか」とクレームが入った。後に判明したのは、その女性が妊娠していたが家族に隠していたこと。精度が高すぎるレコメンドを属性に配慮せずダイレクトメール（家族全員が見られる媒体）で送ったことで、本人が隠したかった事実が公になってしまった。 ← S05

例2: 人事採用における書類選考の自動化リスク

シナリオ
過去の社員データから「出世する/しない」のクラス分類モデルを作成 → 履歴書が「出世する」に分類された場合のみ書類選考を通過とする自動化。← S03:p.21
利点問題
採用可否の自動化による時間短縮（採用担当者・応募者双方のメリット）使用したデータの偏りが採用に直接反映 → 意図しない差別の発生
音声補足（具体的な差別の発生）
例: もともと男性社員が多い企業で「出世する社員のデータ = ほぼ男性データ」となっている場合、そのデータでクラス分類モデルを作ると、どんなに優秀な女性の履歴書でも「出世するデータに当てはまらない」として書類選考で落とされる → 性別による差別が発生。どんな便利なAIでも、入れるデータが偏っていれば偏った結果が出る。 ← S05

例3: 画像認識技術における深層学習のリスク

シナリオ
深層学習による画像認識の精度向上 → 様々なサービスへ活用。← S03:p.22
利点問題
様々なサービスが実現（自動運転、顔認証、画像検索など）学習データの偏りにより、画像認識結果に人種差別的な出力 → 特定の人種に不利益・偏見を助長
音声補足（実際の事例）
ある画像認識サービスに「手に持つタイプの温度計を持った手」の画像を認識させたところ、黒い肌だと「銃器」と認識し、白い肌だと「単眼鏡（別の道具）」と認識するという問題が発生。学習データに人種的な偏見が含まれていたため、リリース後に人種差別的な出力をしてしまった。 ← S05

倫理が必要な理由：法律だけでは不十分

データサイエンスは「諸刃の剣」
データ・AIの利活用は大きな利益をもたらす可能性がある一方、使い方次第では不利益・課題（プライバシー侵害、個人情報漏えい、偏見の助長）をもたらす可能性もある。 ← S03:p.23

「法律や制度を守ればよい」という考え方では不十分な理由が3つある:

⇒
データ・AIを利活用した新技術・新サービスに対し、既存のルールがない・対応できない状況も十分に存在する。だからこそ、開発・利用する側に倫理的な判断が求められる。 ← S03:p.25

倫理的判断の方法：Salganikによる4つの原理

前提
倫理的判断を行うための普遍的な基準は存在しない。その時の状況・社会情勢によっても変わる。しかし、判断のための指針となる原理が提案されている。 ← S03:p.26
外部補足（Salganikとは）
Matthew Salganik（マシュー・サルガニック）はプリンストン大学の社会学者で、計算社会科学の分野を牽引する研究者の一人。本授業での4原理の引用元は、常松淳「第11章 計算社会科学における倫理」（鳥海不二夫 編『計算社会科学入門』丸善出版, pp.274-275, 2021）。Salganikの原著 "Bit by Bit: Social Research in the Digital Age" をベースに日本語で解説したものである。 ← S03:p.26脚注[1]
#原理内容データ・AI利活用の例
① 人格の尊重 一人ひとりの人格を尊重し、その人が行った自主的な判断を最大限に尊重する。尊重できる方法を確保する。 データ収集時の同意取得（オプトイン・オプトアウト）← 第7回で学習
② 善行 人に利益をもたらし、不利益を最小にする必要がある。 AIによる労働力不足解消と、「仕事が奪われる」問題のバランスを考える
③ 正義 特定の人だけが利益を享受し、残りの人が不利益を被る状況を避ける。 例: 学生の内定辞退率を本人の許可なく販売 → 販売した企業・購入した企業は利益を得るが、情報を売られた学生は不利益を被る（第7回のトラブル事例）
④ 法と公益の尊重 社会に対して技術・サービスについて十分な説明ができること。AIの出力結果を利用することへの社会的理解を得られるか考える。 深層学習などのブラックボックスモデルは「なぜその出力になったか」が説明できない → 説明可能なAI（XAI）の活用が特に重要視される。
近年はXAIが特に重要視される傾向がある。 ← S03:p.27
← S03:p.26,27, S05
【定義】説明可能なAI（XAI: eXplainable AI）
AIが出力した結果の根拠・理由を人間が理解できるよう説明可能にする技術・手法。近年特に重要視されている。 ← S03:p.27

第2章 まとめ

✅
  • データサイエンスは「諸刃の剣」= 大きな利益をもたらす一方で、不利益・課題の原因にもなる
  • 法律等だけでは使用の可否の判断が困難な場合も存在する
  • 説明可能なAI（XAI）の活用などによる対応が必要
← S03:p.28

生成AIとは

【定義】識別モデル
データを識別するAI。深層学習の発達により飛躍的に性能が向上した技術の一つ。例: 画像認識（何が映っているかを識別）。代表的なサービスとして Google Vision AI がある（猫の画像に対して「Cat: 96%」のように確信度付きで識別する）。 ← S03:p.32
【定義】生成モデル
入力された情報をもとに文章・画像などを生成する技術。深層学習を活用して急速に進歩した。
・文章生成に使われる → 大規模言語モデル（LLM: Large Language Model）
・画像・動画等を生成するために使われる → 拡散モデル ← S03:p.32
【定義】生成AI
生成モデルを使用したAIの総称。深層学習を使った生成モデルの研究は2014年頃から始まったとされており、まだ比較的新しい研究分野。 ← S03:p.32

主な生成AIサービス（2024年8月現在）: ← S03:p.33

分野代表的なサービス
言語生成ChatGPT, Claude, Gemini, Copilot …
画像生成Stable Diffusion, Midjourney …
音声生成VALL-E, Text-to-Speech AI …
音楽生成Stable Audio, Soundraw, Suno AI, AIVA …
動画生成Make-a-video, Runway Gen-3, Sora …

既存の製品・サービスに組み込まれているものも多数存在する。 ← S03:p.33

【定義】プロンプト
ユーザーが生成AIに伝える命令文（入力テキスト）。 ← S03:p.34
音声補足（ChatGPTの使い方・UI）
授業ではブラウザー版ChatGPTの画面を例に説明。画面下部のオレンジ枠（入力欄）にプロンプトを入力し送信すると、右側にAIが生成した文章が出力される。例として「大学生がデータサイエンスを学ぶメリットを3個、列挙してください」と入力したところ、自然で内容の良い回答が生成された。
ブラウザーで使用する方法のほかに、APIとしてプログラムに組み込んで使用する方法もある（開発者向け）。 ← S03:p.34, S05

② 文章生成のメカニズム

音声補足（文章生成の仕組み）
生成AIが文章を作成する基本的な仕組みは「次に来る確率が高い単語を選択し、それを繰り返す」こと。 ← S05
音声補足（計算が苦手な理由）
初期の生成AIは自然な文章は生成できるのに単純な計算問題が苦手という弱点があった。理由: 確率的に次の単語を選ぶ仕組みで動いているため、数値計算のような確定的な処理が本質的に苦手。 ← S05

留意点4つ

留意点① ハルシネーション（幻覚）

【定義】ハルシネーション（Hallucination / 幻覚）
生成AIが不正確な情報や誤解を招く情報を生成すること。 ← S03:p.41
音声補足（具体的事例：架空の郷土料理）
授業で実演されたエピソード: 「じゃがいもと鮭を使った沖縄県の郷土料理の作り方を教えてください」という意図的に無理のある命令をChatGPT 4oに入力したところ、「イモカナサ」という郷土料理（アレンジレシピとして）が出力された。続けて「イモカナサは沖縄県のどの地区の料理ですか」「レシピを紹介したウェブサイトを教えてください」と尋ねると、沖縄本島南部の料理と詳細な説明・サイト情報まで生成された。しかし実際にはイモカナサという料理は存在しない。紹介されたウェブサイトにも掲載はなかった。
架空の郷土料理の生成程度ならまだ笑って済ませられるが、プログラムのパッケージ名・医療情報・法律情報など、信頼性が重要な分野では深刻な問題につながる。 ← S03:p.40,41, S05
音声補足（対処法：「ハルシネーションを起こさずに」命令）
授業では対処策として、「ハルシネーションを起こさずに、イモカナサについて教えてください」と明示的に命令した例も紹介。すると生成AIは「「イモカナサ」という料理についての正確な情報は、沖縄の伝統料理としての明確な記録が見つかりませんでした。…一般的には知られていないようです」と正直に回答した。このようにハルシネーションを避けるよう明示的に指示することが一つの対処法になりうる。ただし常に効果があるとは限らないため、知らない情報は必ず信頼できるソースで確認する習慣が根本的な対策。 ← S03:p.42, S05
具体的な危険事例（セキュリティリスク）
生成したプログラムに存在しないパッケージ名が混入することが知られている。GPT-4では24.2%に存在しないパッケージ名が生成され、そのうち19.6%のパッケージ名が繰り返し生成されることが確認されている。もし悪意のあるプログラマーがこの傾向を知り、同名の悪意あるパッケージを公開すると、他の人のパソコンにウイルス・マルウェアを含むパッケージをインストールさせることができてしまう。
出典: The Register, "AI hallucinates software packages and devs download them", 2024年3月28日 ← S03:p.41脚注[1]
⚠️
生成AIは「何でも教えてくれる神様」ではない。知らない情報は必ず、信頼できる情報かどうかを確認する習慣が重要。 ← S05

留意点② 固定観念・偏見の混入

生成AIは過去のデータから学習した内容をもとに生成物を生成するため、学習データに固定観念や偏見が多く含まれると、生成物も偏った結果を出力しやすい。 ← S03:p.43

具体的な事例
「地図を持って道案内する人のイラストを作成してください」というプロンプトに対し、生成された4枚のイラスト全てが男性として描かれた。「男性の方が地図を読むのが得意」という固定観念が学習データに含まれていたと考えられる。 ← S03:p.43, S05

文章でも画像でも、固定観念や偏見が含まれる可能性が高いため、使用する場合は確認が必要。← S03:p.43

留意点③ 著作権侵害の可能性

多くの生成AIサービスでは著作権のある生成物を生成しないよう対策されているが、類似物が生成されてしまう可能性は十分にある。そのような生成物を使用すると、使用したユーザーが著作権侵害を問われる場合がある。著作権法等の最新情報を確認し、著作権侵害がないか注意する。 ← S03:p.44

留意点④ 個人情報の流出

プロンプトに入力した個人情報も生成AIの学習対象となる。過去には、入力した個人情報や機密情報が別の人の出力結果として出力されてしまった事件があった。 ← S03:p.44

  • 個人情報は入力しない
  • 入力内容を生成AIに学習させないよう設定を変更する

留意点⑤ 訓練データの時期（カットオフ）

訓練データには時間的な区切り（カットオフ）が存在する。訓練データに含まれない新しい情報は生成AIが生成することができない。 ← S03:p.45

音声補足（具体的なカットオフ日の例）
例: OpenAIのGPT-4oのドキュメントでは、訓練データの対象期間として「Up to Oct 2023（2023年10月まで）」と明記されている。つまり2023年11月以降の情報はGPT-4oの訓練データに含まれない。プログラミング言語の仕様はどんどん変化しているため、最新の変更情報を持たない生成AIからは古い仕様のプログラムしか出力されない可能性がある。生成された情報が最新の情報に基づいているかどうかを常に確認する必要がある。 ← S03:p.45, S05

第3章 まとめ

✅

生成AIの仕組み: 大量の訓練データを用いて大量の特徴を学習し、学習結果をもとに確率的に出力内容を生成する。

留意点5つ:

  1. ハルシネーション（幻覚）: 不正確・誤解を招く情報の生成
  2. 固定観念・偏見の混入: 学習データの偏りが生成物に反映
  3. 著作権侵害の可能性: 類似著作物が生成されユーザーが問われる可能性
  4. 個人情報の流出: プロンプト入力情報が学習対象になる
  5. 訓練データのカットオフ: 訓練データ以降の新情報は生成不可

活用の広がる生成AIだが、こういった留意点があることを念頭に適切に使うようにする。

  • DXの定義と主眼（最頻出）: DXは「デジタル技術を使うこと」ではなく「組織変革・経営の問題（組織文化の変化）」が主眼であり、データ・AIはその実現手段。経産省定義を正確に説明できるか。
  • デジタイゼーション・デジタライゼーション・DXの区別: 3つの違い（意味・変革規模：部門→組織→企業/社会全体）を具体例とともに区別して説明できるか。「デジタイゼーション ＜ デジタライゼーション ＜ DX」の変革規模の大きさも押さえる。
  • データサイエンスは「諸刃の剣」: この表現の意味（利益と不利益の両面）と具体的トラブル例3つ（レコメンド・採用自動化・画像認識）をそれぞれ説明できるか。
  • 倫理が必要な3つの理由: ①法律未整備（クローン羊の例）②問題後に罰則強化（ながらスマホの例）③既存法で対応不能（通信の秘密の例）を具体例とともに説明できるか。
  • Salganikの4原理（頻出）: 4つの名称と内容・データ利活用例を対応させて説明できるか。特に④「法と公益の尊重」とXAIのつながりは重要。
  • 生成AIの文章生成メカニズム: 「確率的な単語選択の繰り返し」という基本原理と、なぜ計算が苦手かの理由（確率選択なので確定的演算が苦手）を説明できるか。
  • ハルシネーションのセキュリティリスク: 存在しないパッケージ名の生成（GPT-4: 24.2%）→悪意あるパッケージ公開→マルウェアインストールという連鎖を説明できるか。
  • AI戦略2019のレベル区分: リテラシーレベル（全大学生必須）・応用基礎レベル・エキスパートの3段階と、本科目の位置付け（リテラシーレベルの網羅）を説明できるか。
  • 3力とドメイン知識: データサイエンス力・データエンジニアリング力・ビジネス力それぞれに必要な知識と、特にビジネス力でドメイン知識が重要な理由を説明できるか。
