02.第1章：ことばをデータにする

【第1章 学習目標】

  • テキストデータの基本的な前処理について説明ができる
  • テキストデータのベクトル化のメリットについて説明ができる

テキストデータとは

【定義】テキストデータ

自然言語で書かれたデータのこと。

【定義】自然言語 vs 人工言語

自然言語：人間が日常的に話す言語（日本語・英語など）
人工言語：人間が意図的に作った言語（プログラミング言語など）

テキストデータの例：アンケート調査の自由記述・SNS上のやりとり・ニュースサイトの記事など、インターネット上に大量に存在する。← S03:p.9

テキストデータの特徴（構造化データと異なる点）

  • データの長さが一定ではない（「こんにちは」1語も、長い文章も、どちらも「1文」）
  • 要素の並び順に文法による制限がある（語順が意味を変える）
← S03:p.9
🎤 音声補足

コンピュータは0と1（数値）しか扱えない。しかし画像・言葉・音声・音楽といった非構造化データは基本的に0と1では表現されていない。そのため、コンピュータで処理できるようにする「工夫」が必要であり、その工夫に数学が必要になる。← S05

📚 外部補足

構造化データとは、表（行×列）で整理されたデータのこと（例：Excelの表、CSVファイル）。これに対して、テキスト・画像・音声などは構造が決まっておらず非構造化データと呼ばれる。データサイエンスで扱うデータ量の約80〜90%は非構造化データと言われている。

データサイエンスに必要な数学

数学分野内容本回との関連
確率・統計確率論、統計的推定—（過去回で扱い済み）
線形代数ベクトル・行列★ 今回（第12・13回）のメインテーマ
解析学微分・積分—（機械学習で重要）
← S03:p.8, S05
🎤 音声補足

今回と第13回では、線形代数のベクトル・行列を「知っていることでデータサイエンスがどのように便利になるか」を体感してもらうことが目的。詳しい証明などは扱わない。← S05

自然言語処理（NLP）

【定義】自然言語処理（NLP: Natural Language Processing）

コンピュータでテキストデータを扱うための技術の総称。

NLPの応用例：

  • かな漢字変換・文法訂正・自然言語生成
  • 検索・機械翻訳・レコメンドシステム・質問応答（対話システム）
  • 文書の要約・分類・情報抽出・キーワード抽出
← S03:p.10
🎤 音声補足

スマホのフリック入力で数文字打つと次の単語が候補として出てくる「予測変換」や、英語→日本語の機械翻訳の精度が昔より大幅に向上しているのも、NLP研究の成果。← S05

テキストデータの前処理（3ステップ）

前処理が重要な理由：

  • データをコンピュータで分析しやすい形に整理する
  • データの「特徴」が明確になり、精度の高い分析結果につながる
  • 分析全体にかかる時間の中で高い割合を占める（重要かつ必須の工程）
← S03:p.11
🎤 音声補足

人によっては分析時間全体の8〜9割がこの前処理にかかると言われるほど、大変手間のかかる作業。しかし、この前処理の精度がそのまま最終的な分析精度に直結する。← S05

📊 前処理後の完成イメージ

例：文書 d₁「今日、私はデータサイエンスの授業を受けた。早朝に受けようと思ったが、急遽用事ができて…」

前処理後 →

d₁ = [1, 1, 1, …, 2, 2, 1, 0, …] ↑今日 ↑私 ↑DS ↑受け ↑た ↑早朝 ↑明日

テキストが数値のベクトルで表現される。← S03:p.12

ステップ①：テキストクレンジング

【定義】テキストクレンジング

分析の目的に応じて、表記の統一や不要な情報の削除・置換を行う処理。

処理内容の例：

種別具体例
表記の統一 「ことば」⇔「言葉」を一方に統一 / 半角「ﾃﾞｰﾀ」→全角「データ」
分析対象外の文字・記号の除去 「今日《きょう》は、」→「今日は」
個人情報・機密情報の除去・置換 「電話番号 080-1234-…」→「電話番号 000-0000-…」
← S03:p.13
🎤 音声補足

個人情報の置換を行わないと、例えば文章生成モデルを作った際に電話番号がそのまま出力されてしまう危険性がある。クレンジングは情報漏洩防止という意味でも非常に重要なステップ。← S05

ステップ②：トークン化（Tokenization）

【定義】トークン化

文章や文書に含まれる単語等を、意味の通る最小単位（トークン）に分割する処理。

言語による違い：

言語特徴処理方法
英語 単語間がスペースで区切られている スペースで分割すればよい（容易）
日本語・中国語 語の区切りが不明確 形態素解析が必要（専用ツール使用）
← S03:p.14
【定義】形態素解析（Morphological Analysis）

文章を意味のある最小の単位（＝形態素）に区切る処理。
「分かち書き」の結果と各形態素の品詞等が返される。

形態素解析の例（MeCabを使用）：

入力：「今日私はデータサイエンスの授業を受けた」

番号形態素品詞
1今日名詞
2私名詞
3は助詞
4データ名詞
5サイエンス名詞
6の助詞
7授業名詞
8を助詞
9受け動詞
10た助動詞
← S03:p.15
📚 外部補足

日本語形態素解析の有名なツール：
MeCab（めかぶ）・Janome・JUMAN など。
Pythonからも mecab-python3 などを通じて使用できる。

ステップ③：ベクトル化（Vectorization）

【定義】ベクトル化

非構造化データを数値（ベクトル）に変換することで、計算機で処理ができる状態にすること。

【定義】Bag of Words（BoW）

ある文書や文章に含まれるトークンの出現回数をカウントして数値化する手法。非構造化データをベクトル化するための基本的な方法の一つ。

BoWの例：「今日私はデータサイエンスの授業を受けた」

形態素解析 → 「今日」「私」「は」「データサイエンス」「の」「授業」「を」「受け」「た」 出現回数カウント（BoW） →「今日」=1, 「私」=1, 「は」=1, 「データサイエンス」=1, …
← S03:p.17

スカラーからベクトルへ：

数字1つ1つ = スカラー 例: 1, 2, 3, 5 1列に並べる = ベクトル 例: [1, 2, 3, 5]ᵀ 文書 d₁ = [1, 1, 1, 1, 1, 1, 1, 1, 1]ᵀ ↑今日 ↑私 ↑は ↑DS ↑の ↑授業 ↑を ↑受け ↑た

文書は単語の集合 → 数字の集合であるベクトルで表現するのが相性がよい。← S03:p.18-19

ベクトル化のメリット①：情報の圧縮・内容推定

文書 d₂ = [1(猫), 1(犬), 1(爬虫類), 1(飼育), 1(人気), …]

→ ベクトルの要素を見るだけで「最近人気のペットに関する内容」と予測できる。
単語情報があればある程度文書内容を予測できる＝情報を圧縮できる。← S03:p.20

ベクトル化のメリット②：文書間の類似度計算

文書 d₂ = [猫, 犬, 爬虫類, 飼育, 人気, …] 文書 d₂' = [室内, 飼う, 健康, 餌, 動物, …]

→ 似た文章には似た単語が使用されている → 単語同士の類似度を計算することで文書間の類似度を計算できる。← S03:p.21

第1章まとめ

学習目標対応内容
テキストデータの基本的な前処理について説明ができる クリーニング（表記統一・個人情報除去）→ トークン化（形態素解析）→ ベクトル化
テキストデータのベクトル化のメリットについて説明ができる ①情報の圧縮・内容推定が可能、②文書間の類似度計算が可能
← S03:p.22

03.第2章：ベクトルの基礎

【第2章 学習目標】

  • ベクトルの簡単な演算ができる
  • ベクトル空間について簡単な説明ができる

ベクトルと行列の基本

【定義】スカラー・ベクトル・行列

スカラー：数値1つ1つがバラバラな状態（例: 1, 3, 5）
ベクトル：スカラーを1列に並べたもの。方向と大きさ（長さ）を持つ。
行列：1列に並べた複数のベクトルを横に結合したもの（第13回で扱う）

【定義】ベクトルの次元

ベクトルの要素の数のこと（空間的な「次元」とは異なる概念）。

[2, 3]ᵀ → 2次元ベクトル（要素が2個） [1, 2, 3]ᵀ → 3次元ベクトル（要素が3個） [1, 2, 3, 4]ᵀ → 4次元ベクトル（要素が4個）
← S03:p.27
🎤 音声補足

「2次元は平面、3次元は空間」というイメージがあるかもしれないが、ここでは空間的な意味は一切ない。ベクトルの要素数がそのまま次元数。まずはこの理解で十分。← S05

構造化データもベクトルの集まりとして見られる：

AさんBさんCさんDさん
身長157165180172
体重45628265
性別1001
握力22465331
50m走8.26.87.27.6
幅とび153237221167
Aさんのベクトル = [157, 45, 1, 22, 8.2, 153]ᵀ （6次元ベクトル）
← S03:p.28-29

大学レベルのベクトル表記：

ベクトル → 太文字 a（高校では矢印付き） 要素 → 細文字 a₁, a₂, …, aₙ

かっこは [ ] でも ( ) でも同じ。転置記号 ᵀ を使って行ベクトルを列ベクトルとして表記することもある。← S03:p.30

ベクトルの計算（足し算・定数倍）

① 足し算（ベクトルの和）

[a₁] [b₁] [a₁+b₁] [a₂] + [b₂] = [a₂+b₂] [⋮ ] [⋮ ] [ ⋮ ] [aₙ] [bₙ] [aₙ+bₙ] ⚠️ 同じ次元のベクトルどうしでしか計算できない（2次元+3次元は不可） 図形的な意味：各ベクトルをつなぎ合わせたときの「最短距離」がベクトルの和
← S03:p.31, 33-34

② 定数倍（スカラー倍）

[b₁] [αb₁] α × [b₂] = [αb₂] [⋮ ] [ ⋮ ] [bₙ] [αbₙ] 各要素にαを掛けるだけ。 図形的な意味：方向はそのまま、長さがα倍になる。 マイナス倍の場合は180°逆方向になる。
← S03:p.32, 35-36
🎤 音声補足

高校でベクトルを習ったとき「ベクトルはどこにあっても同じ」という説明があったと思う。ベクトルは長さと方向だけを持つので、要素は座標の意味ではない。場所を移動しても、長さと方向が同じなら同じベクトル。← S05

計算例：

a = [2, 1]ᵀ, b = [1, 3]ᵀ の場合 a + b = [2+1, 1+3]ᵀ = [3, 4]ᵀ a = [1, 1]ᵀ を2倍すると 2a = [2, 2]ᵀ
← S03:p.38-39

ベクトルの計算③：内積（Dot Product）

【定義】内積

2つのベクトルの各要素を掛け合わせて和を取る演算。結果はスカラー値（1つの数値）になる。

要素による計算（ドット記号で表記）：

a · b = a₁b₁ + a₂b₂ + … + aₙbₙ 例) [1,2,3] · [5,4,1] = 1×5 + 2×4 + 3×1 = 5 + 8 + 3 = 16
← S03:p.40, 43-44

長さと角度による計算：

a · b = ‖a‖ × ‖b‖ × cosθ ここで ‖a‖ = √(a₁² + a₂² + … + aₙ²) ← ノルム（ベクトルの長さ） 例) ‖a‖ = 3, ‖b‖ = 2, 成す角θ = 30°のとき a · b = 3 × 2 × cos30° = 6 × (√3/2) = 3√3
← S03:p.40-44
🎤 音声補足

内積は「要素を使った計算」と「長さと角度を使った計算」の2通りがある。両方が等しいことを使うと、cosθを求めること（＝ベクトル間の角度を求めること）ができる。これがこの後のコサイン類似度につながる重要なポイント。← S05

ベクトル空間

【定義】ベクトル空間

複数のベクトルの根本を1か所にまとめたベクトルの集合。
空間内のベクトルはすべて同じ次元（要素数）でなければならない。

テキストデータとベクトル空間：

前処理でテキストデータをベクトル化することの目的は、文書や単語をベクトル空間上に配置するため。これにより、自然言語の解析や応用的な処理が可能になる。← S03:p.48

第2章まとめ

学習目標対応内容
ベクトルの簡単な演算ができる 足し算・定数倍・内積（要素計算 + 長さ×角度計算）
ベクトル空間について説明ができる 同じ次元のベクトルの集合。文書・単語をここに配置するのが前処理の目標
← S03:p.49

04.第3章：ベクトルの活用

【第3章 学習目標】

  • コサイン類似度の意味と活用例について説明ができる
  • ベクトルを用いたアナロジーについて簡単な説明ができる

コサイン類似度（Cosine Similarity）

🎤 第2章の復習ポイント

内積の2つの計算式（要素版・長さ×角度版）が等しいことを使うと、cosθが求まる。
a·b = ‖a‖‖b‖cosθ → cosθ = a·b / (‖a‖‖b‖)← S05

【定義】コサイン類似度

2つのベクトル間の角度θのcosθ値を使って、類似度を表現したもの。

コサイン類似度 = cosθ = (a · b) / (‖a‖ × ‖b‖) 値域：-1 ≤ cosθ ≤ 1 ・cosθ → 1 （角度 → 0°）：2ベクトルが同じ方向 → 類似度が高い ・cosθ → -1 （角度 → 180°）：2ベクトルが逆方向 → 類似度が低い
← S03:p.54-55
🎤 音声補足

内積の計算式をcosθについて式変形しただけだが、わざわざ「コサイン類似度」という名前がつくほど重要な概念。ベクトル間の角度で類似度を評価できるという発想は、データ分析の中で非常に重要。← S05

コサイン類似度の活用例

設定：3種の文書A・B・CについてBoWで「ケーキ」と「ゲーム」の出現頻度をカウントしてベクトル化した。

文書 a = [1, 2]ᵀ （ゲーム:1回, ケーキ:2回） 文書 b = [3, 4]ᵀ （ゲーム:3回, ケーキ:4回） 文書 c = [2, 1]ᵀ （ゲーム:2回, ケーキ:1回）

→ 「内容が最も似ている文書はどれか？」をコサイン類似度で判断する。← S03:p.56

各文書間のコサイン類似度の計算：

AとBのコサイン類似度: a·b = 1×3 + 2×4 = 11 ‖a‖ = √(1²+2²) = √5, ‖b‖ = √(3²+4²) = 5 cosθ_AB = 11 / (√5 × 5) = 11/(5√5) ≈ 0.98 AとCのコサイン類似度: a·c = 1×2 + 2×1 = 4 ‖a‖ = ‖c‖ = √5 cosθ_AC = 4 / (√5 × √5) = 4/5 = 0.8 BとCのコサイン類似度: b·c = 3×2 + 4×1 = 10 ‖b‖ = 5, ‖c‖ = √5 cosθ_BC = 10 / (5√5) = 2/√5 ≈ 0.89
← S03:p.57-59

結論：

文書ペアコサイン類似度評価
A と B≈ 0.98★ 最も類似度が高い
B と C≈ 0.89—
A と C= 0.80最も低い

→ 文書Aと文書Bが「内容が最も似ている文書ペア」とみなせる。← S03:p.60

🎤 音声補足

文書Aは「ゲーム1回・ケーキ2回」、文書Bは「ゲーム3回・ケーキ4回」。Bはケーキの話題に重点があるという点でAと同じ傾向。文書が長くなっただけで内容は同じようなもの → コサイン類似度が高くなった。← S05

📚 外部補足

コサイン類似度はテキストデータだけでなく、協調フィルタリング（レコメンドシステム）にも応用できる。ユーザーの行動履歴をベクトル化し、コサイン類似度でユーザー間の類似度を計算することで「あなたへのおすすめ」を実現できる。

単語の分散表現・Word2Vec

BoWの問題点：

例：「家を探している」と「物件を探している」は同じ意味だが、BoWでは別の単語として扱われ類似度が低くなってしまう。

→ 単語そのものの意味（意味的類似性）を考慮した手法が必要。← S03:p.61

【定義】単語の分散表現（Distributed Representation）

単語をベクトルで表現したもの。似た単語が近くに配置されるようにベクトルを作成する。

【定義】Word2Vec

単語をベクトルに変換する手法の総称（word to vector）。
2つの代表的なモデル：CBOW（Continuous Bag of Words）とSkip-gram。

Word2Vecの効果：

似た単語（例: ショートケーキ・チョコケーキ・モンブランなど）がベクトル空間上で近くにまとまって配置される。情報端末（スマートフォン・パソコン等）も同様に近くにまとまる。← S03:p.61

アナロジー（類推）とベクトル計算

【定義】アナロジー（Analogy / 類推）

ある単語間の関係から、似た関係にある他の単語を導くこと。

例：「日本では東京」の場合、「フランスでは何？」

→ 「ある国の首都」という関係 → 答えは「パリ」

Word2Vecを使うとこれをベクトル計算で導出できる。← S03:p.62

数式によるアナロジーの表現：

【日本と東京の関係】 a_日本 + b = a_東京 ∴ b = a_東京 - a_日本 … ① 【フランスとパリの関係】 a_フランス + c = a_パリ ∴ c = a_パリ - a_フランス … ② 【b = c と仮定（同じ関係）】 a_東京 - a_日本 = a_パリ - a_フランス 整理すると： a_フランス - a_日本 + a_東京 = a_パリ ← アナロジー完成 → Word2Vecのベクトル空間に 「フランス - 日本 + 東京」を入力すると「パリ」が出てくる！
← S03:p.64-66
🎤 音声補足

言葉で「日本では東京の場合、フランスでは何？」という問いが、数式として「フランス - 日本 + 東京 = ?」で表現できる。これがテキストデータをベクトル化することのすごさ。問題文の中に「パリ」という答えは一切出てきていないのに、ベクトル計算で導出できる。← S05
